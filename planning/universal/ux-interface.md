# 界面与交互设计

> 来源：《体验引擎》第9章

## 象征与象征资源
- **来源**: 《体验引擎》第9章
- **核心**: 用玩家熟悉的外壳包装陌生的游戏机制，降低学习成本。
- **详述**: 象征（Metaphor）是指把一个我们所熟悉的外表给予一个新事物，使它更容易被理解。游戏中所有的虚构层本质上就是一个庞大的象征——我们把复杂的游戏机制用城市、战争、冒险等外壳包装，使玩家能够迅速理解系统关系。

  象征资源有多种来源：现实事物（文件夹、汽车）、文化原型（黑色代表死亡、方下巴代表强壮）、游戏约定（食物治愈伤口、黄金铠甲更坚固）、逻辑体系（牛顿物理、供需经济）以及最抽象的数学/空间系统（棋盘格子利用人类天生的空间思维本能）。其中，对物理和空间系统的象征几乎是最强大的，因为它几乎零学习成本。

  设计师必须构建一套"象征词汇表"，明确哪些虚构元素对应真实机制，并在整个游戏中保持一致。如果象征含糊不清（如《塞尔达传说》中可炸裂的裂缝墙），玩家可能在误解与迷惑之间放弃游戏。
- **适用场景**: 设计新游戏系统或UI元素时，需要让玩家快速理解其功能。
- **示例**: 《波斯王子：遗忘之沙》创建了环境线索词汇表——带有长条纹标记的墙面可贴墙跑，特别突起的砖块可攀爬，且这些线索在整个游戏中保持外观一致。

## 信号与噪声
- **来源**: 《体验引擎》第9章
- **核心**: 一切未能向玩家传递有意义信息的表现都是噪声，必须被消除。
- **详述**: 玩家得到的每种信息都是某个信号的一部分。噪声是指未能传递有意义信息的一种信号——不只是电话杂音那种无意义信号，而是游戏中所有无法向玩家内心传递有意义信息的信号。无论是因为审美因素遮蔽、需要超高技巧才能解读，还是与其他信号纠缠过密，结果都一样：玩家不理解游戏事件，因此不会响应它。

  噪声主要有两种成因。第一是复杂的美术效果：灰盒原型中清晰的敌人和路线，在添加美术资产后反而变得难以辨认，因为绚丽的线条和颜色淹没了有意义的图形。第二是过多的信号：当屏幕上同时出现太多信息时，玩家会遗漏重要的内容。
- **适用场景**: 美术效果完成后玩家频繁"看不到"关键元素时，应排查视觉噪声。
- **示例**: 《军团要塞2》使用卡通风格和简单纹理，《传送门》使用白色墙壁实验室场景，《镜之边缘》使用白色混凝土城市——这些美术风格都在保证美观的同时自然地将噪声降到最低。

## 视觉层次
- **来源**: 《体验引擎》第9章
- **核心**: 让所有信息同时可见，但按重要性赋予不同可见度，使玩家自动按需接收。
- **详述**: 视觉层次（Visual Hierarchy）借鉴自平面设计。其核心是：所有东西都立即出现，但更重要的信息更加显眼，以便人们在第一时间注意到它们。人们会下意识地先注意到体积大、距离近、色彩明亮、速度快的东西。

  这一方法的最大优势在于它能自适应不同水平的玩家。新手只会注意到最显眼的信息（如屏幕中央的巨大威胁），自动忽略小地图和弹药数。随着技巧提升，玩家会逐步察觉第二、第三层信息。设计师只需确保每种信息的相对可见度与其重要性匹配，就能让所有水平的玩家都看到当下最重要的东西。

  重要的是相对可见度而非绝对可见度。如果所有信号都加强，等于没有加强。
- **适用场景**: 不同水平的玩家需要不同信息密度的游戏，尤其是射击、动作类实时游戏。
- **示例**: 典型射击游戏中：近处敌人最大最醒目，远处敌人较小；生命值在危急时才出现红色覆盖层和悲苦音效；弹药数在角落安静显示，快打光时才提高可见度。

## 冗余
- **来源**: 《体验引擎》第9章
- **核心**: 用多种方式重复传递关键信息，确保即使玩家错过一半内容也能理解重点。
- **详述**: 玩家可能在看别处、接电话、和朋友拌嘴——他们会在玩游戏的同时做任何事。如果漏掉了重要信息，他们仍会责怪设计师。强制玩家注意（锁定镜头、弹对话框）会破坏流程和沉浸感。

  冗余的解决方案是接受玩家可能漏掉信息，然后用多种方式多次传递。同类冗余用同样方式多次传递（5个地方放同样声音提示）。可变冗余用不同渠道传递同一信息——如同伴喊话（对话）、角色挥手（动画）、突出木板（关卡视觉）、HUD标记四个渠道同时引导玩家跳出窗口。被动冗余更精细：只有当玩家漏掉第一条信息时，才展示第二、第三条提示。
- **适用场景**: 有关键信息必须传达、但无法强制玩家注意的场景。
- **示例**: 射击游戏中引导玩家跳出窗口：同伴大声喊话 + 角色朝窗口挥手 + 地面放置突出木板 + HUD标记四重冗余，确保即使高度紧张的玩家也能注意到。

## 间接控制
- **来源**: 《体验引擎》第9章
- **核心**: 在玩家不知情的情况下，通过信息排列方式自然引导其行为。
- **详述**: 间接控制（Indirect Control）不是潜意识的诡计，而是UI和工业设计师使用数十年的简单方法。核心思想：不直接命令玩家做什么，而是让期望的行为成为最自然的选择。有三种基本方法——微影响、灌输、社会模仿——它们可以单独使用，也可以组合使用来引导玩家走向设计意图的体验路径。

  间接控制之所以重要，是因为直接命令（锁定镜头、弹窗提示、屏蔽操作）会破坏沉浸感和游戏流程。而间接控制让玩家感觉一切都是自主决定，这正是游戏体验的核心——代理权。
- **适用场景**: 希望玩家采取特定行动但不想破坏沉浸感时，如引导探索路线、暗示解谜思路。
- **示例**: 综合使用微影响（明亮门口吸引玩家）、灌输（展示交谈场景暗示非暴力互动）、社会模仿（NPC先跳过缺口，玩家自然跟随）等方法引导玩家。

## 微影响
- **来源**: 《体验引擎》第9章
- **核心**: 不改变选项内容，只改变选项的呈现方式来影响玩家选择。
- **详述**: 微影响（Nudging）利用了人类"选择阻力最小的路"的倾向。玩家会选默认选项、走最直接的路线。设计师只需要调整可选项的顺序、默认值和视觉呈现，就能自然地引导行为。

  视觉设计是最常用的微影响手段：灯火通明的门口比昏暗的更吸引人，地板上的线条暗示路线，闪烁的按钮邀请点击。关键是：我们没有强迫玩家，他们只是在最自然的选择中恰好走了设计意图的路线。
- **适用场景**: 需要引导玩家选择但不想限制自由的场景，如对话选项、路线引导。
- **示例**: 对话系统把最有趣的结果设为默认选项；明亮的出口自然吸引玩家走设计预期的路线。

## 灌输
- **来源**: 《体验引擎》第9章
- **核心**: 通过激活玩家脑海中的某些概念，间接影响其后续数分钟内的行为。
- **详述**: 灌输（Priming）利用的是概念激活效应。当人看到战争画面，脑海中会浮现战争相关概念；看到小猫照片，战争就被毛茸茸的感觉取代。这些被激活的概念会在接下来几分钟内改变人的行为——战争内容让人更激进，小猫内容让人更关爱。

  在游戏中，灌输从游戏开始前就已发生：游戏名字、封面图片、他人评价都是最早的灌输因素。游戏过程中，看到医生联想到治疗，看到骷髅联想到死亡，看到交谈场景产生社交意愿。每种印象都会影响玩家数分钟后的决策。但灌输不能凭空创造动机，只能在已有选项间切换。
- **适用场景**: 需要让玩家在特定情境下倾向于某种行为（如非暴力互动而非射击）。
- **示例**: 射击游戏中，玩家即将与线人会面。为防止玩家直接射杀线人，设计师让角色自然放低枪口、展示其他NPC交谈场景、同伴说"让我来和他谈"——组合灌输非暴力的交互意图。

## 社会模仿
- **来源**: 《体验引擎》第9章
- **核心**: 玩家会本能地效仿他人（包括NPC）的行为，设计师可借此引导玩家。
- **详述**: 社会模仿（Social Imitation）基于人类通过效仿他人学习的强大本能。在游戏中，玩家会做出和他人一样的举动——多人游戏中模仿其他玩家的策略，单人游戏中模仿NPC的行为。

  这使NPC成为一种强大的间接控制工具。让计算机控制的赛车在弯道前减速，玩家也会学会减速。展示对手持有的商品，玩家会意识到自己策略的差异。同伴角色在游戏中不仅有叙事功能，更是引导玩家的宝贵工具——让同伴跑向下一个目标、做某个谜题、躲避危险，玩家就会下意识地跟随。
- **适用场景**: 需要教导玩家新机制或引导行为时，尤其适合有NPC同伴的游戏。
- **示例**: 赛车游戏中AI在弯道前减速引导玩家减速；同伴NPC率先跳过缺口，玩家自然跟随跳过。

## 操作布局
- **来源**: 《体验引擎》第9章
- **核心**: 物理控制器与游戏行为之间的映射越直观，学习成本越低。
- **详述**: 操作布局依赖两条关键原则：映射（Mapping）和操作互斥（Control Exclusivity）。映射是指在物理控制与游戏效果之间建立相似性，使其成为内置记忆方法。映射不限于物理位置，还可以通过形状、颜色、移动等数百种信号建立。

  操作互斥是指互斥的物理操作必须映射到互斥的游戏行为上。如果两个按键只能用同一根手指操作，它们就不应映射到需要同时使用的游戏行为上。当物理互斥正好反映游戏机制的互斥时，玩家会自然学会这种关系。

  映射对新手尤其重要。Wii的动作控制器、Kinect体感系统、《摇滚乐队》的吉他控制器，都是通过建立高相似度映射来吸引休闲玩家的昂贵物理界面。
- **适用场景**: 设计按键配置、手柄布局或体感操作映射时。
- **示例**: 《生化奇兵》中左手质体对应左触发器、右手武器对应右触发器——玩家真实的手和角色手的映射；Xbox手柄上bumper和trigger用同一食指，完美映射了"不能同时换武器和攻击"的互斥关系。

## 操作感
- **来源**: 《体验引擎》第9章
- **核心**: 通过精心调整的物理规则（而非真实物理）创造出令人愉悦的操作体验。
- **详述**: 操作感（Control Feel）是通过界面传递意图而带来的即时体验。宫本茂在1985年的《超级马里奥》中展示了操作感的极致：马里奥跳跃的每一条规则都经过精心调整，与现实物理完全不同——按键时间决定跳跃高度、上升阶段重力最小而最高点重力变为三倍、下落速度有上限、跑动跳跃更高、着陆前可以预输入下次跳跃。

  这些看似荒谬的复杂规则是唯一能创造良好操作感的方法。有些规则提升技巧上限（跑动增加跳跃高度），有些消除挫败感（预输入系统）。单独看每条规则只有细微差异，但结合起来使用成千上万次后，体验会大幅超越所有竞争对手。

  这是优雅原则的例外——大量且不可见的复杂度，但玩家完全没有成本，因为他们根本感觉不到这些规则的存在。
- **适用场景**: 核心操作反复执行（跳跃、射击、驾驶）的游戏，需要长时间打磨手感。
- **示例**: 《超级马里奥》的跳跃系统通过十多条精心调整的非现实物理规则，让仅用一个按键操作的感觉远超竞争对手。马里奥系列能赚大钱，正是因为控制马里奥本身就是一种乐趣。

## 输入协助
- **来源**: 《体验引擎》第9章
- **核心**: 悄悄猜测玩家意图并修正其输入，让玩家永远察觉不到受到了帮助。
- **详述**: 输入协助（Input Assistance）是对玩家原始输入的预处理。其核心思想是巧妙猜测玩家意图并悄悄影响输入，理想情况下玩家永远不会意识到自己接受了协助。

  以瞄准协助为例：《光环》在2001年展示了如何让射击游戏在家用主机上焕发光彩。其瞄准协助由多个互相交错的子系统组成——一个子系统将部分摇杆输入转换为追踪移动目标，另一个在松开摇杆时悄悄把准星移向附近目标，还有一个在准星划过目标时轻微调整方向。关键在于只影响玩家本来就察觉不到的变化（少移动几个像素、稍微快一点），所以玩家完全不会察觉。

  输入协助还可作为平衡手段。《光环》中狙击枪近距离取消瞄准协助，隐性增加了近距离使用难度，引导玩家按设计意图远距离使用。
- **适用场景**: 物理控制器精度不足以支撑设计意图时，如主机手柄射击、平台跳跃着陆、赛车防滑。
- **示例**: 《光环》的瞄准协助让家用主机射击游戏成为可能；跳跃协助帮助角色安全着陆；攻击协助帮助玩家干净利落地发起攻击。这些系统在玩家不知情的情况下运行，让操作感觉自然流畅。

## 操作延迟
- **来源**: 《体验引擎》第9章
- **核心**: 从输入到反馈的延迟决定了玩家必须提前多久预测，直接影响操作的同步感和技巧上限。
- **详述**: 操作延迟（Control Latency）是指从游戏接受输入到对应反馈显示出来的时间。由于多重渲染管道的硬件架构，即使没有编程错误，延迟也至少有3~4帧。再加上电视信号处理，实际延迟可能达到100~183毫秒。

  延迟的影响是：玩家必须学会提前几帧输入。延迟越长，预测时间越长，技巧要求越高，游戏感觉越不同步。帧率是控制延迟的核心手段：30fps下总延迟约100~133ms，60fps下降至50~67ms。

  选择30fps还是60fps取决于设计需要。强调美术或故事的游戏（回合制策略等不需要快速输入的类型）适合30fps，让出处理器资源给画面。射击、赛车、格斗等高水平玩家交互且要求快速响应的游戏则需要60fps。但60fps意味着每帧只有1/3的处理器机能可用，这是一个涉及图像质量和响应速度的代价不菲的取舍。
- **适用场景**: 确定游戏帧率目标、优化输入响应、平衡画面质量与操作手感时。
- **示例**: 《雷神之锤3》竞技玩家用极高配置追求200fps以获得最大竞争优势；射击游戏、赛车游戏和格斗游戏是最常见需要60fps的类型；强调剧情的游戏则可选30fps换取更好的画面表现。

## 信息通道与维度的界面架构
- **来源**: 《游戏设计艺术》第15章/透镜66
- **核心**: 界面设计的核心是选择如何将游戏信息映射到通道（位置/媒介）和维度（大小/颜色/字体等）——优雅的映射用单一通道传递多种信息。
- **详述**: 设计界面的四步法：第一步列出所有玩家需要的信息并按重要性排序（每一刻都需要→时不时一眼→有时需要）；第二步列出可用的信息通道（屏幕上方、右下角、音效、角色头上等）；第三步将信息映射到通道（最重要的信息放在最显眼的通道）；第四步检查每个通道的维度使用（一个伤害数字可以同时传达数值、颜色表示敌友、大小表示剩余血量）。风险在于多重维度需要教育玩家理解，但收益是优雅的信息密度。一条通道上用多个维度强调同一信息可以使其"多汁"——小伤害白色小号，中等伤害黄色中号，致命伤害红色巨大。
- **适用场景**: 设计HUD布局、战斗信息显示、多层级UI时。
- **示例**: 《塞尔达传说》的HUD将最重要的当下环境占据主屏幕，次重要的红宝石/钥匙/体力/武器放在上方控制面板，偶尔需要的物品清单放在需要按键切换的子屏幕——完美的信息优先级排序。

## 原始性——利用大脑的底层本能
- **来源**: 《游戏设计艺术》第15章/透镜65
- **核心**: 界面和游戏活动的"原始性"（连动物都能做的操作）决定了它的直观程度——触摸比鼠标更原始，收集果实比解数学题更原始，利用原始本能的操作天然直观。
- **详述**: 触摸界面之所以如此直观（连小孩都能上手），不是因为"高科技"而是因为它利用了动物在3-4亿年前就有的本能行为。使用工具（键盘/鼠标/手柄）需要大脑新皮层参与，而触摸只涉及最底层的"爬行动物"大脑。这个原理延伸到游戏设计：收集水果类元素、和敌人作战、在不熟悉环境中寻路——这些都是如此原始连动物都可以做的活动，所以它们天然直观。判断原始性的简单方法：动物能操作它吗？如果能，你很可能已经在利用原始的力量。
- **适用场景**: 选择输入方式、设计核心交互行为、评估操作方案的直观性时。
- **示例**: 《愤怒的小鸟》的成功部分源于核心交互（手指拉弹弓然后松开）极其原始——连猫都会用爪子拨弄东西再松手。

## 强化道具设计七问
- **来源**: 《通关！游戏设计之道》第13关
- **核心**: 设计强化道具时必须回答七个核心问题：用途、外观、组合性、影响、通知方式、妥协、消退提示。
- **详述**: 强化道具是"高度浓缩的行动"，设计时需要回答：它有什么用？长什么样（发光/旋转/跳跃）？能否互相组合？怎样影响玩家行动（速度/攻击/生命值）？影响如何让玩家知晓（视觉变化/声音提醒）？使用它需要什么妥协（速度降低/行动减少）？临时效果消退时用什么线索提醒（HUD元素/视觉效果/声音/音乐）？强化道具分四类：防御型（生命回复、无敌、防护）、攻击型（弹药补给、武器强化、伤害加成）、运动型（改变速度、特殊技能、改变体积）和大逆转型（改变分数/财宝、改变状态、磁力、隐身）。每类都有独特功能但设计原则一致。
- **适用场景**: 设计任何游戏内的道具系统、buff机制和可收集物品。
- **示例**: 《超级马里奥》的超级蘑菇（改变体积+多挨一下）、《吃豆人》的能量豆（改变状态让玩家可攻击幽灵）、《马里奥赛车》的超级星星（无敌+自动消灭碰到的敌人）。

## 动态难度平衡——给玩家真正需要的东西
- **来源**: 《通关！游戏设计之道》第13关
- **核心**: 根据玩家表现动态调整挑战与奖励，给玩家真正需要的东西助其走向成功，而非固定不变的难度。
- **详述**: 动态难度平衡（DDB）是一种根据玩家表现实时调整的方法。如果玩家在战斗中死了多次，就减少敌人血量或降低攻击频率；如果玩家开宝箱时生命危急，就爆出回复道具，如果血量充足就给宝藏。DDB的目标不是降低游戏难度，而是确保每个玩家都能体验完整内容。相关机制还包括：难度调节（发现死亡次数过多时给玩家调低难度的机会）、橡皮筋作用（赛车游戏中落后太多时对手减速）、自动保存（定期自动存档让玩家专注游戏本身）。
- **适用场景**: 希望让各种水平的玩家都能通关并享受游戏时。
- **示例**: 《马里奥赛车》的橡皮筋机制——玩家落后时对手自动减速，避免空荡荡赛道上独自奔跑的无聊体验；反之如果玩家领先太多，对手也会加速追赶保持紧张感。

## 奖励系统设计——频繁、多样、有意义
- **来源**: 《通关！游戏设计之道》第13关
- **核心**: 在游戏早期就告诉玩家有什么奖励，尽快奖励、经常奖励、制造变化和惊喜，并且让奖励对后续进程有意义。
- **详述**: 奖励是玩家的核心动力之一，设计原则：在游戏早期就展示奖励系统让玩家规划目标；尽快且经常地奖励玩家，制造变化和惊喜；胜利展示要夸张和激动人心（摄影机环绕、角色跳跃、视觉特效）；奖励必须在后续关卡中有意义——最好的奖励是得到之前玩家根本没意识到自己需要的东西。奖励形式包括：得分（适合竞技类游戏）、成就（引导玩家尝试新玩法）、财宝（需要配套经济系统）、战利品（最好的奖励最难赢得）、纪念品（在玩家基地展示冒险记忆）、惊喜和乐趣（纯粹为了好玩的小时刻）。
- **适用场景**: 设计游戏整体奖励循环、成就系统和经济系统。
- **示例**: 《宝岛Z：红胡子的宝藏》在玩家正确完成操作时同时触发：胜利音效、角色反应动画、粒子特效、助手祝贺文字——多维度称赞让玩家感觉"自己是世界上最聪明的玩家"。

## 输入过滤——在信号进入模拟前塑造手感
- **来源**: 《游戏感》第16章
- **核心**: 输入信号在进入物理模拟之前应先经过一层过滤（引力、敏感度、阈值），用ADSR包络图来塑造操作的"上冲-保持-释放"感受，这是手感调优的第一道关卡。
- **详述**: 《Raptor Safari》中，虽然使用键盘的简单两态信号，但Unity的输入管理器会为每种输入施加引力（回到中心的速度）、敏感度（到达目标的速度）和快速变值（反方向输入时重置为0）三个过滤参数。结果是转向输入不会瞬间到达最大值，而是平滑上冲然后保持；松开按键后也不会瞬间归零，而是以引力速度平滑释放。这种过滤让即使是离散的键盘输入也能产生类似摇杆的连续感。上冲阶段的速度（敏感度）决定了操作的"紧致感"——敏感度高则响应灵敏但可能过于生硬；释放阶段的速度（引力）决定了操作的"松弛感"——引力低则操作更平滑但感觉更沉重。调整这三个参数是塑造不同操作风格（赛车vs飞机vs角色）的基础手段。
- **适用场景**: 调整操作手感的第一步——在进入物理模拟之前先用输入过滤塑造基本的响应曲线。
- **示例**: 《Raptor Safari》中转向的敏感度=2、引力=3，带来平滑但不迟钝的转向感；前进的敏感度=3，带来更快的加速响应。

## 操作歧义的识别与解决
- **来源**: 《游戏感》第17章
- **核心**: 同一组输入映射到多个不确定的结果是操作手感的大敌——必须为每种输入组合确定唯一结果，否则玩家的练习将毫无意义。
- **详述**: 当同时按下A键和Z键可能得到落地重击、后空翻或普通跳跃中的任意一种时，玩家会反复做同一件事却得到不同结果，这打破了"可预测的结果"原理。解决操作歧义需要：首先识别所有可能的输入组合，检查是否存在交叉映射；然后为每个组合确定优先级或使用时间差区分（例如先按Z再按A是后空翻，先按A再按Z是落地重击）。设计师必须以毫秒为单位去思考输入时序，但玩家感知不到毫秒级的差别。最好的解决方案是让不兼容的操作使用不重叠的输入组合，从根源上消除歧义。
- **适用场景**: 设计复杂的操作映射时，特别是有状态敏感输入和时间敏感输入的系统。
- **示例**: 《超级马里奥64》的落地重击/后空翻键位冲突是经典反面案例——Mick West指出这种歧义让玩家"永远都不清楚如何才能正确操作"。

## 状态切换的可感知性
- **来源**: 《游戏感》第17章
- **核心**: 当游戏有大量不同状态时，状态切换必须对玩家有明显且清晰的感知信号，否则输入结果会显得随机——玩家在不知情的状态切换中完全无法预测操作结果。
- **详述**: 《超级马里奥兄弟》中空中状态和地面状态的差异是容易感知的——角色明显在空中，方向键的响应变弱是合理的。但《托尼霍克》有空中、地面、Manual、手持滑板、滑行、花式动作等大量状态，每个状态下12个按键的意义都不同，新手完全无法感知当前处于哪种状态。一旦输入产生太多可能结果，最合理的反应就是随机按键。格斗游戏的新手行为就是典型——没有清楚意图地随机按键，因为状态太多太复杂。设计状态系统时，要么控制状态数量，要么为每次状态切换提供极其明显的视觉/听觉反馈，确保玩家始终清楚"我现在能做什么"。
- **适用场景**: 设计多状态的操作系统时，确保每个状态的进入和退出都有不可忽视的感知信号。
- **示例**: 《马里奥》的空中状态通过角色明显离地来传达；《托尼霍克》对新手是状态混乱的典型——12个按键在6+种状态下有完全不同的含义。

## 称赞玩家的艺术
- **来源**: 《通关！游戏设计之道》第13关
- **核心**: 游戏设计师应成为"玩家身后的温柔推手"，用多种形式的称赞让玩家感觉自己是世界上最好的人。
- **详述**: 游戏设计师擅长杀死玩家、设置不可能完成的任务、偷走胜利——但很不擅长称赞玩家。称赞是让玩家继续游戏的最好方法。有效称赞需要同时使用多种形式：角色动画（英雄做出得意动作）、视觉特效（粒子效果、屏幕喷彩）、音乐（胜利旋律）、文字（助手/旁白的祝贺语）、游戏内物品展示（展示刚获得的道具）。关键原则：如果玩家什么都没做就称赞，或者称赞不在地方，就毫无意义。用玩家的自我意识作为奖励——《神鬼寓言》中NPC会根据玩家阵营选择畏缩或赞美，让玩家从他人反应中验证自己的角色定位。
- **适用场景**: 设计玩家成功时刻（解决谜题、打败敌人、收集物品）的反馈系统。
- **示例**: 《宝岛Z》的成功反馈：音效→角色动画→粒子特效→助手祝贺文字→屏幕喷彩→成功音乐→英雄展示道具→道具名显示，八层反馈同时触发。
