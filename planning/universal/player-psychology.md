# 玩家心理与动机

> 来源：《体验引擎》第3章、第8章、《任天堂的体验设计》第4章

## 心流与古迪洛克地带
- **来源**: 《体验引擎》第3章
- **核心**: 挑战难度与玩家技巧匹配时产生心流体验，太难则沮丧，太易则无聊。
- **详述**: 任何具有挑战性的游戏都必须考虑玩家技巧。如果挑战的难度太大，玩家会感到沮丧，而太容易则又会让人觉得无聊。能够产生心流的优秀体验存在于这两个极端之间的"古迪洛克地带"（Goldilocks zone）。问题在于，这个地带对于每一个玩家来说都各不相同。技巧性的处理意味着需要理解技巧性的上限和下限，决定如何拓展这些限制来吸纳更多的玩家，以及如何创造有意义的技巧测试。
- **适用场景**: 设计任何有挑战性的游戏系统时，评估难度曲线是否合理。
- **示例**: 《星际争霸》对专业玩家具有深度挑战，但对新手太难；婴儿车是婴儿的最爱，对成年人毫无意义。设计师需要针对目标受众调整难度。

## 技巧深度
- **来源**: 《体验引擎》第3章
- **核心**: 深度游戏能够通过高水平的技巧创造有意义的玩法，技巧上限超越人类极限的游戏永无止境。
- **详述**: 深度（depth）描述游戏中需要学习的内容有多少。一个深度游戏会通过细微的差别和变化提供很多新内容，让游戏持续很长时间。衡量游戏深度的一个方法是，用理论上完美的选手和最高技巧的人类选手做对比。如果完美选手比任何人类都强，这个游戏就是无止境的深度游戏。具有讽刺意味的是，玩家总是绞尽脑汁地解决难题，但一旦成功就会对游戏嗤之以鼻——技巧性的游戏只有在人们没有完全了解它之前才有价值。
- **适用场景**: 判断游戏是否有长期留存价值，设计多人竞技游戏的深度。
- **示例**: 象棋中人和计算机组队比人类大师更强，说明象棋是无止境的深度游戏。井字游戏一旦掌握规律就没有新花样，是浅显的游戏。

## 无障碍与技巧下限
- **来源**: 《体验引擎》第3章
- **核心**: 游戏的技巧障碍决定了有多少潜在玩家能够进入游戏。
- **详述**: 无障碍游戏对技巧的要求较低。一个游戏的技巧障碍（skill barrier）是指玩这个游戏必须掌握的基础技巧。比如FPS游戏要求玩家掌握移动、转身和射击，这对新手是巨大的障碍。游戏设计师经常低估无障碍的重要性，因为老玩家不会留意游戏的技巧障碍。然而有太多的潜在玩家，只要他们知道怎么玩的话，是很乐意参与进来的。
- **适用场景**: 扩大游戏受众时，降低入门门槛。
- **示例**: 《刺客信条2》认为美感和无障碍比单纯追求深度更重要，通过降低战斗技巧上限让更多玩家享受游戏。

## 技巧范围
- **来源**: 《体验引擎》第3章
- **核心**: 宽泛的技巧范围让初学者和专家都觉得好玩——容易上手却难以精通。
- **详述**: 技巧范围（skill range）指一个游戏展现其挑战性所需要的技巧级别的范围。宽泛的技巧范围意味着游戏可以让初学者和专家都觉得好玩。狭窄的技巧范围则说明玩家一旦掌握玩法就可以很快精通。制作一个有深度且无障碍的游戏十分困难，而制作一个两者皆有并且拥有宽泛技巧范围的游戏更是难上加难。
- **适用场景**: 评估游戏能覆盖多少类型的玩家，决定是否需要拓宽技巧范围。
- **示例**: 《星际争霸2》单人模式技巧范围宽泛但有限（脚本化任务可精通），多人模式则具有无止境深度但门槛更高。

## 自我再造（技巧的三个阶段）
- **来源**: 《体验引擎》第3章
- **核心**: 随着玩家技巧提升，游戏会通过手动→情景→心理三个再造阶段不断自我扩展技巧范围。
- **详述**: 深度游戏通过在游戏内嵌套游戏来扩展技巧范围。每当玩家掌握了一种技巧，更深一层的技巧就会展现出来。手动再造阶段关注每时每刻的操作技巧（瞄准、旋转方块）；情景再造阶段挑战变为何时行动、目标是谁（状况感知、阅读比赛格局）；心理再造阶段需要预判和操纵对手的思想。大多数无止境深度的游戏最终都会进入扑克牌似的终极状态——因为你几乎能掌握任何游戏系统，但永远无法完全理解另一个人的思想。
- **适用场景**: 设计长期可玩的多人竞技游戏，规划玩家的技巧成长路径。
- **示例**: 《虚幻竞技场》中，玩家从关注爆炸效果→瞄准技巧→地图控制→追踪敌人位置→心理博弈，游戏不断自我再造。

## 弹性挑战
- **来源**: 《体验引擎》第3章
- **核心**: 弹性挑战允许不同程度的成功或失败，为不同技巧范围的玩家提供相匹配的挑战。
- **详述**: "成功或失败"的设计方案只适合水平刚好通过挑战的玩家。弹性挑战通过设置不同级别的成功或失败来解决这个问题。由于存在不同程度的成功，游戏的技巧范围变得更加宽泛，因为每个人都有可以达到且同时具有挑战性的目标。成功和失败的幅度可以弹性地向上或向下延伸，而不是直接宣布胜利或失败后结束游戏。
- **适用场景**: 让同一游戏同时满足不同水平的玩家。
- **示例**: 《杀手任务》中，玩家可以一路横冲直撞完成任务，也可以完美潜入获得"沉默刺客"评价，不同程度的成功满足不同水平的玩家。经典街机游戏的计分系统也是一种弹性挑战。

## 潜移默化的训练
- **来源**: 《体验引擎》第3章
- **核心**: 最好的训练方式是在玩家没有觉察到的情况下进行教导。
- **详述**: 有些训练系统给玩家灌输大量指令让玩家思绪从游戏中脱离，或每时每刻都明确告诉玩家该做什么让玩家感觉受到摆布。好的训练方式是潜移默化式的。可以把训练融入背景故事中，可以把训练转变成弹性挑战，或者通过适应性训练只在察觉到玩家缺少某种知识时才出现教学内容。出现得最少的训练内容就是不需要的训练内容。
- **适用场景**: 设计游戏新手引导和教学系统。
- **示例**: 《使命召唤4》将训练融入障碍课程的故事情节中，同时作为时间测试让新手学到东西、专家得到挑战。《生化奇兵》通过引人入胜的场景和剧情悄无声息地增加交互复杂度，玩家甚至没有留意到自己学会了这一切。

## 情感维持
- **来源**: 《体验引擎》第3章
- **核心**: 在玩家技巧不足的早期阶段，利用不需要技巧的情感触发器维持玩家兴趣。
- **详述**: 在游戏早期，玩家需要度过技巧不足的阶段，许多人会在这时放弃。如果不想让玩家在越过技巧障碍之前就放弃游戏，可以利用不需要技巧的情感触发器——展示伟大的艺术、介绍亮丽的角色、播放音乐或演示动画。将这些低技巧的情感触发器融入游戏的早期体验中，无聊透顶的早期学习阶段就可以转换为半交互式的游戏介绍影片。
- **适用场景**: 设计游戏开场的体验，降低早期流失率。
- **示例**: 《生化奇兵》以坠机场景开场，利用音乐、美术风格、哲学标语等带来丰富体验，同时玩家在探索中学习基本操作，将训练变成难以忘却的体验。

## 处理失败与失败陷阱
- **来源**: 《体验引擎》第3章
- **核心**: 不要因为失败而惩罚玩家；失败陷阱——玩家长时间停留在肯定会失败的状况——必须被识别和解决。
- **详述**: 制造紧张意味着让玩家感到痛苦，但秘诀是不要因为失败而惩罚玩家。可以对游戏角色动点手脚，可以不让玩家获得成功或制造小挫折，只要不让游戏陷入不断的重复之中就没有问题。失败陷阱指的是玩家长时间停留在肯定会失败的状况，比如赛车中被远远甩在身后无法追上，或在复活点没有弹药反复死亡。解决方案包括弹性失败条件、投降机制、给予提示或替代挑战。
- **适用场景**: 设计游戏的失败处理机制，评估是否存在潜在的失败陷阱。
- **示例**: 《超级食肉男孩》中角色死亡不到一秒就复活，游戏流程不中断。《矮人要塞》中失败本身带来奖励性体验——要塞倒塌过程中的各种事件让玩家享受每一分钟。

## 多巴胺动机（而非快乐）
- **来源**: 《体验引擎》第8章
- **核心**: 多巴胺并不是快乐的源泉，而是动机的制造者；我们想要某些不喜欢的东西，或喜欢某些不想要的东西。
- **详述**: 早期研究认为多巴胺等于快乐，但后续发现多巴胺在奖励之前就会产生，而非获得奖励时。实验证明，注射多巴胺的人们并不是感到快乐，而是被激发了动机。被杀死了所有多巴胺接受细胞的白鼠不再做任何事（甚至不吃东西），但当糖水喷入口中时仍会露出高兴的表情——它们仍享受食物，只是失去了追求食物的动机。这说明"想要快乐才能快乐"的常识性观点是错误的。多巴胺驱动的动机和实际的快乐感受之间并不总是息息相关的。
- **适用场景**: 理解为什么玩家会沉迷于并不真正享受的游戏，设计激励机制。
- **示例**: 许多游戏玩家发现自己即便很久之前就厌倦了某个游戏，也无法停下来不玩。吸毒的瘾君子越来越想要毒品，却越来越不喜欢毒品。

## 虚拟奖励与奖励预期
- **来源**: 《体验引擎》第8章
- **核心**: 人类大脑无法分辨奖励是真实的还是虚拟的，虚拟奖励同样可以产生强烈的多巴胺动机。
- **详述**: 每一个游戏都需要利用多巴胺动机，因为游戏总会产生让人不高兴的时刻。制造多巴胺动机最主要的方法是制造令人期待的奖励。游戏不能提供真正的食物、金钱或社会地位，但虚拟奖励依然有效，因为人类大脑是在缺乏现代游戏概念的环境下进化的。经验值、游戏金钱、装备、能力等提升角色威力的奖励，过场动画等故事内容奖励，或单纯的成就记录（如高分），都能激发多巴胺动机。
- **适用场景**: 设计游戏的奖励系统，在玩家可能放弃的关键时刻注入动机。
- **示例**: 20世纪80年代的街机游戏只记录最高分，这种数字化的奖励相当于小孩把名字写在最高的树枝上，在游戏以外什么都不是，但玩家仍然乐此不疲地追求高分。

## 强化程序
- **来源**: 《体验引擎》第8章
- **核心**: 奖励出现的时机比奖励本身更重要；不同的强化程序会产生截然不同的玩家行为。
- **详述**: 强化程序（reinforcement schedule）是一个由许多规则组成的系统，定义了玩家什么时候会得到奖励。这个概念来自斯金纳的"操作性条件反射"研究。强化程序在游戏中无处不在——当你击败兽人得到金币时，你就在玩设计师设计的强化程序。强化程序的细节会极大影响玩家持续玩游戏的动机。最重要的两种强化程序是固定比例和可变比例。斯金纳发现，强化程序即使只有细微的改变，也可能导致行为发生翻天覆地的变化。
- **适用场景**: 设计任何涉及奖励发放的游戏系统，规划玩家的行为节奏。
- **示例**: 斯金纳箱中白鼠的行为随着杠杆与食物的关联规则变化而彻底改变，同样的原理适用于游戏中的掉落、升级等系统。

---

## 固定比例与可变比例
- **来源**: 《体验引擎》第8章
- **核心**: 可变比例是最强有力的强化程序，能持续激励玩家；固定比例则会在奖励后产生"尘封时刻"。
- **详述**: 固定比例对行为产出固定比例的奖励（如每次打败兽人获得一个金币），但它会让玩家长时间处于沉寂状态，只有想要下一个奖励时才活跃起来。可变比例的奖励比例每次都变化（如10%概率获得10枚金币），即使平均产出相同，激励效果也截然不同——玩家总有机会在下一次获得大量奖励，所以行为总是频繁和连续的。可变比例在游戏中无处不在，所有赌博游戏都基于可变比例，RPG游戏中每个怪物对应随机奖励列表也是其体现。
- **适用场景**: 选择奖励发放方式以控制玩家动机的强度和持续性。
- **示例**: 你总是希望下一个怪物会掉落一件极品装备——这就是可变比例的力量。RPG游戏将其作为核心玩法。

## 叠加的强化程序
- **来源**: 《体验引擎》第8章
- **核心**: 多个强化程序叠加时，总是至少存在一个产生强烈动机的程序，使玩家无法停止。
- **详述**: 对于大多数奖励的强化程序，在特定时间内玩家的动机都会有所降低，这会产生空档让玩家停下来。强化程序的强大之处体现在多个程序叠加的时候。当一个强化程序出现动机不足时，其他程序刚好处于动机最强烈的时刻。关键在于不让玩家同时只关注一种奖励，保证多个强化程序不会在同一时间被触发。那些被称作"折磨型RPG"的游戏最擅长使用这种技巧——玩家距离下一个重要奖励永远只有不到几分钟。
- **适用场景**: 设计长期留存系统，避免玩家在奖励之间的空档期流失。
- **示例**: 《文明5》产生"再多一个回合综合征"——同时运行超过30个强化程序，总是至少有1个让玩家保持高涨积极性。

## 浮现的强化程序
- **来源**: 《体验引擎》第8章
- **核心**: 大多数强化程序不是直接设计的，而是从游戏的底层系统自然浮现出来的。
- **详述**: 大多数强化程序不是像斯金纳箱那样精准决定的，而是从游戏底层机制的交互中自然浮现。在国际象棋中，吃子的节奏不是设计师预设的，而是随攻防转换自然产生可变比例奖励。对于浮现的和显式的奖励程序，玩家的反应是相同的。区别在于：一个会牢牢抓住你，另一个会让你离开游戏。当浮现的强化程序类似可变比例时，玩家总是劲头十足；当类似固定比例时，则会在赛后出现动机断层。
- **适用场景**: 分析为什么某些游戏模式天然让人上瘾或让人厌倦，优化底层系统以产生良好的浮现奖励节奏。
- **示例**: 某些射击游戏死亡竞赛模式中击杀数量稳定可预测（类似固定比例），玩家赛后并不急于开始下一局；而随机性更大的模式类似可变比例，玩家总是劲头十足。

## 外在动机与内在动机的冲突
- **来源**: 《体验引擎》第8章
- **核心**: 外在奖励可以扭曲、取代甚至摧毁内在动机——给有趣的任务加上奖励反而会减少玩家的兴趣。
- **详述**: 外在奖励是活动本身以外的奖励（如金币），内在奖励和活动本身不可分割（如击败兽人本身就觉得愉快）。研究表明，外在动机能扭曲、取代甚至摧毁内在动机。实验中，给予奖励的一组在奖励取消后兴致全无，而从未得到奖励的一组仍然玩得高兴。这种效应对充满探索性或创意性的工作伤害最大——如果给予奖励，人们就不会满怀欣喜地探索系统，而是希望用最小代价获得奖励。任务本身越有趣，外在奖励的破坏效应就越强。对于单调无趣的任务，外在动机则十分合适。
- **适用场景**: 决定是否在某项游戏活动中加入外在奖励，避免破坏核心体验。
- **示例**: 有奖励时孩子们画的画就没那么有趣；有偿下棋的棋手在无偿时实力大打折扣；学生诗人想到写作可以挣钱时，优秀诗词反而减少。

## 奖励的一致性
- **来源**: 《体验引擎》第8章
- **核心**: 奖励系统鼓励的行为应接近玩家在没有奖励时会做出的行为；仅奖励玩家原本就想要做的事情。
- **详述**: 给予奖励且不破坏游戏体验的关键在于奖励的一致性——奖励系统鼓励的行为十分接近玩家在没有奖励时会做出的行为。最好的情况是奖励系统完全符合玩家内在需求，两种动机效果叠加。但有些游戏无法检测到玩家的内在需求（如创意性、社交性游戏），这种情况下任何奖励系统带来的破坏都大于产生的动机，唯一的解决方案是不设置奖励系统。
- **适用场景**: 设计具体的奖励规则和评分系统，评估奖励是否会扭曲玩家行为。
- **示例**: 《滑板3》精心制作计分系统，关注每一次动作细节并精确匹配视觉冲击力，使得即使没有奖励，玩家想要做的动作也和有奖励时如出一辙。

## 玩家的悔恨感
- **来源**: 《体验引擎》第8章
- **核心**: 利用强化程序制造动机却不给予满足，会导致玩家在事后产生悔恨感。
- **详述**: 当玩家花了一段时间玩游戏后，如果游戏给予了他动机但最终没有让他获得满足，就会产生悔恨感。史前人类的大脑没有进化到可以抵御精心打造的强化程序的地步。多巴胺驱动的游戏能让人做出不合理甚至自我毁灭的举动。位于严重性范围最末端的游戏甚至没有考虑过让玩家获得满足，每个设计决策都围绕着如何最大化引发玩家动机——它们不是体验引擎，只是令人上瘾的机器。
- **适用场景**: 评估游戏设计的道德维度，避免纯粹利用多巴胺机制而忽视体验质量。
- **示例**: 《奶牛点击器》中，即使开发者让所有牛都消失了只剩空地，人们还是不停地点击。多巴胺简直是刺激大脑的迷幻药。

## 体验→感情→记忆
- **来源**: 《任天堂的体验设计》第4章
- **核心**: 情景是否给人留下深刻印象，取决于体验者的感情是否强烈；"体验→感情→记忆"这一流程推动着我们的人生。
- **详述**: 游戏通过无数的体验设计调动着玩家的感情。每一种体验设计都传达出许多感情，一边制造当时的情境，一边打动玩家的心。体验一旦在大脑中作为情景记忆被储存起来，就转换成语义记忆长期保存。而什么样的情景记忆能长期保存？其筛选标准是体验者的感情是否强烈。某种体验让你产生感情的话，它就会留在记忆里。反过来说，你现在记得的，应该是强烈震撼、激发你情感的体验。在人们记忆深刻的著名游戏场景中，一定隐藏着打动人心的体验设计。
- **适用场景**: 理解为什么某些游戏时刻让玩家终生难忘，设计能产生长期记忆的游戏体验。
- **示例**: 回忆你最难忘的游戏时刻——那些时刻一定伴随着强烈的情感波动，无论是惊喜、感动还是紧张。

## 情景记忆与语义记忆
- **来源**: 《任天堂的体验设计》第4章
- **核心**: 记忆先以情景记忆（体验记忆）生成，再从中提取语义记忆长期保存；强烈的情景记忆直接被保存。
- **详述**: 长时记忆分为有意识记住的陈述性记忆和不能用语言表达的非陈述性记忆（如骑自行车的方法）。陈述性记忆主要有两种：语义记忆（对语言含义本身的记忆，适合长期保存）和情景记忆（"发生了什么"的体验记忆）。记忆的形成过程是：先将体验作为情景记忆生成，然后只有时间和事件的语义记忆从情景记忆中提取出来。强烈的情景记忆会直接被保存下来。体验设计的起点就是追溯设计者自己的记忆，以确实激发个人感情的体验为基础进行设计。
- **适用场景**: 理解玩家如何记住游戏内容，设计能从情景记忆转化为长期语义记忆的游戏体验。
- **示例**: "哥伦布1492年发现美洲大陆"是语义记忆；"在社会课上学过哥伦布发现美洲大陆"是情景记忆。两者层次不同，但都来自同一体验。

## 改变难度的三种方法
- **来源**: 《体验引擎》第3章
- **核心**: 显式难度选择、适应性难度、隐式难度选择三种方式可以调整游戏挑战级别，扩大技巧范围。
- **详述**: 显式的难度选择简单有效但可能让玩家费解；适应性的难度根据玩家水平悄悄调整，在玩家毫不知情时效果最好，但专家级玩家会发现并利用系统；隐式的难度选择允许玩家通过战略性决策自行调整挑战级别，如选择不同技巧要求的角色职业。这三种方式可以共存，形成非常宽广的技巧范围。
- **适用场景**: 为不同水平的玩家设计合适的难度系统。
- **示例**: 《生化危机5》将显式与适应性结合——4种可选难度下内部实际有10级难度，随玩家表现浮动；《军团要塞2》通过不同角色职业实现隐式难度选择。

## 心流状态的设计条件
- **来源**: 《游戏设计艺术》第10章/透镜21
- **核心**: 心流是玩家完全沉浸于游戏、忘我操作的状态，其核心条件是挑战与技能的精确匹配——太难产生焦虑，太简单产生无聊。
- **详述**: 心理学家米哈里·契克森米哈赖提出的"心流"理论对游戏设计至关重要。心流发生在挑战难度与玩家技能水平恰好匹配的"甜蜜点"上。当挑战超过技能时产生焦虑，当技能超过挑战时产生无聊。随着玩家技能提升，挑战也必须同步提升才能维持心流。游戏设计师需要设计平滑的难度曲线，确保玩家始终处于心流通道中。这要求准确的难度递进、充分的练习机会和清晰的即时反馈。
- **适用场景**: 设计难度曲线和技能递进系统时；评估玩家是否会在游戏中感到无聊或沮丧时。
- **示例**: 《俄罗斯方块》通过越来越快的方块下落速度，自然地将挑战与玩家逐渐提升的技能匹配，是心流设计的经典案例。

## 玩家需求层次的自我决定理论
- **来源**: 《游戏设计艺术》第10章/透镜22、透镜23
- **核心**: 玩家的深层动机来自三种基本心理需求：自主性（自己做选择）、胜任感（感到自己在进步）、关联性（与他人产生联系），满足这三种需求的游戏最具吸引力。
- **详述**: 自我决定理论（Self-Determination Theory）揭示了人类的三种基本心理需求。自主性指玩家感到自己的行为出于自身选择而非外部强迫——给予玩家有意义的选择而非假选择。胜任感指玩家感到自己在不断进步和掌握技能——可见的进步反馈至关重要。关联性指玩家与他人产生有意义的联系——即使是单人游戏，NPC、角色和世界也能满足这种需求。这三种需求的满足程度直接决定玩家是否"想玩"而非"不得不玩"。
- **适用场景**: 分析玩家为什么喜欢或放弃某款游戏时；设计核心动机系统时。
- **示例**: 《我的世界》同时满足三种需求：自主性（自由建造任何东西）、胜任感（从简陋小屋到红石电路的持续进步）、关联性（分享作品和多人合作）。

## 技巧的多维性
- **来源**: 《游戏设计艺术》第12章/透镜34
- **核心**: 游戏技巧不只是反应速度——它包括心智技巧、物理技巧和社交技巧三大类，优秀的游戏设计会综合利用多种技巧类型来创造丰富的挑战。
- **详述**: 技巧远比"操作精准"广泛。心智技巧包括策略规划、概率判断、模式识别、资源管理。物理技巧包括反应速度、精确瞄准、时序把握、节奏控制。社交技巧包括谈判、团队协作、欺骗识破、领导力。优秀的游戏会让玩家同时运用多种技巧，并在不同阶段侧重不同技巧以创造变化。更重要的是，玩家的技巧会随时间提升，游戏难度必须与技巧增长同步，否则玩家会感到无聊（技巧超过挑战）或沮丧（挑战超过技巧）。
- **适用场景**: 设计游戏的核心挑战类型时；确保游戏不会因只用一种技巧类型而变得单调。
- **示例**: 《塞尔达传说》系列交替使用解谜（心智技巧）、战斗（物理技巧）和NPC互动（社交技巧），使体验始终丰富多变。

## 游戏是安全的学习行为
- **来源**: 《创世学说》第4章
- **核心**: 游戏是生物进化的产物，本质上是一种低成本、低代价、安全的学习工具——人类和高等动物都在虚拟环境中练习未来可能有用的生存本领。
- **详述**: 小男孩玩刀玩枪、假装射击，是在安全的幻想情景中学习狩猎和应对部落冲突；小女孩玩过家家、照顾洋娃娃，是在无代价的环境中学习家务和养育；小狗追逐球和撕咬布偶，是在学习追逐和狩猎而不至于受伤。这些行为不是随机的——小男孩不打扮洋娃娃，小狗不玩刀枪，因为不同物种的祖先需要学习的本领不同，基因决定了各自的兴趣方向。个体的兴趣差异也源于基因的细微差别。这意味着游戏中能让人感兴趣的事物，都和现实世界（或祖先的现实世界）有关：合作对抗的游戏之所以有趣，因为我们的祖先是群居动物；跑酷竞速之所以吸引人，因为祖先需要在大草原上追捕猎物。洞察和分析现实世界中人类的行为和爱好，就不会在游戏设计之路上迷失方向。
- **适用场景**: 判断一个新玩法方向是否可能有趣时，回归生物学本质进行验证；理解为什么某些品类天然比其他品类更有吸引力。
- **示例**: 设计一个合作对抗的游戏，它可能就会很有趣，因为我们的祖先一直在这么做；而设计一款"被人揍"的游戏，可能多半不会受欢迎，因为那些喜欢被揍的祖先早就被大自然淘汰了。

## 过程趣味与结果趣味
- **来源**: 《创世学说》第3章
- **核心**: 一个系统的趣味性由"过程趣味"和"结果趣味"两部分构成，必须两者兼备才能让系统真正有趣。
- **详述**: 以《欢乐斗地主》为例进行拆解。过程趣味：每局中玩家与对手之间牌的大小成为胜负关键，这个胜负过程被拆分成一手手的牌流转。对手先出牌，玩家对手牌产生预估形成"能否压住"的感觉（内化为目标），压住了就产生兴奋感，压不住也没关系还有下一手——只要牌面不是差到极点，一局中多少都能压住几次。手牌的不确定性加重了这种兴奋感。结果趣味：一局打牌过程中形势逐渐明朗，玩家对能否胜利产生预期，无论输赢都会形成目标——赢了达成目标获得兴奋感和奖励，输了也达成了预期。牌局的随机性更加重了这种兴奋感。综上，无论输赢玩家都能获得兴奋感，只要不是持续极端的负面情况。设计系统时应拆分过程趣味和引导结果趣味，两者结合才能打造出有趣的系统。
- **适用场景**: 分析和设计任何游戏系统的趣味性时，分别评估过程和结果是否都有趣。
- **示例**: FPS类游戏的核心趣味集中在武器系统和地图关卡设计的过程趣味上（每次射击和走位都是有趣的过程）；MMORPG的核心趣味集中在数值成长系统的结果趣味上（每次升级和获得装备都是有满足感的结果）。

## 多样性×技巧天花板=无限重玩价值
- **来源**: 《创世学说》第4章
- **核心**: 当一个游戏同时拥有很大的多样性（丰富的可能性）和很高的技巧天花板（技巧成长性）时，其容积是乘数模型，可以产生天文数字般的重玩价值。
- **详述**: 以MOBA游戏为例：100个英雄池中10个不可重复选择，出场组合就有C(100,10)≈17万亿种可能性。再加上不同出装、分路和战术导致的过程差异，几乎不可能有两局完全相同的游戏。每个英雄至少需要几十上百局才能熟练掌握，英雄之间的相生相克又将技巧天花板推到极高。多样性满足多巴胺的"新鲜刺激"通道——每局都是不同的组合；技巧天花板满足多巴胺的"增强刺激"通道——每次掌握新技巧就获得更强的刺激。两条通道同时激活多巴胺分泌，使玩家欲罢不能。但这一公式的关键是乘法而非加法——如果只有多样性没有技巧深度（如换皮卡牌），玩家很快厌倦；如果只有技巧深度没有多样性（如只有单一角色），玩家很快到达天花板。
- **适用场景**: 评估游戏的长期留存潜力；设计新系统时思考如何同时提升多样性和技巧天花板。
- **示例**: MOBA游戏的英雄设计：6类职责（坦克/ADC/APC/辅助/刺客/战士），每个英雄有独特技能组合，出装策略根据对面阵容灵活调整，使得同一局游戏中10个英雄之间的相生相克关系产生了几乎无穷的战术变化。

## 新手引导的核心——尽快触达核心乐趣
- **来源**: 《创世学说》第3章
- **核心**: 新手引导不仅是线性的教学流程，更重要的是让玩家尽快体验到游戏的核心乐趣；教学本身应该引人入胜而非让用户痛苦难忍。
- **详述**: 手游的次日留存率、3日留存率、7日留存率等核心数据与新手引导有直接关系。设计要点包括：引导不仅仅是任务，必须让玩家尽快体验到核心乐趣——如果核心玩法本身不易上手，应将乐趣点进行拆分，切忌过度过长的引导。优秀MOBA游戏会在新手进入时将最简单的英雄默认推荐，而非强制引导用户使用所有主流英雄。玩家对教学内容普遍排斥，尤其是无聊的、与核心玩法无关的内容。一个设计巧妙的新手引导往往是引人入胜的——它本身就是一段令人愉悦的体验。核心原则是"不打扰用户的设计才是好的设计"，引导应该服务核心玩法而非成为核心玩法之前的障碍。
- **适用场景**: 设计游戏的新手引导系统时；优化游戏早期留存数据时。
- **示例**: 《生化奇兵》以坠机场景开场，利用音乐、美术风格、哲学标语带来丰富体验，同时玩家在探索中自然学会基本操作——训练变成了难以忘却的体验而非枯燥教程。

## 感知的主动性与统一物理现实的挑战
- **来源**: 《游戏感》第17章
- **核心**: 人类对日常物理交互的感知是主动的而非被动的——任何微小的物理不一致都会被潜意识放大。游戏世界必须在视觉、声音、运动、模拟和规则上保持统一的"抽象一致性"才能通过玩家的细致审视。
- **详述**: 游戏世界面临"主动感知"的挑战：人们在与现实交互相关的感知上极其敏感，一个球弹跳的感觉不对、一本书翻倒的样子不对都会被注意到。这是无法控制的——我们每天每时都在锻炼感知技能来导航现实世界。抽象的一致性是指：如果游戏的图像、声音、运动、模拟和规则都有同样的抽象程度，那游戏就是协调统一的。做一个卡通形象化的游戏比做写实的游戏更容易满足玩家期望，因为写实度越高，玩家审视标准越高。最难保持一致的是运动——由玩家控制的模拟运动很难做到内在协调，例如角色贴墙跑是几乎所有游戏的不一致点。
- **适用场景**: 评估游戏整体物理感的统一性，排查"哪里看起来不对劲"的感受问题。
- **示例**: 《战争机器》让玩家碰到表面后做出不同行为，声音、特效和规则都协调良好，在主动感知下也站得住脚；写实赛车游戏中任何微小的物理不一致都会被老玩家察觉。

## 润色效果传达物理真实感
- **来源**: 《游戏感》第14-15章
- **核心**: 动画、声音和视觉特效（润色效果）是传达游戏世界物理真实感的关键手段——细微的动画线索（重心转移、脚部刹车）和声音设计（金属撞击声、脚步材质声）能让玩家从潜意识层面相信游戏世界的真实性。
- **详述**: 《生化尖兵》仅凭几个像素的动画和三种声音就让玩家感受到金属钩爪的分量和环境的物理特征——钩爪发射时角色踮脚站立表现出重心转移，依附时发出"咔哒"声传达金属交互，弹回时发出沉闷重击声。在《超级马里奥64》中，去掉动画后角色变为灰色胶囊，大部分重量感和表现力都丧失了。声音设计尤其有效：三种不同级别的撞击声（轻/中/重）传达出交互强度；脚步声根据材质改变帮助玩家识别地面类型；马里奥跳跃时的升调叫声与跳跃高度匹配，让动作和声音协调。《Raptor Safari》的碰撞声分为塑料撞击、金属变形、玻璃碎裂三个层次，根据碰撞速度从列表中随机抽取，避免审美疲劳。这些润色效果的成本极低但效果极大。
- **适用场景**: 在核心机制确定后，通过润色效果提升物理感和交互真实感，是性价比最高的打磨手段。
- **示例**: 《生化尖兵》用3种钩爪声音传达了整个金属交互世界的物理感；《马里奥64》的脚部刹车动画和脚步声共同让玩家相信世界的真实性。

## 操作的表现放大——小输入控制大系统
- **来源**: 《游戏感》第19章
- **核心**: 游戏操作最大的快乐在于"拇指上的扩音器"——用简单的输入控制复杂的系统。填平执行鸿沟（让操作完全真实）会摧毁这种核心快乐。
- **详述**: 现实生活复杂、不公平、难以掌握。游戏可以干净且易于掌握。通过一个低带宽的输入设备（如手柄），玩家能接入一个复杂的系统，体验操控该系统的快乐。这正是游戏中学习比现实生活中学习更有趣的原因。《黄昏公主》用Wiimote挥剑替代按键是一个反面案例：高敏感度的输入（大幅挥动）映射到低响应敏感度（预录动画），让原本在《风之杖》中通过按键实现的剑术精通感荡然无存。输入设备应该追求"自然且符合直觉"的映射，而不一定是"真实"——真实和直觉之间存在关键区别。最优秀的输入是那些看似简单但能控制复杂行为的工具。
- **适用场景**: 评估新输入方案（体感、VR、触控等）时，不要只追求"真实感"，而要确保简单输入能产生丰富有趣的响应。
- **示例**: 一个摇杆就能控制马里奥的全部运动，包括跑步、转弯、跳跃方向——输入简单但响应丰富；《黄昏公主》用Wiimote的挥剑操作虽然更"真实"，但丧失了按键操作的精确度和精通感。

---
- **核心**: 游戏设计需要理性知识规划框架和数值，感性知识设计核心玩法——只讲理做出来不好玩，只凭感受做方向可能歪掉。
- **详述**: 核心玩法类似于武林绝技——知道每个细节但做出的东西差了神韵，因为过多注意细节反而忽略了真实感受。正确的方式是亲身去体验每一个操作带来的感受，将自己融入游戏沉浸在节奏中去体会，然后记录并力图让感受更加良好。直观获得感知的器官是眼、耳、身，设计需要在感性上避免对这三者造成过分刺激（违和感）。核心玩法通过视觉、触觉、听觉引起玩家情感变化，需要切身感受并判断这种变化是否合适。如果核心玩法在感受上能形成较好的循环，游戏生命力就会强劲；如果能把这个循环打磨得毫无阻力非常流畅，游戏就具备了大成的基本要素。同时要明确定位自己代表哪种类型的玩家，有针对性地收集主要用户群体的直观反馈进行汇总。
- **适用场景**: 设计核心玩法时切换到感性模式去体验；设计系统框架和数值时切换到理性模式去规划。
- **示例**: 《全民飞机大战》中曾制作弹幕非常华丽的战机，结果玩家反馈长时间操作使眼睛无法承受——设计损伤了直观感受。好的核心玩法设计需要先切身感受再理性优化。
