# 敌人设计与AI

> 来源：《游戏设计的236个技巧》第2章

## 敌人意符设计（Signifier）
- **来源**: 《游戏设计的236个技巧》第2章/2.1节
- **核心**: 通过敌人外观设计表现其功能，让玩家无需文字说明即可理解敌人特性和应对方式。
- **详述**: 《超级马里奥3D大陆》中的敌人设计遵循意符原则：尖刺表示不可踩踏、翅膀表示飞行能力、缩回甲壳表示无法行动。3D游戏中敌人需要从任何角度都能一眼看出其独特功能和轮廓，包括脸、尾巴等细节。敌人状态变化（如慢慢龟被踩后缩回甲壳）也通过外观传达"现在不能动""碰到也不会死"等信息。外部发射子弹的敌人身上必须有发射口，否则玩家搞不清子弹来源。
- **适用场景**: 设计任何需要玩家直觉理解应对方式的敌人，特别是面向大众的动作游戏。
- **示例**: 《超级马里奥3D大陆》中翅膀板栗仔通过翅膀造型让玩家瞬间明白敌人在飞行；刺猬的尖刺传达"不可踩踏"的信息。

## 敌人移动速度与玩家体验
- **来源**: 《游戏设计的236个技巧》第2章/2.1节
- **核心**: 敌人移动速度通常应低于玩家角色的普通冲刺速度，以丰富玩家利用敌人的玩法。
- **详述**: 敌人速度低于玩家冲刺有三方面原因：一是便于踩踏消灭从背后逃走的敌人；二是避免画面滚动方向与敌人移动方向相反导致相对速度过大、难度陡增；三是方便玩家将敌人作为踏板跳到更高处。速度越慢成功时机越容易把握，但过慢会减弱紧张感。敌人移动速度对游戏的影响有时甚至超过攻击动作，需要严格把握好"可以利用但稍有不慎就会吃大亏"的平衡。
- **适用场景**: 平台跳跃游戏、允许利用敌人角色的动作游戏。
- **示例**: 《超级马里奥3D大陆》中几乎所有敌人的移动速度都低于马里奥的普通冲刺，玩家可将慢慢龟踢向其他敌人作为武器。

## 敌人动作引导玩家学习
- **来源**: 《游戏设计的236个技巧》第2章/2.1节
- **核心**: 通过敌人自身的行为动作，巧妙地向玩家传达新的操作方法和攻击手段。
- **详述**: 开发者常为"希望玩家使用某个动作，但如何让玩家注意到"而烦恼。《超级马里奥3D大陆》的解决方案是让敌人来演示动作：玩家变身狸猫马里奥后立刻遇到"尾巴板栗仔"，尾巴板栗仔会主动发动尾巴攻击且具有碰撞检测，让玩家注意到"被尾巴攻击到会很痛"，从而自然联想到用尾巴攻击消灭敌人。这种手法无需文字提示，让初次接触游戏的玩家也能自然学会操作。
- **适用场景**: 需要在不使用文字教程的情况下教会玩家新技能的游戏。
- **示例**: 《超级马里奥3D大陆》关卡1-1中，尾巴板栗仔的尾巴攻击引导玩家学习狸猫马里奥的尾巴攻击。

## 3D游戏中敌人轮廓的空间认知功能
- **来源**: 《游戏设计的236个技巧》第2章/2.2节
- **核心**: 在3D游戏中设计辨识度高的敌人轮廓，帮助玩家快速判断敌人的种类、大小、朝向和距离。
- **详述**: 3D游戏画面本质是2D输出，玩家需在脑中重构三维空间。敌人轮廓中的"脸"和"身体"等细节可以瞬间将"大小""朝向""位置""运动方向"传达给玩家。脸在人类空间认知中担当重要角色——只要敌人身上有脸部结构，就能瞬间辨别其朝向和大小。在昏暗洞窟或沙漠等外观单调的地形上特别需要注意轮廓设计，可以让敌人摆出前倾姿势或伸出一部分肢体来明确方向性。
- **适用场景**: 所有3D动作游戏、需要快速判断敌人位置和朝向的战斗系统。
- **示例**: 《战神Ⅲ》中的各种敌人都具有鲜明的轮廓特征，帮助玩家在快速战斗中辨识敌人类型和距离。

## 敌人移动轴的限定与攻防乐趣
- **来源**: 《游戏设计的236个技巧》第2章/2.2节
- **核心**: 在以近距离战斗为主的3D游戏中，大胆限制敌人移动的"轴"，让玩家享受攻防乐趣。
- **详述**: 敌人相对于玩家的前后直线移动便于玩家把握距离，近距离格斗攻击为主的游戏要求敌人以玩家所在方向为基准移动。强力冲锋攻击可设计为平行于画面方向移动，让玩家瞬间分辨距离。只在地面上纵向或横向移动的敌人容易被纵劈或横砍命中；加入"高度"的立体运动（如空中飞行、墙壁爬行）则使敌人难以命中。在割草类游戏中，敢于限制移动轴是让玩家享受攻防的关键。
- **适用场景**: 割草类游戏、以近距离格斗为主的3D动作游戏。
- **示例**: 《战神Ⅲ》中敌人在距离较远时奔跑接近，进入肉搏范围后改为行走，移动速度略慢于玩家角色，保证玩家能自由穿梭选择目标。

## 攻击动作的六要素调整
- **来源**: 《游戏设计的236个技巧》第2章/2.2节
- **核心**: 敌人攻击动作由"上摇""前摆""击打""跟进""硬直""收招动作"六个要素构成，调整各要素可大幅改变游戏手感。
- **详述**: 上摇（预备动作）越短攻击越难闪避，面向大众的动作游戏通常将上摇设为0.5～1秒以上。前摆的碰撞检测时间越长玩家越难接近。硬直和收招动作越长敌人破绽越大。攻击的蓄力发力节奏也有四种典型模式：蓄力发力时间相同（平稳但无力量感）、蓄力快发力慢（跟进阶段超预期）、蓄力慢发力快（上摇长但发力突然）、充满节奏感的蓄力发力（张弛明显、符号化易识别）。通过将各种节奏分工组合，可为每场战斗创造特有的速度与节奏。
- **适用场景**: 任何需要设计敌人攻击动作的3D动作游戏。
- **示例**: 《战神Ⅲ》采用比以往动作游戏更长的上摇时间，确保玩家能看清攻击并做出反应。

## 敌人角色构成的三角牵制
- **来源**: 《游戏设计的236个技巧》第2章/2.2节
- **核心**: 在非对称的一对多游戏中，将拥有不同要素（攻击、投技、防御等）的多种敌人组合，从集团层面构成三角牵制。
- **详述**: 仅用一种敌人组成的集团，玩家一旦找到攻略窍门数量便不再是问题。将"擅长攻击的敌人"与"擅长打破防御的敌人"组合后，玩家被迫在"攻击""防御""闪避"中不断选择。如奥林匹斯卫兵（攻击+投技）搭配戈耳贡（石化攻击即死），使原本不足挂齿的卫兵变得极具威胁。这种"三角牵制"思想不单适用于敌人设计，在攻击动作和魔法系统等领域同样有效。
- **适用场景**: 割草类游戏、一对多战斗系统设计。
- **示例**: 《战神Ⅲ》中奥林匹斯卫兵与奥林匹斯杂兵（魔法飞弹破防）组合，让玩家思考先消灭哪种敌人；奥林匹斯卫兵与戈耳贡（石化攻击）组合使战术稍有疏忽即招致死亡。

## 战斗AI（集团AI）机制
- **来源**: 《游戏设计的236个技巧》第2章/2.2节
- **核心**: 在同时与大量敌人交战的游戏中，需要一个管理所有敌人的战斗AI来控制攻击时机、包围阵型和战斗节奏。
- **详述**: 如果每名敌人只运行独立AI，最终会挤成一团。战斗AI为每名敌人设置移动目标实现包围玩家；控制攻击时机避免多名敌人同时攻击导致玩家无法反击；管理战斗速度和节奏，在游戏初期禁止敌人攻击连击中的玩家。战斗AI的反击时机设计有两种方法：在敌人每次攻击后设置反击机会（由多名敌人共同构成连击），以及让两三名敌人同时攻击以扩展反击时的攻击模式（强攻单体或大范围反击）。
- **适用场景**: 割草类游戏、大量杂兵同时登场的动作游戏。
- **示例**: 《战神Ⅲ》通过战斗AI管理所有敌人的攻击节奏，既保证了割草的爽快感，又确保玩家有反击机会。《蝙蝠侠：阿甘之城》的战斗AI会命令两名敌人同时接近玩家，有意创造"双重压制"的机会。

## BOSS设计的六要素
- **来源**: 《游戏设计的236个技巧》第2章/2.2节
- **核心**: 让玩家产生主动消灭BOSS动机的设计包含：剧情障碍、不可击倒的外观、不消灭玩家不罢休的心态、出其不意、明确弱点、以及让玩家感受到成长。
- **详述**: 超巨大BOSS常与关卡设计一体化，战斗围绕弱点展开，包含"可闪避并反击的攻击""只能闪避的攻击""闪避至远距离""远距离攻击""暴露弱点"等攻击模式。BOSS战需要电影般具有起承转合的流程（如波塞冬的四场战斗：触手1→触手2→巨大本体→人类形态）。设计BOSS时的关键不在于参数设置得多强，而是通过BOSS独有的攻击动作带动战斗整体的速度与节奏。同时BOSS设计要接纳玩家的感情——思考"要接纳何种感情""需要怎样的敌意""用什么方式被消灭""提供怎样的报酬"。
- **适用场景**: 所有需要BOSS战的游戏，特别是动作游戏和冒险游戏。
- **示例**: 《战神Ⅲ》的超巨大BOSS波塞冬分四场战斗，具有起承转合的完整流程，最后以QTE终结让玩家体验残虐的爽快感。

## 为玩家攻击动作赋予意义的敌人机制
- **来源**: 《游戏设计的236个技巧》第2章/2.3节
- **核心**: 通过为敌人设置不同方向的弱点，为玩家的各种攻击动作赋予意义和存在的价值。
- **详述**: 《塞尔达传说：天空之剑》中德库食人草根据花瓣开启方向需要不同方向的挥砍才能伤害；哥布林的防御分方向，需要判断从哪里砍才能突破；四臂骸骨剑豪同时使用四柄剑，需要调动全部技巧判断攻击方向。敌人有着为玩家行动赋予意义的功能。随着玩家深入了解攻击动作的意义，会主动尝试新用法，每发现一个新用途乐趣都会进一步扩展。反过来说，一个攻击动作由许多不同的敌人来承受（反应），攻击动作的乐趣才得以扩展。
- **适用场景**: 剑战型动作游戏、体感操作游戏、拥有多种攻击方向的游戏。
- **示例**: 《塞尔达传说：天空之剑》中的哥布林防御机制要求玩家从不同方向攻击，为Wii遥控器的多彩剑术动作赋予了意义。

## 拥有"秘密"的BOSS设计
- **来源**: 《游戏设计的236个技巧》第2章/2.3节
- **核心**: 为BOSS设置玩家需要"发现"的弱点秘密，将解谜乐趣融入BOSS战。
- **详述**: 《塞尔达传说》系列的BOSS无法靠蛮力消灭，玩家必须先发现其弱点。BOSS基拉希姆会迎合玩家持剑方向防御，玩家需一度切换持剑姿势从其举手的一侧斩击。BOSS贝拉·达玛则需发现"向嘴里扔炸弹"的秘密才能取胜。为防止秘密过于隐晦，每名BOSS身上都设计了方便发现的提示（如野兽型BOSS的弱点统一为眼睛）。关键技巧是：在玩家遇到BOSS之前，先在杂兵的应对方法及关卡设计中埋下"对BOSS秘密的提示"。
- **适用场景**: 探索型动作冒险游戏、注重发现乐趣的BOSS战。
- **示例**: 《塞尔达传说：天空之剑》中基拉希姆的防御手位弱点、贝拉·达玛的"向嘴里扔炸弹"秘密，都是需要玩家发现才能取胜的BOSS设计。

## 潜行游戏中的敌人探测机制
- **来源**: 《游戏设计的236个技巧》第2章/2.4节
- **核心**: 通过视觉和听觉双重探测机制，以及敌人心率参数，创造潜行游戏的紧张感和策略深度。
- **详述**: 视觉探测以扇形视野为基准，墙壁等障碍物可遮挡；听觉探测覆盖360度范围，敌人能察觉声音方向前去侦察。玩家可利用声音分散敌人注意力、调虎离山。敌人发现玩家后会通知所有敌人进入警戒状态，将单个探测升级为集团探测。敌人心率参数（90BPM冷静→120BPM紧张→140BPM恐惧→40BPM无意识）可让玩家透过游戏感受蝙蝠侠给敌人带来的恐惧，恐惧越大玩家越占优势。
- **适用场景**: 潜行动作游戏、恐怖游戏、需要隐蔽要素的游戏。
- **示例**: 《蝙蝠侠：阿甘之城》中敌人心率从90BPM冷静到140BPM恐惧的变化，让玩家能判断最佳攻击时机；投掷蝙蝠镖可引开敌人。

## 功能可供性指向型敌人设计
- **来源**: 《游戏设计的236个技巧》第2章/2.4节
- **核心**: 每个敌人都有明确的"攻击方式"和"击溃方法"，与玩家角色的功能可供性一一对应。
- **详述**: 《蝙蝠侠：阿甘之城》中每种敌人对应特定的击溃方式：电棍敌人需从背后攻击、持刀敌人需用利器闪避化解、盾牌敌人需击昏后空中攻击、防具敌人需击昏后才有效。增加玩家角色的功能可供性时，与之相对应的敌人种类或反应也必须增加。这些与功能可供性相对应的敌人堪称"明星炮灰角色"，其AI不仅要考虑如何攻击玩家，还要研究如何被玩家击溃——如同功夫电影的打戏拍摄。
- **适用场景**: 功能可供性指向型游戏设计、功夫战斗类游戏、一对多徒手格斗游戏。
- **示例**: 《蝙蝠侠：阿甘之城》中电棍敌人只能闪避后从背后攻击、盾牌敌人需击昏后使用空中攻击，每种敌人都引导玩家使用不同的动作。

## 自由流程战斗的敌人AI
- **来源**: 《游戏设计的236个技巧》第2章/2.4节
- **核心**: 为实现无缝连续战斗，敌人AI在后台实时生成"走台剧本"，确保玩家始终有可攻击的敌人。
- **详述**: 自由流程战斗需要玩家不断闪避敌人攻击并反击命中，否则战斗无法无缝衔接。系统必须实时准备下一名要发动攻击的敌人。战斗AI会有意创造特殊攻击机会（如命令两名敌人同时接近创造"双重压制"机会），并在画面中显示反击标记提示玩家。这种设计与功夫电影打戏异曲同工：成龙让攻击者用吼声将攻击招式告知对方以掌握一对多的节奏。敌人不仅要设计如何攻击玩家，更要研究如何被玩家帅气地击溃。
- **适用场景**: 自由流程战斗系统、功夫电影风格的格斗游戏。
- **示例**: 《蝙蝠侠：阿甘之城》中战斗AI命令两名敌人同时接近并攻击，有意创造"双重压制"机会；反击标记让菜鸟也能轻松发动反击。

## 假动作机制
- **来源**: 《游戏设计的236个技巧》第2章/2.6节
- **核心**: 通过"攻击种类的假动作""连击的假动作""移动的假动作"，让玩家在与敌人的攻防中体验紧张感和识破假动作的喜悦。
- **详述**: 攻击种类的假动作是将多种攻击的上摇设计得十分相似，让玩家误判攻击时机；连击的假动作是通过改变连击中的招式数诱导玩家误判反击时机；移动的假动作是敌人自主使用撤步或侧步引诱玩家攻击挥空再反击。成功的移动假动作会给玩家"被看穿了？"的心理打击，而识破假动作后攻击成功则带来"我看穿你了！"的爽快感。制作假动作时必须保证留有提示和破绽，让玩家通过仔细观察能够识别出来。
- **适用场景**: 高难度硬核动作游戏、注重一对一攻防的战斗系统。
- **示例**: 《黑暗之魂》中敌人的攻击假动作让上摇阶段相似但挥动速度不同，连击假动作突然改变招式数，移动假动作用撤步引诱玩家出招后反击。

## 敌人势力范围机制
- **来源**: 《游戏设计的236个技巧》第2章/2.6节
- **核心**: 为每个敌人设置"势力范围"，限制其追击距离，兼顾游戏难度控制和程序性能优化。
- **详述**: 势力范围的必要性有两方面：难度上，无限追击会导致玩家身后追兵越来越多、杂兵打乱战斗节奏；程序上，跨区域追击会导致内存溢出或掉帧。常用实现方法包括：用立方体或圆柱体指定每名敌人的守卫范围、在通道和房间边界设置敌人无法通过的碰撞检测、使用"拴狗桩"为每个敌人规定移动有效范围或追踪时间。势力范围也常被用于包围宝箱等重要场所。
- **适用场景**: 开放世界或半开放关卡结构的游戏、需要控制敌人追击行为的系统。
- **示例**: 《黑暗之魂》中敌人追击玩家出一定区域后放弃返回原位，玩家可利用此机制逐个引诱敌人各个击破。

## 巨型BOSS攻击命中设计
- **来源**: 《游戏设计的236个技巧》第2章/2.6节
- **核心**: 巨型BOSS因体型悬殊难以命中玩家，需要综合运用IK技术、身体跟踪和扩大碰撞检测等方法解决，同时将距离分为"贴身""近""中""远"设计攻击模式。
- **详述**: BOSS体型越大纵向攻击越难命中矮小玩家（手臂长度和身体姿势限制了落点）。解决方案包括：IK技术实时计算劈下动作（自然但受限）、让BOSS身体跟踪目标位置（命中但可能滑步）、扩大碰撞检测范围加入冲击波（大范围但不自然）。巨型BOSS还需大范围攻击保护自身（踩踏、喷吐毒液、回旋攻击），并在设计时增加"贴身"距离概念，让攻击动作分别覆盖贴身、近距离、中距离、远距离。若主打近距离攻防乐趣，建议少用魔法或远程武器。
- **适用场景**: 存在巨型BOSS的3D动作游戏、BOSS与玩家体型悬殊的战斗设计。
- **示例**: 《黑暗之魂》中钢铁傀儡的踩踏属于贴身攻击、斧头下劈属于近距离攻击、冲击波属于中远距离攻击，覆盖所有距离层级。

## 恐怖游戏的敌人移动与攻击设计
- **来源**: 《游戏设计的236个技巧》第2章/2.7节
- **核心**: 通过敌人移动速度的"远快近慢"切换、突然迅猛的近距离攻击，以及让人联想到死亡的攻击动作，制造恐怖体验。
- **详述**: 《生化危机1～3》的丧尸缓慢接近让玩家体验丧尸电影的危急时刻；移动速度和生命值调至恰到好处——从被发现的距离开始射击，在被咬到前恰好能击毙。《生化危机4》的宿主在远距离奔跑接近，近距离后缓步警戒，给玩家摸索部位攻击的时间。敌人被不明集团缓慢包围的恐怖是该作独有效果。恐怖游戏中上摇很短的突然攻击非但不会降低品质，反而衍生出"突然受到攻击"的恐怖气氛。让人联想到死亡方式的攻击动作可通过玩家的死亡反应提升恐惧感。
- **适用场景**: 恐怖求生游戏、需要营造恐惧感的游戏。
- **示例**: 《生化危机4》中宿主远距离奔跑接近、近距离缓步警戒的切换；电锯一击分尸的死亡镜头让玩家恐惧；《死亡空间》中致死攻击后身体四分五裂的镜头。

## TPS中的敌人死角与AI调整
- **来源**: 《游戏设计的236个技巧》第2章/2.7节
- **核心**: 在TPS游戏中，敌人AI应尽量让敌人出现在玩家可见位置，避免从不可见死角频繁攻击引发烦躁感。
- **详述**: TPS中玩家角色占据画面较大空间，死角也大。敌人、玩家角色、镜头排成直线时产生视觉死角。如果敌人频繁从侧面或背后攻击，无论何种高手都会烦躁。《生化危机4》让绝大部分敌人绕至玩家角色面前，仅极少数从背后攻击。敌人AI通过折线移动和兵分两路创造射击技术方面的挑战，而非单纯依赖死角。受伤反应会解除瞄准状态并保证攻击者映入镜头，帮助玩家分辨攻击来源。有受伤反应的射击游戏需特别调整敌人AI防止连续受伤导致无法动弹。
- **适用场景**: TPS射击游戏、玩家角色占画面较大的游戏。
- **示例**: 《生化危机4》中绝大部分敌人绕至玩家面前发动攻击；受伤后解除瞄准状态并自动将攻击者映入镜头。

## 部位攻击与敌人弱点系统
- **来源**: 《游戏设计的236个技巧》第2章/2.7节
- **核心**: 通过射击敌人不同部位产生不同效果（爆头、断腿、打落武器等），为射击游戏增加战术深度。
- **详述**: 部位攻击的前提是敌人移动速度足够慢，让玩家有时间瞄准所需部位。射击头部造成最高伤害且有概率秒杀；射击腿部让敌人跪地后可使用格斗攻击消灭；射击手臂打落武器。配合部位破坏系统创造弱点不同的敌人，可使射击乐趣成倍增长（如持盾敌人需射击脚部让其站不稳）。在游戏流程中改变敌人设计（前期近距离攻防、后期加入远程武器和重型敌人）可让游戏从恐怖游戏动态转换为动作游戏。
- **适用场景**: TPS/FPS射击游戏、恐怖求生游戏。
- **示例**: 《生化危机4》中射击宿主腿部后可使用回旋踢消灭；宿主被爆头后出现寄生体的新敌人形态；《死亡空间》中不同敌人的弱点各不相同（手腕、触手等）。

## 射击类游戏中的敌人隐蔽动作与攻防
- **来源**: 《游戏设计的236个技巧》第2章/2.8节
- **核心**: TPS/FPS中敌人最重要的动作是"隐蔽动作"，兼具攻击和防御效果，且大幅影响游戏节奏。
- **详述**: 隐蔽动作需要关卡数据中预置功能可供性信息，否则再高明的AI也做不出隐蔽动作。不同隐蔽动作带来的攻防大不相同：固定位置探头容易被消灭；从上方和侧面两个位置随机探头产生攻防；佯装隐蔽实则迂回接近扩展玩家警戒范围至360度。隐蔽动作让双方节奏有张有弛，但站位缺乏变化——需要敌人突击、投掷手榴弹或掩体破坏等机制迫使玩家放弃一味隐蔽，防止节奏停滞。
- **适用场景**: TPS/FPS射击游戏、拥有掩体系统的枪战游戏。
- **示例**: 《神秘海域：德雷克的欺骗》中敌人从掩体上方和侧面随机探头、佯装隐蔽后迂回接近等多种隐蔽动作，让枪战攻防变化多端。

## 敌人AI的目标选择与感知系统
- **来源**: 《游戏设计的236个技巧》第2章/2.8节
- **核心**: 通过"距离""可视性""条件""状态"等参数计算目标的权重值，让敌人像真人一样选择最值得攻击的目标，并通过主/次圆锥的双层视觉结构模拟人类模糊感知。
- **详述**: 目标参数中距离越近权重越高、越显眼权重越高、正在瞄准或刚攻击过自己的目标权重越高；同时包含"是否已被选为目标"的条件防止单名角色受集中攻击。视觉感知分为辨识时间短但范围小的主圆锥和辨识时间长但范围大的次圆锥，敌人根据玩家在圆锥内的滞留时间做出"有什么东西在动""找到了！就是他！"等模糊感知。不同战况下主/次圆锥范围会变化（巡逻时窄长、战斗时扩大）。
- **适用场景**: 多目标战斗的TPS/FPS游戏、需要敌人有真实感知行为的游戏。
- **示例**: 《神秘海域：德雷克的欺骗》中躲在掩体后不显眼的角色不会被优先攻击；探出掩体的显眼角色更容易被选中，玩家可利用此机制制定战术。

## 团队意识敌人AI与导演AI
- **来源**: 《游戏设计的236个技巧》第2章/2.8节/2.9节
- **核心**: 通过表现敌人的团队协作行为和导演AI的"放松→铺垫→极限紧张→紧张衰减"循环，营造真实战场气氛和控制玩家情绪节奏。
- **详述**: 团队意识AI让敌人根据战况做出有组织的行动：发现者高喊并攻击、领队发出"把他引出来"的指示、投掷手榴弹逼出掩体、跟丢后重新搜索等。将敌人AI所想的内容表现出来，可将敌人的感情真实传递给玩家。《求生之路》的导演AI则通过感情强度标记（遭遇敌人+10、受攻击+20、恢复生命值-10）模拟玩家情感波动，在紧张感达到顶峰时刷新大批丧尸，创造极限紧张状态后进入衰减期，循环往复。
- **适用场景**: 需要营造真实战场气氛的射击游戏、需要动态控制恐怖节奏的恐怖游戏。
- **示例**: 《神秘海域：德雷克的欺骗》中敌人领队下达战术指令，团队协作包围玩家；《求生之路》中导演AI通过"放松→铺垫→极限紧张→紧张衰减"循环精确控制玩家的恐惧节奏。

## 敌人AI的种类与选型
- **来源**: 《游戏设计的236个技巧》第2章/2.9节
- **核心**: 敌人AI主要有基于规则的AI、基于态的AI（FSM）、基于行为的AI（行为树）、面向目标型AI和基于效果的AI等多种类型，应根据游戏规模和需求选型。
- **详述**: 基于规则的AI简单但容易暴露弱点（如《吃豆人》四色怪物各有固定规则）；基于态的AI通过态迁移处理时间差问题，可分群分层实现模式转换，但态数量增多后管理困难；基于行为的AI以行为树管理，设计·实现·调整都简单，适合TPS/FPS；面向目标型AI通过回报与风险评估自主编制计划，适合战略·战术浓厚的游戏；基于效果的AI通过内在需求参数模拟人类模糊行动，适合模拟生活类游戏。多层次AI（战略层→部队层→士兵层）可控制成百上千名士兵，根据与玩家的距离开关AI以优化性能。
- **适用场景**: 所有需要敌人AI的游戏开发，根据游戏类型和团队规模选择合适方案。
- **示例**: 《杀戮地带2》采用策略AI→班AI→士兵AI的多层次结构加基于任务的AI；《合金猎犬》使用面向目标型AI让每个机器人自主思考；《模拟人生》使用基于效果的AI模拟人类日常行为。

## 敌人踉跄与命中停止机制
- **来源**: 《游戏设计的236个技巧》第2章/2.10节
- **核心**: 踉跄机制通过"伤害累积到阈值才触发受伤反应"表现敌人耐打程度，命中停止通过暂停命中瞬间提升攻击的分量感和操作手感。
- **详述**: 踉跄机制让角色在伤害累积到一定值前不出现反应，超过后才执行踉跄动作。《黑暗之魂》将武器"踉跄值"与敌人"强韧度"对比，踉跄值高于强韧度才执行受伤反应，使弱小敌人弱不禁风、强大敌人坚如磐石。命中停止则在攻击命中时暂停双方动作，既能延长预输入时间作为额外奖励，又能增强伤害表现力。命中停止时间放在"击打之后"（跟进阶段）比"击打瞬间"更有力量感，但会损失真实感，需权衡。
- **适用场景**: 格斗游戏、硬核动作游戏、注重攻击手感的战斗系统。
- **示例**: 《黑暗之魂》中高强韧度的敌人挨打不踉跄（霸体状态），使玩家处于"一旦被反击必受伤"的危险中，增加了战术深度。

## 死亡反应作为玩家报酬
- **来源**: 《游戏设计的236个技巧》第2章/2.10节
- **核心**: 敌人的死亡反应是玩家消灭敌人时获得的"最大奖赏"，精心设计的死亡反应能增强玩家的成就感和游戏动力。
- **详述**: 死亡反应具有三重功能：给玩家的报酬（残酷或炫美的灭敌演出）、让玩家行动更帅气的表演（慢动作品味胜利瞬间）、表现游戏世界观（尸体消失的处理方式影响真实感）。《战神Ⅲ》用CS攻击消灭敌人时出现"头被拧掉"等残酷死亡反应作为心理报酬；《生化危机4》让玩家在踢倒敌人后用回旋踢或背摔帅气消灭；《蝙蝠侠：阿甘之城》在消灭最后一名敌人时触发慢动作。死亡反应必不可少，因为它是"让玩家享受成就感余韵的时间"。
- **适用场景**: 所有有战斗系统的游戏，特别是动作游戏和格斗游戏。
- **示例**: 《战神Ⅲ》CS攻击的残酷死亡反应；《蝙蝠侠：阿甘之城》最后一名敌人倒下时的电影级慢动作；黑泽明《用心棒》中敌人在血雨中缓慢倒下的经典镜头。

## 信息不对称创造AI的策略深度
- **来源**: 《游戏设计艺术》第12章/透镜29（秘密）
- **核心**: 游戏中最引人入胜的元素往往是秘密——信息的不对称（玩家知道敌人不知道的、或敌人知道玩家不知道的）是创造战术深度和戏剧性的核心机制。
- **详述**: 秘密在游戏中无处不在：隐藏关卡、未揭示的物品属性、敌人未公开的弱点、迷雾中的战争。当玩家发现秘密时会产生强烈的"啊哈"惊喜。在AI设计层面，秘密意味着信息不对称——敌人不知道玩家的位置时玩家可以进行潜行暗杀；玩家不知道敌人的巡逻路线时游戏产生紧张感。最优雅的秘密设计是让玩家在发现秘密后感觉"原来如此"而非"这不公平"。秘密还创造了重复游玩的价值——"也许我错过了什么"。
- **适用场景**: 设计敌人感知系统、潜行机制、信息揭示系统时。
- **示例**: 《合金装备》系列将信息不对称作为核心玩法——玩家需要在不被敌人发现的情况下获取情报，敌人不知道玩家位置但一旦发现就会追击。

## 行为模式的可读性与可利用性
- **来源**: 《游戏设计艺术》第15章/透镜67（模式）
- **核心**: 复杂系统需要模式，但模式必须让玩家可感知——当玩家能识别敌人的行为模式并学会利用它时，游戏产生了技巧深度和成就感。
- **详述**: 模式是给游戏添加多样性的好办法，但也必须非常小心——如果玩家不能意识到模式发生了切换，会产生困惑。在敌人AI设计中，行为模式必须是可读的：玩家需要能够观察、学习、预测敌人的行为规律，然后制定对策。如果敌人行为完全随机或模式切换没有视觉提示，玩家会觉得"不公平"。最佳设计是让敌人有可读的行为模式，但切换模式时有明确的视觉/听觉信号，给予玩家时间适应和反应。当玩家成功识别并利用模式时，获得的成就感远大于简单的反应速度考验。
- **适用场景**: 设计敌人AI行为状态机、Boss战阶段转换时。
- **示例**: 《黑暗之魂》中Boss的行为模式高度可读——玩家通过多次尝试学习攻击间隙和躲避时机，掌握后的成就感是游戏的核心吸引力。

## 挑战≠困难——趣味曲线与心流
- **来源**: 《通关！游戏设计之道》第12关
- **核心**: 困难增加痛苦和损失，挑战提高技巧和上升空间。游戏应该富有挑战性，而非靠难度折磨玩家。
- **详述**: 困难的游戏总是想方设法惩罚玩家，充满挑战的游戏则在玩家面前放置可用技巧和知识克服的障碍。罗杰斯提出了"趣味曲线"概念：游戏中存在一个点，挑战性急转直下只剩挫败感。设计师的目标是永远不要越过趣味曲线。超越趣味曲线的解决方案是创造"阶梯式递增的玩点"——将一个玩点建立在上一个玩点之上，教会玩家新的行动，再让他们用掌握的技巧对付机关和敌人。这些玩法要素随游戏进程相互结合并慢慢提高难度。
- **适用场景**: 调整游戏整体难度曲线，确保玩家保持心流状态而不掉入挫败区域。
- **示例**: 《恶魔之魂》《忍者龙剑传》《斑鸠》等高难度游戏虽有忠实粉丝，但始终是少数。如果希望大多数玩家通关，游戏应富有挑战性而非靠难度。

## 陷阱与敌人的组合设计
- **来源**: 《通关！游戏设计之道》第12关
- **核心**: 精心设计的敌人和陷阱能像花生酱和巧克力一样完美搭配——设计时就让它们彼此配合，放进关卡时就会变成多用途工具。
- **详述**: 敌人和陷阱的组合应从设计阶段就规划好彼此的互补关系。经典组合包括：玩家跳过危险陷坑时对岸有敌人（增加时机压力）；敌人躲在移动机关后射击（玩家利用移动障碍物作掩护接近）；旋转电锯既威胁玩家也威胁敌人（玩家可以把敌人引到电锯上）。关键原则：给玩家足够的机会把死亡陷阱转向设置它们的人——害人终害己。只需喷火坑、移动障碍、旋转刀片三种机关，就能组合出大量不同体验。
- **适用场景**: 关卡中战斗区域和危险区域的混合设计。
- **示例**: 顽皮狗的游戏（《古惑狼》《神秘海域》）和Treasure的游戏（《火枪英雄》《炸弹小子》）都是高效利用少量机关组合创造丰富体验的典范。

## 一击毙命陷阱是反设计
- **来源**: 《通关！游戏设计之道》第12关
- **核心**: 一击毙命的陷阱糟糕透顶——如果玩家因陷阱丢命，应让他们因疏忽或时机失误而死，而非被设计者故意杀死。
- **详述**: 罗杰斯用亲身经历总结出"不要朝玩家脑袋扔石头"的原则：一击毙命的陷阱卑鄙且刻意。死亡永远不是教育玩家的好方法，它只会让玩家感到沮丧和伤心。如果玩家必须因陷阱死亡，死因必须是他们自己的失误（疏忽或时机判断错误），让他们意识到"是我自己没做好"而非"设计者故意让我死"。反面案例包括：让玩家在刚进入关卡时赶船，错过了就用石头砸死重新开始；创意总监要求"玩家必须死3次才能前进"的设计理念——用死亡作为游戏设计的尺码是负强化。
- **适用场景**: 任何有惩罚机制和死亡系统的游戏，尤其是陷阱密集的关卡。
- **示例**: 《魔界英雄记》中的宝箱怪（伪装成宝箱的敌人）让玩家之后开真宝箱也变得畏缩，违反了游戏"制造乐趣"的初衷——应该给玩家足够暗示让他们辨识这种敌人。

## 敌人必须融入关卡主题
- **来源**: 《通关！游戏设计之道》第12关
- **核心**: 陷阱和敌人的视觉设计应从关卡主题中寻找灵感，使用看起来属于该关卡特有环境的东西。
- **详述**: 创造陷阱和敌人时，应确保它们在视觉和概念上属于所在关卡的环境。不要在地下城关卡放置与主题无关的高科技陷阱，也不要在丛林关卡中使用工业机械。同时让小道具机关也遵循主题——它们应该看起来像关卡环境中自然存在的物品。当设计者站在玩家角度思考时，应该问"如果我走进这个环境，什么东西看起来是危险的？"从现实世界中寻找灵感（仙人掌、铁丝网），利用形状、颜色、音效和粒子效果让玩家清楚识别危险。
- **适用场景**: 关卡中所有视觉元素的设计，确保世界观一致性。
- **示例**: 《魔界英雄记》地下城关卡中的敌人与陷阱完全贴合主题：从大门伸出的手、装甲人、滴毒铁球、毒水沟棺材盖——每样东西都属于地下城的世界。
