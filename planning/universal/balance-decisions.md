# 决策与平衡性

> 来源：《体验引擎》第5章、第6章

## 决策即情感引擎
- **来源**: 《体验引擎》第5章
- **核心**: 决策是游戏独有的情感触发器，决策在多种可能未来之间进行选择，激发的情感与将来可能发生的事情有关，而非当下。
- **详述**: 大多数情感触发器通过正在发生的事情起作用，但决策不同——它是关于未来可能性的选择。玩家在决策时，潜意识会模拟各种可能的结果（赢或输、生或死），这些可能性本身就会产生情感，无论它们是否真的发生。高水平的象棋棋手长时间凝视棋盘并不发呆，他们的思维在千百种可能性间穿梭，由此产生的情绪起伏正是象棋的魅力所在。决策设计是最纯粹的游戏设计——游戏的核心是交互性，而交互性的核心就是做出决策的那些瞬间。
- **适用场景**: 分析游戏为什么好玩或无聊时；评估游戏情感深度时；设计核心玩法循环时。
- **示例**: 《文明5》中"现在还是下一回合入侵巴比伦"的决策让玩家反复权衡；而一个流程单线化的动作游戏，虽然画面和操作看起来与开放世界游戏一样，但玩家每次只做了唯一能做的事，缺乏决策，情感层面十分空虚。

## 感受未来
- **来源**: 《体验引擎》第5章
- **核心**: 人类拥有感知未来情感的能力，决策激发的情感并不是关于现在发生的事，而是关于将来可能发生的事。
- **详述**: 站在帝国大厦86层的边缘会恐惧，站在花坛上则平静，两种情况实际发生的事一样——站在某处边缘、有风吹过。区别在于潜意识感知到的未来危险程度不同。这意味着游戏设计师不能只考虑玩家会做什么和游戏如何反馈，还要考虑玩家会怎么思考以及他认为游戏会怎么反馈。即使玩家预想的情况不会变成现实，它们依然影响体验。开放世界的动作游戏赋予玩家数百种可能性，让思维不断评估未来；而单线化流程的游戏只有一种可能性，没有决策，情感空虚。
- **适用场景**: 设计关卡和玩法时思考"玩家能感知到哪些可能性"；评估线性流程与开放式设计的情感差异。
- **示例**: 两个操作几乎相同的第三人动作游戏，开放世界版本的每一刻都充满数百种可能——游泳、潜行、搏斗，而单线版本只有一个入口一个出口，动作虽相同但思维过程和情感截然不同。

## 可预测性
- **来源**: 《体验引擎》第5章
- **核心**: 有意义的决策需要部分可预测的未来——完全不可预测则决策无从谈起，完全可预测则决策失去意义。
- **详述**: 如果完全无法预测未来，规划和决策就消失了，游戏退化为纯粹考验反应的活动。如果完全可预测，则不需要思考，只需选择最喜欢的结果。可预测性需要两个条件：一致性（机制在不同情况下作用相同，如马里奥跳跃的物理规则不变）和易于理解（足够简单，玩家可以在其上运用思维）。如果系统不一致或过于复杂，即使存在威胁和机会，玩家也无法感知，情感就不会出现。
- **适用场景**: 评估游戏机制是否让玩家能合理预测结果；判断随机性和复杂度的合理边界。
- **示例**: 马里奥的跳跃以固定方式运作，极限高度固定，下落速度不变，玩家经过练习后能在看到障碍时准确预判如何跳跃、面临什么危险。如果跳跃系统随机变化或复杂到难以理解，玩家就无法规划路线，能做的只是下意识反应。

## 预设决策与系统决策
- **来源**: 《体验引擎》第5章
- **核心**: 预设决策（剧情分支）的结果取决于设计师提前设计的内容，而非玩家能学习和预测的游戏系统，两者创造方式和体验截然不同。
- **详述**: 预设决策如"选择自己的冒险"小说，每个选项和结果都是设计师手工编写的。最好的预设决策让玩家出于虚构层面的原因做出选择（如拒绝毒品因为想做好人）；最坏的情况是玩家猜测设计师意图以获取最优结果。与之对比，系统驱动的决策（如象棋走一步棋、格斗游戏中一次攻击）取决于可学习可预测的游戏系统，思考过程完全不同。预设决策允许玩家和设计师一起创作剧情，这是系统驱动决策无法做到的，但两者不应混淆。
- **适用场景**: 区分剧情选择与玩法决策的设计方法；决定何时使用脚本分支、何时使用系统涌现。
- **示例**: 游戏中NPC向玩家提供毒品——接受触发警察追捕脚本，拒绝则罪犯袭击——这是预设决策，玩家思考的是设计师的意图或虚构价值，而非游戏系统的因果关系。

## 可预测性与AI设计
- **来源**: 《体验引擎》第5章
- **核心**: 游戏AI应更多地被视为可预测的游戏机制而非真正的人类模拟；简单明确的行为比复杂的"聪明"AI更能支撑玩家的策略规划。
- **详述**: "聪明的AI才是好AI"是一个常见但有问题观点。游戏不是模拟真实生活，而是用虚构包装的机制。聪明的AI经常做出不可预期的行为，破坏玩家对未来的规划能力。AI能思考的东西越多，玩家能思考的东西就越少。大多数即时策略游戏中，战士像自动机一样准确执行命令，相同战斗导致相同结果，这种一致性是玩家规划复杂策略的基础。例外是当虚构层（如AI角色之间的复杂交互）比机制驱动的决策更为核心时，可以牺牲简单AI的可预测性。
- **适用场景**: 设计NPC和敌人AI行为时；决定AI复杂度与可预测性的取舍。
- **示例**: 即时策略游戏中，战士行为像自动机而非人类——你下达命令他们立刻准确执行且每次方式相同。如果战士有复杂AI逻辑，同一场战斗可能因AI决定向左走而非向右走而结果不同，此时是AI而非玩家决定了战斗结果。

## 信息平衡
- **来源**: 《体验引擎》第5章
- **核心**: 通过向玩家提供或屏蔽信息来优化决策体验，使之易于理解又不至于太浅显——信息匮乏使决策混乱无序，信息过剩则直接抹杀决策。
- **详述**: 信息平衡是一种优雅的设计方式，它不要求改变机制本身，只需适当隐藏或显示某些部分，通常比重新研究系统要迅速简单得多。信息匮乏让玩家无法预测未来，决策退化为随机猜测；信息过剩让答案过于明显，思考过程消失。设计师尤其难发现信息匮乏，因为自己对游戏了解充分，无法获得新手体验，而且发现信息匮乏意味着承认辛苦制作的机制在新手面前一团乱麻，这在情感上难以接受。信息过剩同样隐蔽，因为游戏会运行得相当顺畅，测试者不会抱怨，问题只是错过了创造更有趣决策的机会。
- **适用场景**: 优化任何决策点的体验质量；诊断"决策不好玩"的问题根源。
- **示例**: 《现代战争2》的心跳传感器能显示墙壁后敌人位置，最初的设计会导致信息过剩（敌人在哪一目了然），设计师添加两个限制——无法侦测"忍者"能力的玩家、周期性而非持续显示——将信息激流分解成缓慢水滴，重新创造出决策和策略。

## 信息匮乏
- **来源**: 《体验引擎》第5章
- **核心**: 当玩家缺少足够信息来理解当前状况时，可预测性被破坏，决策退化为随机猜测和纯粹反应。
- **详述**: 信息匮乏有明显的形式（如棋盘中间竖纸板），也有隐蔽的形式。一种常见情况是"已设计的挑战准备问题"：游戏要求玩家为尚未见过的未来挑战做准备，但玩家在游戏开始时对战斗平衡、道具属性、敌人类型几乎一无所知。结果80%的玩家选择最熟悉的"战士"职业，错过游戏最有趣的部分。另一种隐蔽的信息匮乏来自虚构层传达的信息总是模棱两可——纯粹机制信息（伤害统计、移动频率）难以通过虚构层表达，更好的做法是直接将机制信息传递给玩家。有用的FAQ（如GameFAQs）能让游戏体验明显提升时，这就是游戏信息匮乏的信号。
- **适用场景**: 设计角色创建和初始选择系统时；评估玩家是否拥有足够信息做出有意义决策。
- **示例**: RPG游戏开局就要选择种族、职业、技能、属性，这些决策可能影响50小时的游戏内容，但玩家此时对游戏几乎一无所知。《质量效应2》有6个职业，但80%的玩家选择了最熟悉的"战士"，错过了许多亮点。

## 信息过剩与隐藏信息的方法
- **来源**: 《体验引擎》第5章
- **核心**: 信息过多会直接暴露答案、消灭决策；即使信息完整的游戏也可以通过复杂的因果关系、玩家内心状态、以及速度来隐藏信息。
- **详述**: 隐藏信息至少有三种非传统方式。第一，信息隐藏于未来的复杂因果关系之中——象棋是信息完整的游戏，但三步之后棋盘变成什么样子需要大量思考。第二，信息隐藏在玩家内心状态中——如果能读出对手在想什么，象棋会变得简单得多；多人游戏最吸引人的体验通常来自推断和利用对手内心信息。第三，速度使得信息不可见——大脑察觉、处理和使用信息都需要时间，某一秒最后一瞬间传达的信息完全被隐藏。"石头剪子布"就利用速度隐藏信息，双方几乎同时出拳，没有时间根据对方做出对应决策。
- **适用场景**: 设计多人游戏的隐藏信息机制时；创造信息完整但仍有决策深度的系统。
- **示例**: 扑克的发展史完美诠释信息平衡——从几乎纯随机（信息匮乏）的20张牌版本，到换牌扑克（引入更多信息），到种马扑克（信息过剩，经常一眼看出谁赢），再到德州扑克（公共牌机制达到近乎完美的信息平衡，不可见信息仅两张牌面朝下的牌）。

## 虚构的歧义
- **来源**: 《体验引擎》第5章
- **核心**: 虚构层面的信息经常产生歧义，因为玩家无法分辨哪些虚构是真正的游戏机制、哪些不是，这会导致决策信息匮乏。
- **详述**: 看到桌上有一只烤火鸡，你能抓起它放背包吗？能吃掉吗？能卖掉吗？能扔出去攻击动物吗？在现实中能做的事很多，但没有游戏允许所有行为。玩家看到火鸡图像时无法知道哪些交互是游戏真正支持的，这个图像只是某个不为人知的机制的标志。经典冒险游戏中这个问题极为普遍——玩家背包里有棍子、绳子、泥巴，从虚构角度有无数种从守卫旁溜过的方法，但游戏要求用这些做面具。解法是将难题从虚构层面剥离，用易于理解的游戏机制来构建，让玩家知道与问题有关的全部机制，决策关键在于如何组合使用这些机制。
- **适用场景**: 设计谜题和交互式难题时；评估虚构层是否清晰传达了机制信息。
- **示例**: 《超级马里奥》的跳跃类难题——跳跃机制具有一致性且易于理解，玩家可以细心思考跳跃路线而不担心机制引发的模棱两可。当找到解决方案时，玩家立刻就会意识到这一点。

## 游戏超信息
- **来源**: 《体验引擎》第5章
- **核心**: 玩家从游戏以外获取的知识（游戏类型惯例、技术水平、设计师习惯）会扭曲游戏体验，通常导致信息过剩。
- **详述**: 玩游戏前，玩家已经通过类型猜测游戏时长，通过交流了解难度，通过老套做法预测NPC行为，通过技术限制知道不可能同时出现一万个角色。这种超信息常削弱虚构层的紧张感——玩家知道游戏会保持公平，长时间无资源的情况不会出现，因为那不公平，而游戏目的是娱乐。于是玩家不根据虚构情节思考，而根据设计师习惯和类型常规思考，紧张感和沉浸感随之减弱。处理超信息有两种方法：标准方法承认其存在并确保游戏系统在玩家拥有超信息时仍能产生预期体验；疯狂方法彻底击溃玩家的期望，但危险且难以实现。
- **适用场景**: 评估虚构层紧张感的可信度；设计资源稀缺感和恐怖氛围。
- **示例**: 《系统冲击2》利用疯狂方法——场景是大量尸体和怪物的飞船，资源极度稀缺，敌人数量庞大，几乎没有公平性可言，粗心玩家很快被撕碎。这种看似拙劣的设计却让玩家充分沉浸，因为游戏中的无助感和绝望与现实情况一致。

## 决策与心流
- **来源**: 《体验引擎》第5章
- **核心**: 心流需要以适当频率不断传递决策给玩家大脑——决策太少则无聊（心流断层），决策太多则不堪重负（决策溢出）。
- **详述**: 将玩家大脑想象成一个底部有洞的茶杯，游戏不断加水（决策），保持心流意味着水持续从洞漏出而不溢出。心流持续的时间非常短暂，即使一秒钟的停顿也会影响心流。心流断层有成百上千种呈现方式：技能冷却期间的无所事事、菜单特效和动画的阻碍、被眩晕后完全无法行动。最严重的设计错误就是破坏心流，因为它削弱思维和游戏之间的联系并干扰其他方面的体验。好的心流设计应该在无法消除断层时向其中填充替代决策（如法杖冷却时仍可用刀攻击），或将完全无法动弹的眩晕改为降低但不消除操作能力（如闪光弹致盲但保留移动和听觉）。
- **适用场景**: 评估游戏节奏是否流畅；设计眩晕、冷却、菜单等中断性机制时。
- **示例**: 《现代战争2》的眩晕炸弹减缓移动速度并模糊视线但玩家仍可行动；《反恐精英》的闪光弹让屏幕变白但玩家仍可操作并凭记忆和听觉继续游戏——这些设计保留了眩晕效果的价值，同时维持了心流。

## 决策范围与分类
- **来源**: 《体验引擎》第5章
- **核心**: 决策按思考量分为五类（无须决策、快捷决策、战略决策、深奥决策、不可能决策），混合不同范围的决策决定游戏的节奏感和感受。
- **详述**: "无须决策"答案太明显，仅凭习惯完成，不引发心流。"快捷决策"不到一秒思考，如出拳还是踢、朝左看还是朝右看，是动作游戏的心流基础，因为实现难度不高。"战略决策"需1~5秒思考，明显占据主观意识，需考虑多个角色位置、能力和状态。"深奥决策"需10秒或更多时间，信息超出游戏本身，需要情感、文化和人性知识，只有最优雅微妙的游戏系统才能制造。"不可能决策"超越玩家能力，变成干扰而非心流。决策范围取决于技巧水平——新手眼中的深奥决策对专家来说小菜一碟，游戏技巧上限由范围最大的决策决定。
- **适用场景**: 分析游戏节奏和情感强度；确定游戏的技巧范围和技巧上限。
- **示例**: 大量快捷决策构成血脉贲张的动作游戏；一系列深奥决策构成节奏缓慢的策略游戏；几乎不可能完成的脑力挑战构成解谜游戏。《反恐精英》巧妙混合紧凑的快捷决策（射击）和战略决策（移动与团队策略），加上令人无法企及的技巧上限，提供了源源不断的美妙体验。

## 决策变化与节奏
- **来源**: 《体验引擎》第5章
- **核心**: 避免让玩家长时间面对相同范围和频率的决策，应使用不同强度、范围和主题的决策交替调剂。
- **详述**: 如果一个游戏通过一长串相同范围和频率的决策维持心流，玩家会感到单调乏味。应该抛给玩家一些紧凑的快捷决策，然后给一个深奥难题，接着又是战略决策，依此类推。还可以通过决策作用使同范围决策主题不同——策略游戏中移动军队和建造工厂范围相近但交替出现比单重复效果好。回合制游戏中决策范围不当也有经典问题：太多小范围决策导致"微管理"（操作符号比决策更耗时），决策范围太大导致"分析瘫痪"（玩家需要大量时间思考）。心流节奏的唯一准则是：节奏需要产生变化，不要长时间缓慢也不要长时间高度紧张。
- **适用场景**: 设计游戏节奏曲线和情感节奏；诊断"单调"或"疲惫"的游戏体验。
- **示例**: 《反恐精英》的决策节奏激烈且不可预测——在令人紧张的搜索和枪战之间循环，穿插着快捷决策（射击）和战略决策（团队策略），两局之间有强制的策略规划时间，游戏中的武器有一击毙命的威力使高强度决策段总是很短。

## 公平性平衡
- **来源**: 《体验引擎》第6章
- **核心**: 公平性指游戏开始时没有玩家具备明显优势；对称性游戏天然公平，非对称性游戏需要通过平衡性调整来确保公平。
- **详述**: 公平让玩家觉得输赢是合理的。在竞技游戏中，如果游戏本身不公平，竞技性就无从谈起——输的一方抱怨游戏不公，赢的一方也不满意因为没有让对方心悦诚服。对称性游戏（如冰球，双方以相同状况开始、相同规则）天生公平。但真正的对称性游戏很少，大多数游戏是非对称性的——格斗游戏中角色攻击方式不同、国际象棋白棋先走、《星际争霸》种族不同。非对称性往往是游戏的固有特性，无法回避。此外还有第三个选择：刻意创建不公平游戏，放弃竞技性但获取独特体验，如历史战争游戏的不公平关卡或《太空战场》为每个玩家配备奇怪的不公平能力。
- **适用场景**: 设计多人竞技游戏的初始条件时；评估游戏公平性。
- **示例**: 国际象棋必须有一方先走，这造成先天非对称性，需要通过平衡性调整确保公平。《太空战场》刻意为每个玩家配备奇怪不公平的能力，因为设计师认为"公平一点都不好玩"。

## 深度平衡
- **来源**: 《体验引擎》第6章
- **核心**: 深度平衡的目标不是让所有道具平均好用，而是平衡玩家在各种状态下所选择的策略，使最佳答案不明显，创造细致入微的思考过程。
- **详述**: 一个深度游戏产生的决策必须足够平衡，以至于专家也没有把握给出最佳答案。如果某个选项明显威力更强，决策就失去意义。但平衡不应理解为让不同工具之间相互匹敌——因为没有上下文，"剑和火焰魔法哪个更好"的比较毫无意义。火焰魔法对付食人魔奏效但对付一群哥布林可能不灵，用剑对付哥布林不错但对付食人魔收效甚微。真正的目标是平衡玩家在各种状态下选择的策略。策略可以极其复杂——由多种偶然性组成，依赖多种道具在具体情况下的共同作用，并针对性地抗衡对手的策略。当策略被平衡后，所有玩家的决策都会涉及更加细致的思维过程，游戏深度随之增长。
- **适用场景**: 设计能力/武器/角色选择时的平衡性目标设定；区分"平衡道具"与"平衡策略"。
- **示例**: 英雄对抗食人魔，用剑立刻造成大量伤害，火焰魔法持续造成较少伤害。如果设计师让剑在30秒内伤害更多，超过30秒火焰魔法更优，玩家就必须猜测战斗时长——这种对未来的猜测是一种吸引人的、具有情感意义的思考过程。

## 策略退化
- **来源**: 《体验引擎》第6章
- **核心**: 策略退化指某个策略明显地成为当前最好的选择，它会削弱游戏深度，因为不再需要玩家决定使用什么——答案总是相同的。
- **详述**: 添加一个过于强大的新道具并不会让游戏更好玩，反而会毁掉它，因为不管什么情况答案都一样。真正的策略退化从来不会如此明显，它们只在不同道具和机制之间交互时显露。玩家总是在寻找策略退化——无休止地搜寻游戏设计中的缺陷，利用一切不平衡点来更容易地赢。但一旦真的发现一个，就会咒骂设计师允许他们毁掉游戏。玩家总是在寻找策略退化，内心却希望找不到。平衡性的挑战在于：调整一个机制不仅改变我们希望改变的策略，同时改变所有与它相关联的策略。改变会引发连锁反应，牵连的策略以指数级速度增长。
- **适用场景**: 评估新机制或新内容是否可能引发策略退化；诊断"只有一个最优解"的问题。
- **示例**: 《上古卷轴3：晨风》中通过制作提升智力的药剂→提升后制作更强智力药剂→循环上百倍后合成永久提升全部能力的药剂，游戏开始几分钟就能创造无敌角色。篮球中的"砍鲨战术"——利用奥尼尔罚球差的特点故意对他犯规——也是一种策略退化。

## 可选策略数量的误区
- **来源**: 《体验引擎》第6章
- **核心**: 可选策略数量不是越多越好；一旦有两种可行策略，继续增加不会自动提升体验，真正的目的是丰富玩家的内在思考过程。
- **详述**: 很多人以为两个可选策略比一个好，三个一定比两个更好——这是错误的。"石头—剪子—布—蜥蜴—斯波克"有5种可选策略，每种赢的概率一样，可以继续添加更多策略，但游戏深度并没有增加，只是变得更复杂。扑克每次出牌只有少数几种应对方法（德州扑克经常只有两种：跟或不跟），却非常优秀。平衡游戏深度的真正目的不是最大化策略数量，而是在玩家脑海中创造丰富的思考过程——让玩家顿悟、疑惑、进退两难，并且即使高水平玩家也依然如此。这对设计师意义重大：通过不断增加选择性来提升深度是错误的，通常会徒增复杂度。设计师真正的目的是丰富玩家的内在体验，这很难达到也难以衡量，但追求这个目标会使设计更加小巧、简练、重点突出、优雅。
- **适用场景**: 评估平衡性改进是否真的提升了深度；抵制"多加选项"的设计冲动。
- **示例**: 扑克只有少量可选策略却极度好玩，而一个拥有大量策略的烂游戏可能远不如扑克有趣。增加更多的"石头剪子布"变体（蜥蜴、斯波克）只是增加复杂度而非深度。

## 平衡的目标技巧水平
- **来源**: 《体验引擎》第6章
- **核心**: 让所有技巧水平的玩家都感到平衡几乎不可能，设计师必须决定在哪种技巧水平下追求平衡，允许其他水平下出现策略退化。
- **详述**: 同一个游戏对不同技巧水平的玩家平衡性不同。对专家的策略退化对新手可能是未解之谜，但新手发现了其他专家不用的退化策略。《星际争霸2》的"速攻"战术对新手而言容易实施难以防御，对专家则可以完美化解甚至反制——速攻是一种策略退化，但仅针对低水平玩家。《星际争霸2》团队公开声明宁可牺牲低技巧范围的平衡性以保证最高技巧范围的平衡性。故事驱动游戏则相反——注重中低技巧范围的平衡性。《生化奇兵》有许多退化策略（用炸药桶秒杀强敌），但大多数玩家在不到10小时的通关时间内不会发现。追求高技巧范围的完美平衡需要付出很高代价：剔除可能导致退化的想法、数月测试、发售后的长期补丁。
- **适用场景**: 确定游戏的平衡性目标受众；分配平衡性测试资源。
- **示例**: 《星际争霸2》首席设计师白劳德："当你用枪指着我的脑袋说必须做出决定的时候，我们倾向于让水平最高的玩家们满意。"《生化奇兵》有杀死奶爸的退化策略，但少数玩家能在10小时内发现，即使发现也不影响以故事为核心的体验。

## 平衡方法：最大化核心特征
- **来源**: 《体验引擎》第6章
- **核心**: 找出每个道具的基础作用并将其最大化固定，然后通过调整非核心特征来解决平衡性问题；必要时忍痛割爱删除无法平衡的道具。
- **详述**: 火箭背包必须又远又快地发射人，盔甲必须能保护人，加农炮射程必须远——这些是基本属性，必须最大化并固定。一旦固定，这些道具的作用变得清晰，游戏体验广度扩展，更多策略涌现。然后通过调整其他特性（价格、重量、缺点，甚至创造新机制）来平衡。如果减慢火箭背包的速度，它就不像火箭背包了——可以在受损时爆炸、漏油、缓慢降低生命值来限制它，但绝不能减慢速度。如果无法在不破坏核心特征的前提下平衡，最好的做法是直接删除，而不是削弱——又慢又不管用的火箭背包只会让人恼火，还不如没有。
- **适用场景**: 设计新道具/能力时的平衡性框架；决定是调整还是删除不平衡元素。
- **示例**: 暴雪在《星际争霸2》中将巨大的"雷神"单位改造成普通大小——原始设计太酷但平衡性问题严重（可平地建造导致批量生产无法反制，可修复导致难以击败），暴雪没有用特殊规则掩盖问题而是忍痛割爱，最终效果比充满小陷阱的版本好得多。

## 平衡方法：系统化思维与测试
- **来源**: 《体验引擎》第6章
- **核心**: 不要冲动调整——每个改变会引发连锁反应；通过大量测试构建系统思维模型，收集玩家体验而非建议，才能做出正确决定。
- **详述**: 如果试图一个一个解决单一问题来调整平衡性，就像从贴纸上挤气泡——每个解决方案又引发新问题，游戏在原地打转。人类天性会忽略看不到的问题，已存在的问题如鲠在喉，潜在问题只是抽象可能性。必须在解决的问题比带来的问题更多时，调整才能真正改进游戏。游戏测试的真正目的不是收集意见而是了解其他玩家的体验——通过观察玩家选择和表情就能了解很多，必要时用精心设计的问题套出思考过程。经过10~20次测试后，设计师不再考虑单个故事，而是根据系统和关系来思考，脑海中的思维模型包含上百个因果关系细节。此时才能充分考虑某个改变如何影响整个系统，而非仅考虑最明显的方面。
- **适用场景**: 执行游戏平衡性测试；分析测试反馈并制定调整方案。
- **示例**: 如果测试者建议删除步枪，不要照做——追问为什么：是瞄准系统问题？弹药经济不平衡？不理解某个功能？每条建议背后都有具体的体验原因，找到体验根源才能对症下药。

## 体验结构稳定性
- **来源**: 《创世学说》第3章
- **核心**: 系统的体验结构由数值、成长、操作、运气四个维度组成，保持这些维度的稳定比例关系是玩家持续产生兴趣的基础。
- **详述**: 以DotA为例，影响胜负的四个维度占比约为：英雄选择的数值定位20%、装备成长的打钱速度20%、队友配合的运气成分35%、操作发挥25%。当这个体验结构遭到破坏（如版本更新导致某英雄过强），玩家会感到"游戏变得不熟悉了"——对手选到过强英雄则沮丧，队友选到则胜利与自己无关，自己选到则高估自身能力。这些偏移的体验会让玩家产生迅速拉开距离的想法。体验结构稳定不仅适用于战斗系统，所有系统都是如此——功能性系统可以视为"玩家用货币和时间为战斗力进行战斗得到产出"的战斗系统。只有体验结构稳定，才有可能吸引到目标玩家。上线后体验结构可根据数据调整，但需快速趋于稳定。
- **适用场景**: 设计新系统时确定各影响因素的比例分配；版本更新时评估变更是否破坏了既有体验结构。
- **示例**: 推出全新阶位装备时，必须为各层次玩家手中的装备等级制作对应的替代装备，保证新系统上线后各层次玩家的数值能力不发生突变，避免体验结构剧变导致流失。

## 系统设计的逻辑完整性
- **来源**: 《创世学说》第3章
- **核心**: 系统逻辑的完整性是新手策划最容易忽略的问题——不完整的逻辑往往对整个游戏造成致命影响，尤其在涉及伤害和资源产出的系统中必须慎之又慎。
- **详述**: 《全民飞机大战》的悬停子弹案例：子弹撞击敌机造成一次伤害A后减速悬停，低速过程中造成每秒B点伤害。看似正常，但BOSS后退时悬停子弹恢复初始速度，下一帧又撞击BOSS造成一次A伤害——游戏60帧/秒，BOSS每后退1秒就承受30次A伤害，导致秒杀。修复方法仅是一条规则："每发悬停子弹只能对同一目标造成一次初始伤害"。设计任何系统时，必须考虑所有可能的交互情况：不同帧率下、不同目标行为下、与其他系统联动时。特别是在涉及伤害、资源产出的系统时，一条遗漏的规则就可能被玩家利用来刷钱、刷装备、刷经验，导致经济系统崩溃。深思熟虑、多与有经验的策划沟通、勤勉测试是提升完整性的最好方法。
- **适用场景**: 设计涉及数值计算、资源流转的系统时进行逻辑审查；测试阶段进行极端情况验证。
- **示例**: 装备强化系统设计限购道具时，需考虑间接交易渠道：附魔、镶嵌、邮件寄送、拍卖行、赠送、丢弃拾取、卖商店赎回等所有暗含交易属性的操作都可能导致限购被绕过。

## 数值平衡的迭代循环
- **来源**: 《创世学说》第4章
- **核心**: 数值平衡不是一次到位的，而是"设计→估算→配置→测试→观察反馈→调整"的迭代循环，每一次迭代都让平衡性上一个台阶。
- **详述**: 以MOBA英雄设计为例，数值配置和调整的完整流程：首先根据英雄定位估算初始数值（肉盾多给血和防护，ADC多给攻击和成长），根据技能需要划分数值的成长等级；配置完成后用原型试玩，观察是否达到设计定位目标、平衡性是否过强或过弱、是否出现意料之外的因素（如意外被某些英雄克制，或与特定装备/英雄组合产生意外效果）；根据观察结果评估是否需要调整技能或数值，以及怎么调整；然后重复估算→配置→测试→观察反馈的循环。内部迭代多次后再放到外网灰度测试。伤害公式有三种基本设计方法：减法公式、除法公式和曲线公式，通过拉出散点图可以分辨种类，再根据种类反推公式。关键是不仅关注公式本身，更要注意数值变化背后对应的玩家感受。
- **适用场景**: 设计新角色/武器/技能的数值体系时；上线后根据数据进行平衡性调整时。
- **示例**: 设计一个克制坦克的战士英雄：技能1按目标最大生命值百分比扣血（克制高血量），技能2攻击附带吸血（打野续航），技能3闪现到敌方身后附带击晕（gank留人），大招死亡后3秒原地复活并从周围单位吸血（霸气不怕死）。每个技能数值都需要通过多轮测试迭代才能达到"克制坦克但不过强"的平衡。

## 系统设计的容错与安全机制
- **来源**: 《创世学说》第2章
- **核心**: 系统设计必须预留容错机制——为每个关键功能准备开关，一旦出现漏洞可以在后台快速关闭，争取排查时间而不影响游戏运行。
- **详述**: 装备强化系统让玩家花费大量金钱提升装备能力，一旦出现漏洞被玩家利用，后果可能是整个经济体系崩溃。好的做法是提前为每个功能准备开关机制：淬炼升星功能出问题时，后台通过简单命令行即可让整个功能暂时失效。这样做对游戏运行的影响最小（游戏其他部分还能正常玩），同时争取到了排查问题的时间。此外，还需要考虑反作弊需求：在系统关键节点上记录数据，一旦遭遇外挂可以通过数据追溯封除。对于iOS游戏，还需要注意策划配置表格的设计合理性——因为iOS代码修改需要重新审核（周期约10天），只有通过表格配置修改才能绕过审核，这就要求关键内容都能通过配置表控制。
- **适用场景**: 设计涉及资源产出和消耗的核心系统时；制定线上事故应急方案时。
- **示例**: 游戏上线后发现某活动道具可以通过小号反复领取再交易给大号，如果预先设计了该道具领取功能的远程开关，就可以在几分钟内关闭问题功能，而不需要紧急停服维护。

## 有意义的选择与三角形设计
- **来源**: 《游戏设计艺术》第13章/透镜39、透镜40
- **核心**: 有意义的选择是游戏经济系统的基础——玩家必须面临真正的权衡取舍（三角形：安全但收益低vs冒险但收益高），选择才有意义。
- **详述**: 有意义的选择意味着每个选项都有明显优势和明显劣势，玩家必须权衡取舍。如果某个选项在所有情况下都优于其他选项，那根本不是选择。三角形是一种经典的有意义选择：走安全路线获得少量奖励，还是走危险路线获得大量奖励？三角形与可变奖励结合时效果更强——如果危险路线的奖励是随机的（有时一无所获有时获得巨额），比固定奖励更吸引人。游戏中几乎所有有趣的决定都可以归结为某种形式的三角形。
- **适用场景**: 设计资源获取机制、路线选择、风险回报系统时。
- **示例**: 《吃豆人》中豆子让玩家减速（危险），但吃掉豆子得分并朝通关前进（奖励）——完美的三角形。

## 游戏经济系统的双重循环
- **来源**: 《游戏设计艺术》第13章/透镜52
- **核心**: 游戏经济由"怎么赚钱"和"怎么花钱"两个有意义的选择组成循环——赚钱和花钱交替推动玩家前进，循环的深度决定了经济的深度。
- **详述**: "钱"可以换成一切能交易的东西——技能点、经验值、资源、装备。重要的是玩家面临两个选择：如何获取资源，以及如何消耗资源。这两个选择构成循环，因为花钱的方式通常帮助玩家赚更多钱，又给更多机会花钱。平衡游戏经济需要同时考虑：公平（有玩家通过特定方式获利吗）、挑战（有玩家能买到让游戏太容易的东西吗）、选择（有足够的方式赚钱和花钱吗）、概率（赚钱基于技巧还是运气）、时间（赚钱花费的时间太长还是太短）、奖励（赚钱有奖励感吗）、自由（玩家能以希望的方式买到想要的东西吗）。
- **适用场景**: 设计游戏的资源循环系统、货币获取和消耗机制时。
- **示例**: 《暗黑破坏神》的装备系统是经济循环的经典——打怪获得装备/金币（赚钱），用装备打更强的怪或交易更好的装备（花钱），循环往复。

## 平衡直觉的加倍减半法则
- **来源**: 《游戏设计艺术》第13章/席德·梅尔法则
- **核心**: 调整数值平衡时，不要做微量调整（加减10%），而是直接加倍或减半——大幅调整才能清晰理解数值在游戏中如何运作。
- **详述**: 这条规则归功于席德·梅尔。如果火箭造成100点伤害觉得太多，不要减10或20点调成90或80，而要直接设成50。如果太低，再取50和100的中间值75。把数值推向直觉反应越远的地方，优秀平衡的界限就越明晰。微量调整的问题是：你不确定修改是否生效了（也许是随机波动），于是反复微调浪费大量时间。大幅调整让你"马上感到实际差别"，快速理解数值的运作方式。同时，通过"精确猜测"（猜13.8而非"大概14"）训练你的设计直觉。
- **适用场景**: 数值平衡的初始调参阶段；训练设计师的数值直觉时。
- **示例**: 席德·梅尔在《殖民帝国》开发中反复教导年轻设计师：修改东西要能马上感到差别，不要反复微调——这让你理解数值如何运作，而不是怀疑修改是否生效。

## 人类对抗机器人的迭代平衡法
- **来源**: 《游戏设计梦工厂》第11章
- **核心**: 通过让人类玩家团队对抗由脚本控制的AI实体来调整平衡性——如果AI过早被摧毁则增强AI，如果AI成功逃脱则削减AI强度，直到战斗过程呈现惊心动魄的剧本式效果。
- **详述**: 这是一个系统化的平衡练习：先设计机器人AI的行动列表（如前进、转向、激光攻击、击打），然后让多辆坦克（由玩家控制）对战机器人。机器人按行动列表逐一执行指令，坦克从行动列表中选择3个行动（单一行动至多执行两次）。观察战斗结果后调整参数：如果机器人过早被摧毁则增强（增加伤害、增加行动、添加新能力）；如果机器人轻松逃脱则削弱；如果被摧毁的坦克太少则给机器人增加攻击指令或提升伤害。核心目标是让战斗结果像好莱坞大片——机器人在即将逃脱的瞬间被击毁，大多数坦克也被干掉。这种"对抗式调整"可以精确地将游戏体验推到设计者期望的情感区间。
- **适用场景**: 设计PvE战斗系统时调整Boss与玩家团队的平衡性；通过迭代对抗测试找到最佳难度参数。
- **示例**: 设计一个机器人Boss时，初始版本太弱（被坦克轻松击败）→增强激光和击打伤害→仍然太强（坦克全灭）→减少行动列表中的攻击频率→经过3~8轮迭代后达到"惊心动魄的最后一刻"效果。

## 实力均衡的怪圈——打破僵局
- **来源**: 《游戏设计梦工厂》第11章
- **核心**: 多人游戏中当任何领先者都会被其余玩家联手压制时，游戏会陷入无人获胜的僵局——设计师必须制造条件打破均衡，让领先者有机会击败联合力量。
- **详述**: 在一款3名玩家互相角逐称霸世界的游戏中，无论谁领先，另外两名玩家都会合伙将领先者拉下水，于是创造了无人能够获胜的怪圈。这种停滞源于系统中的负反馈过于强大。解决方案不是削弱负反馈（那会导致滚雪球效应），而是有策略地添加"打破均衡"的条件——比如给予领先者一种特殊能力或资源优势，使其在面对联合对抗时仍有获胜可能。关键是在"防止滚雪球"和"允许游戏推进"之间找到精确的平衡点。类似的问题在《大富翁》和《冒险》（Risk）等经典桌游的后期阶段都普遍存在。
- **适用场景**: 设计多人游戏的后期阶段；处理负反馈过强导致游戏陷入僵局的问题。
- **示例**: 3人策略游戏中领先者获得"终极武器"使用权，使其有能力突破另外两方的联合防线，从而打破无人获胜的僵局——但要控制这种能力的强度，避免变成单纯的正反馈。

## 奖励与惩罚的平衡怪圈
- **来源**: 《游戏设计梦工厂》第11章
- **核心**: 当游戏中的奖励和惩罚总是趋于平衡时，玩家会陷入无法摆脱的困境——全部收益都用于偿还债务，无论多么努力都无法前进，形成一种特殊的停滞。
- **详述**: 这种平衡怪圈在模拟经营类游戏中尤为常见。玩家的全部收益都用于偿还债务，无论多么努力都无法摆脱。这是一种隐蔽的停滞——游戏看似正常运行（有收入有支出），但玩家实际上无法取得任何实质性进展。解决方案包括：通过意外事件改变局势（意外收获让玩家摆脱债务，或天灾彻底重置局面）；调整游戏系统从根本上避免玩家陷入此境（给予债务减免、提高利率等）；或者设置一个将游戏推向明确方向的机制。核心原则是：游戏应该始终让玩家感到自己的努力在推动前进，而非在原地打转。
- **适用场景**: 设计游戏经济系统时避免"收入全部用于支出"的死循环；诊断游戏中玩家感到"无论怎么努力都没用"的问题。
- **示例**: 模拟经营游戏中玩家陷入全部收入还债的困境→通过随机事件"中彩票"获得意外收入→玩家终于有资金投入新的生产→摆脱恶性循环开始正向发展。
