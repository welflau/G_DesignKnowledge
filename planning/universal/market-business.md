# 商业与市场设计

> 来源：《体验引擎》第10章

## 设计目的
- **来源**: 《体验引擎》第10章
- **核心**: 每一个设计决策都会受到游戏商业目的的影响，好游戏不一定成功，成功取决于对目的性的理解、市场切入和定位
- **详述**: 不同商业模式对设计有截然不同的要求。街机游戏的设计目的是让玩家投币：操作简单以降低门槛、几分钟内结束以加速周转、弹性成功条件和高技巧上限以吸引回头客。MMO游戏采用包月制，目的是让玩家玩的时间尽可能长，因此需要大量内容、深度升级系统和增加依赖性的社交社区。传统拷贝销售则鼓励纯粹的体验设计，因为获得好评的最好方法就是创造优秀体验。非商业游戏（艺术类、学术实验）则各有不同目的——表达观点、获取地位或纯粹自娱。每一种模式都是不同的设计挑战，也都蕴含巨大潜力。
- **适用场景**: 确定游戏的商业模式和目标受众，以此指导所有后续设计决策
- **示例**: 街机游戏《吃豆人》设计了简单直观的操作和几分钟一局的节奏——这并非偶然，而是服务于"让路过的人投币"这一核心商业目的

## 市场竞争
- **来源**: 《体验引擎》第10章
- **核心**: 游戏是非竞争性商品，市场呈"赢者通吃"格局——最好的游戏获得所有用户，其他游戏即使不错也可能无人问津
- **详述**: 与水管工这种"竞争性商品"不同（一个大师级水管工同时能服务的客户有限，普通水管工总有活干），游戏一旦被创造出来，复制和分发的边际成本几乎为零。这意味着一个卓越的游戏可以卖给无限多的用户，而二流产品则无人问津。这种赢者通吃的特性使得游戏市场竞争极为残酷：大多数游戏都会输，而失败者往往被遗忘。我们看到的总是成功者的访谈和获奖展示，但只要看一眼未经筛选的游戏发售列表，就会被数量所震惊。
- **适用场景**: 理解游戏市场的残酷性，明确必须做到最好才能生存
- **示例**: 《使命召唤：黑色行动》的利润超过利比里亚整个国家的GDP，因为它占领了第一人称射击游戏的市场

## 马太效应
- **来源**: 《体验引擎》第10章
- **核心**: 受欢迎的游戏会变得更加受欢迎——更多玩家带来更健壮的社区、更多口碑传播和更强竞争力，形成自我强化的正循环
- **详述**: "凡有的，还要加给他，叫他有余。没有的，连他所有的，也要夺过来。"马太效应在游戏行业有多个驱动力。首先，游戏是反竞争性商品——玩的人越多，社区越健壮，用户创造内容越丰富，口碑传播越广。其次，成功开发者在经济、信誉和地位上有优势，可以招募更多天才、获得更多创意自主权。此外，用户天然倾向于熟悉的事物，成功游戏因此拥有额外的竞争手段。马太效应预示赢家似乎命中注定，但在现实中，创新者的困境可以打破这一循环。
- **适用场景**: 理解为什么已有竞品难以超越，以及成功如何自我巩固
- **示例**: 《我的世界》通过玩家社区传播、用户创造内容和口碑效应，形成了其他沙盒游戏难以撼动的网络效应

## 创新者的困境
- **来源**: 《体验引擎》第10章
- **核心**: 成功的工作室会因股东压力、团队固化、习惯根深蒂固和自满而失去创新能力，给后起之秀留下机会
- **详述**: 新兴工作室一无所有，因此敢于冒险尝试疯狂想法。但成功带来黑暗面：股东开始关注业绩，团队成员树立期望和设想并形成专业群体，习惯根深蒂固，官僚主义取代了灵活的小公司文化。更糟糕的是，成功的设计师可能不再质疑自己——或因自负、或因害怕失去已有的一切、或只是懒得再付出创新所需的努力。最终，功成名就的工作室变成了"不会移动的靶子"，等待着被某个带着疯狂点子的菜鸟击倒。创新者的困境抵消了马太效应：大公司虽有规模和资源优势，但被成功所牵制，无法像小公司那样大胆创新。
- **适用场景**: 理解为什么市场领导者会被颠覆，以及小团队如何利用大公司的保守性找到突破口
- **示例**: 一个无名设计师没有包袱，敢于尝试前所未有的游戏概念；而他成功后的工作室却因害怕风险而只做安全续作，最终被下一个无名菜鸟颠覆

## 细分市场
- **来源**: 《体验引擎》第10章
- **核心**: 市场按兴趣、技能、付费能力、文化等维度划分为不同细分领域，大细分市场利润高但竞争激烈，小细分市场竞争少但回报有限
- **详述**: 细分市场会以复杂的方式互相交叠，一个游戏可以适用于许多不同的细分市场。大细分市场的丰厚利润吸引大量开发者涌入，竞争加剧导致利润下降，最终市场趋于平衡。然而现实中无人能完美理解市场规模——有多少人喜欢和马有关的游戏？细分市场总以复杂方式交叠，文化和科技持续变化，因此总有未开发的细分市场能创造高额利润，前提是你能发现它们。发现未开发的细分市场是最好的市场策略，但难度和风险也最大——没有确定的玩家、没有参考对象、没有规定可以遵循，甚至无法知道会有多少人喜欢一个完全原创的游戏。
- **适用场景**: 确定游戏的目标市场规模、竞争强度和潜在回报，寻找市场空白
- **示例**: 《模拟人生》开创了家庭管理类游戏的细分市场——当时所有人（包括测试人员、公司主管、甚至设计师赖特本人）都不确定这个市场是否存在，最终它卖出了超过1亿份

## 价值曲线
- **来源**: 《体验引擎》第10章
- **核心**: 通过将不同游戏在各种"市场价值"维度上打分并绘制对比图，可以清晰看出哪些价值重要、哪些不重要、哪些完全没有竞争
- **详述**: 市场价值是指能用特定方法吸引某些玩家群体的游戏体验。价值曲线是游戏对比图，通过展示不同游戏在各市场价值上的得分来揭示竞争格局。只有在竞争对手之上或完全没有竞争对手的价值才是重要的——如果一个价值在别处有更好的替代品，它就没有卖点。最伟大的胜利是创建完全没有竞争对手的价值：放弃在对手擅长的领域竞争，转而发明原创的独特价值。通过价值曲线的对比，设计师可以明确应该把开发重点放在何处——那些最独特的和做到最好的价值上。
- **适用场景**: 分析市场竞争格局，确定游戏的差异化方向和开发优先级
- **示例**: 《生化奇兵》没有在潜入系统深度上与《杀出重围》竞争，而是创造了独有的"艺术装饰风格的游戏世界"和"奶爸/小女孩生态系统"——这些价值在市场上完全没有竞争对手

## 价值焦点
- **来源**: 《体验引擎》第10章
- **核心**: 把一件事做到极致远胜于做十件平庸的事——成功的小游戏几乎都坚持不懈地专注于一两个独特价值
- **详述**: 每一种价值都消耗资源，引入过多价值会增加游戏模糊和过于复杂的风险，有些价值甚至会产生不兼容。想要在市场中体现卓越价值，仅靠做得好或努力工作是不够的，必须把资源和重心集中在能做到极致的方面。成功的小游戏价值曲线看起来像"又高又细的钉子"——一两个突出价值加上少量次要价值。失败的游戏价值曲线则像"矮树桩"——什么都有但什么都不突出，通常源于天真的雄心壮志：看到喜欢的游戏就想做一个更好的，但没有优越的研发资源，几乎注定失败。团队越小，越需要聚焦于唯一的价值并排除其他。
- **适用场景**: 确定开发资源分配，特别是在团队规模有限时决定哪些功能值得投入、哪些应该放弃
- **示例**: 《盖瑞模组》由一个人开发，只专注于"千奇百怪的创意性"这一个价值，在几乎无竞争的情况下卖出超过100万份；《超级食肉男孩》专注于"超高速的平台跳跃"；《我的世界》专注于"创意十足地构建整个世界"

## 证实偏见
- **来源**: 《体验引擎》第10章
- **核心**: 玩家根据已知信息来玩游戏——他们的预期会决定他们看到什么、记住什么、如何理解游戏事件，甚至能完全逆转对同一事件的解读
- **详述**: 证实偏见（confirmation bias）指人们在认知事物时总倾向于证实已知的观点。啤酒加醋在盲测中胜出，但告知成分后人们就讨厌它；世界级小提琴家在地铁站演奏被完全忽略；价格高的商品被认为质量更好。在游戏中，证实偏见无处不在：听说游戏充满艺术感，你就在游戏中寻找并发现艺术细节；看到评价不高，你会特别注意每一次挫折。证实偏见甚至能逆转对同一事件的解读——如果预期是"令人满意的挑战"，一次失败会被视为精心设计的体验；如果预期是"游戏不平衡"，同样的失败就会被归咎于设计缺陷。
- **适用场景**: 理解营销和口碑如何深刻影响玩家对游戏体验的感知，以及为什么营销是体验设计的一部分
- **示例**: 百事挑战中盲测时百事可乐胜出可口可乐，但展示商标后结果就不同了——品牌预期改变了味觉体验

---

## 设置预期
- **来源**: 《体验引擎》第10章
- **核心**: 游戏名称、营销信息和玩家口碑是设置预期的三大渠道，精心设置的预期能让玩家获得最好的游戏体验
- **详述**: 预期设置有三条主要渠道。第一，游戏名称是玩家接触到的第一个预期：《毁灭战士》暗示暴力和力量，《模拟城市》暗示创造和管理，《地狱边境》暗示黑暗和氛围。第二，营销信息（广告、访谈、文章）进一步塑造预期——开发者说灵感来自花园和孩子的游戏，与说灵感来自暴力影片的游戏，会建立完全不同的玩家期望。第三，玩家间的口口相传是最强大的预期设置方式，因为它通过社会压力放大——人类天生会将意见与周围人保持一致，甚至会调整记忆使其与他人吻合。第一个评论可能引发滚雪球效应，最终成为这个游戏的文化标签。设计师应把市场人员当作核心体验团队的一部分。
- **适用场景**: 制定游戏的命名策略、营销方向和社区管理策略，确保玩家带着正确的心态进入游戏
- **示例**: 《黑暗之魂》通过名字和"高难度"的口碑，让玩家将死亡视为刻意设计的挑战体验而非设计缺陷——同样的死亡机制在另一款游戏中可能被视为"不公平"

## LeBlanc的八种乐趣分类
- **来源**: 《游戏设计艺术》第10章/透镜20
- **核心**: 玩家从游戏中获得的乐趣可以归纳为八种基本类型——感知乐趣、叙事乐趣、挑战乐趣、同伴乐趣、发现乐趣、表达乐趣、 submission（顺从）乐趣、沉浸乐趣。了解你的游戏提供哪种乐趣才能精准定位受众。
- **详述**: Marc LeBlanc提出的八种乐趣类型：1）感知乐趣——感官愉悦（视觉、听觉、触觉）；2）叙事乐趣——故事和戏剧性；3）挑战乐趣——克服困难的成就感；4）同伴乐趣——社交互动、友谊、竞争；5）发现乐趣——探索和发现新事物；6）表达乐趣——自我表达和创造；7）顺从乐趣——服从规则、享受游戏作为避风港；8）沉浸乐趣——进入心流状态忘我游玩。每款游戏通常提供2-3种主要乐趣类型。如果你的游戏试图满足所有乐趣类型，可能反而什么都不够深入。了解你的核心乐趣类型才能精准营销——"喜欢挑战乐趣"的玩家和"喜欢同伴乐趣"的玩家是截然不同的受众。
- **适用场景**: 确定游戏的核心体验定位、制定营销信息、分析竞品差异化时。
- **示例**: 《暗黑破坏神》主要提供挑战乐趣+发现乐趣+感知乐趣；《模拟人生》主要提供表达乐趣+叙事乐趣；《动物森友会》主要提供同伴乐趣+顺从乐趣+发现乐趣。

## 创新的评判标准
- **来源**: 《游戏设计艺术》第10章/透镜24、透镜25
- **核心**: 创新不是目的而是手段——用"评断透镜"检验创新是否真正改善了体验：如果去掉这个创新元素游戏是否变得无聊？如果是，创新有价值；如果去掉后体验不变，创新就是装饰。
- **详述**: 创新透镜要求设计师问：我的游戏有什么前所未有的元素？如果去掉创新元素，游戏是否就不那么好了？创新是否服务于体验（改善了核心玩法）还是只是"不一样"（为创新而创新）？评断透镜则更广泛：这个设计决策是否让游戏体验更好？如果不确定，去掉它看看会不会有人注意到。最危险的创新是"看起来很酷但对体验没有贡献"的创新——它增加了开发成本和复杂度，却没有增加玩家乐趣。成功的创新通常是将一个有价值的新想法深度融入核心玩法，而非在边缘添加花哨但无关的功能。
- **适用场景**: 评估新功能的必要性、砍掉不必要的创新、聚焦核心体验时。
- **示例**: 《传送门》的创新（传送门机制）是核心玩法的基础而非装饰——去掉传送门游戏就不存在了，这是有价值的创新。

## 游戏产业的规模与数字化转型
- **来源**: 《游戏设计梦工厂》第15章
- **核心**: 全球游戏业产值已达数百亿美元并持续增长，但增长领域在数字分发和新兴平台（移动、社交），而非传统零售——设计师必须理解这个转型对设计决策的影响。
- **详述**: 2013年全球游戏业产值660亿美元，预计2017年达780亿。关键趋势是：零售软件收入持续下降，而数字分发、订阅和社交游戏收入不断上涨。美国58%的人经常玩游戏，45%玩家为女性，36%玩家超过36岁。玩家构成的变化意味着游戏不再只是"年轻男性"的专属——成年女性占游戏人口31%，远超未成年男性的19%。亚太占全球市场36%（北美仅32%），休闲、社交和移动游戏占据27%的全球市场。这些数据传达的信息是：游戏的受众在扩大，平台在多元化，数字分发正在取代实体零售。设计师需要根据目标受众和平台特性来调整设计策略。
- **适用场景**: 确定游戏的目标平台、目标受众和商业模式时；理解行业趋势对设计的影响。
- **示例**: 移动游戏的崛起不是因为手机比主机性能更强，而是因为手机覆盖了更广泛的受众群体（包括大量非传统玩家），且分发成本极低。

## 发行商与开发商的生态关系
- **来源**: 《游戏设计梦工厂》第15章
- **核心**: 发行商预付版税给开发商作为开发资金，开发商交付产品——理解这个基本经济关系及其对设计决策的约束是职业设计师的必备知识。
- **详述**: 发行商的核心职能包括：提出游戏概念、投资开发、提供QA测试、营销推广、分发游戏。开发商的核心职能是利用发行商资金完成游戏设计、程序、美术和音效。发行商根据三点评估项目：团队（经验、历史、协作能力）、项目规划（进度和预算）、创意（是否迎合目标受众的购买趋势）。发行商最看重的是团队能否在规定时间和预算内做出优秀游戏——创意反而是最不被看重的。发行商依赖销售数据判断玩家想要哪类游戏，如果游戏不属于热卖类型会有所顾虑。授权内容（知名角色、IP）可以提高曝光率和销量、降低投资风险，因此被发行商大量使用。主机平台还有严格的授权协议和质量审核。
- **适用场景**: 与发行商洽谈项目时理解其评估标准；理解为什么某些设计决策受商业因素制约。
- **示例**: 《杀出重围》虽然极具创新性，但其玩法建立在成熟的第一人称射击和角色扮演两个类型之上——发行商更愿意投资在成熟类型基础上的创新。

## 独立游戏的机会——游戏节与社区生态
- **来源**: 《游戏设计梦工厂》第13章/第15章
- **核心**: 独立游戏开发者可以通过游戏节（IndieCade、IGF）、GameJam和数字分发平台直接触达玩家和业界——这些渠道不仅展示作品，更是建立人脉、获得反馈和找到发行机会的跳板。
- **详述**: IndieCade等游戏节侧重展示独立游戏中的卓越设计和创新，提供认识同行、建立人脉的宝贵机会。许多成功游戏首次亮相于IndieCade：Johann Sebastian Joust、P.B. Winterbottom、《环绕走廊》、《 skulls of the Shogun》等。GameJam（如Global Game Jam）是年轻设计师锻炼技术、融入社区的好渠道。独立开发者的优势是：技术不再成为瓶颈（Unity、Flash、GameMaker等工具降低了门槛）；数字分发（Steam、App Store、Google Play）绕过了传统零售壁垒；玩家社区渴望新鲜体验。独立开发者的建议是："为自己所爱而设计，探索最离经叛道的想法"——游戏节寻求的正是开发者心血之作。
- **适用场景**: 独立开发者规划发布策略；选择参加哪个游戏节或GameJam来展示作品。
- **示例**: 《风之旅人》团队的Robin Hunicke建议"多动手、体验各种感受，所有这些将转换成你需要的设计素材"；高桥庆太（《块境》设计师）则说"不要在乎别人对你创意的看法，先建造自己的世界之后再听取反馈"。

## 游戏类型的商业价值与创意限制
- **来源**: 《游戏设计梦工厂》第15章
- **核心**: 游戏类型对设计师而言是一把双刃剑——它提供了描述游戏风格的共同语言和商业定位工具，但也会限制创意，让设计师倾向已有方案。
- **详述**: 游戏类型让设计师和发行商在描述游戏风格时有共同语言，快速了解游戏的目标市场、最适合的平台、谁更合适开发等。2012年最热卖的类型是动作（22.3%）和射击（21.2%），其次是体育（15.3%）。不同平台的类型分布差异巨大：移动平台上休闲和策略游戏占据主导。然而游戏类型也会限制创意。Fullerton在讨论设计时从不强调类型，因为过度关注类型会让设计师被已有的类型惯例所束缚。正确的态度是：把游戏类型当作项目在商业方面的考虑，不要让它在设计过程中扼杀了你的创意。关键是在创新尺度和商业可行性之间取得平衡。
- **适用场景**: 项目立项时确定游戏类型定位；向发行商提案时用类型语言沟通；评估创新的商业风险。
- **示例**: 《杀出重围》将第一人称射击和角色扮演两种成熟类型融合，在创新尺度上大刀阔斧同时具有超一流团队支持——这比创造一个全新类型更容易获得发行商投资。
