## 正文
<h2 align="center">快速开发框架</h2>
<p>
游戏产品的非业务型功能、系统和工具，都有高度的相似性和可复用性，基于这样的框架来开发项目，将会大幅地提高项目开发的效率。
在产品研发中不断地积累整理可复用的开发资源，并应用到不同的项目中，让新的项目可以非常快速、稳定地启动并完成。
</p>

## 概要

| 维度 | 内容 |
|------|------|
| **作用** | 沉淀一套可复用的前后端框架、工具库和生产线，让新项目可以站在已有成果之上快速起步，而不是每次都从零搭建 |
| **应用场景** | 新项目快速启动、提高开发效率、减少重复开发、统一技术栈 |

| 问题 | 解决方向 |
|------|----------|
| **如何构建快速开发框架** 需要构建可复用的开发框架 | 先从「通用层」和「工具库」开始抽象，再逐步抽取业务层的可复用模块；在真实项目中验证和打磨，而不是为了框架而框架；制定框架构建计划；建立框架规范 |

| 要点和思考方向 |
|----------------|
| 快速开发框架是提高开发效率的重要手段 |
| 从通用层和工具库开始，逐步构建 |
| 在真实项目中验证和打磨 |
| 避免过度设计，保持实用性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「通用层」「工具库」「可复用」「模块化」「新项目」 |
| **方法** | 让 AI 设计框架分层、抽取策略、验证方案 |
| **应用** | 框架规划、模块抽取、新项目模板 |
| **提示词范例** | 「团队需快速开发框架，从通用层和工具库开始，请设计 C# 客户端框架的分层架构、模块化接口，以及新项目如何按需启用/替换模块的接入方案」 |

## 通用层

| 维度 | 内容 |
|------|------|
| **作用** | 为所有前端项目（游戏客户端）提供统一的基础能力 |
| **应用场景** | 游戏客户端开发、统一技术栈、快速项目启动 |

| 问题 | 解决方向 |

| **如何构建通用层** 需要构建统一的通用层 | 设计模块架构；实现模块接口；优化模块性能 |

| **资源系统** | 资源加载、资源管理、资源缓存 |
| **数据系统** | 数据管理、数据持久化、配置系统 |
| **UI 系统** | UI 框架、UI 管理、UI 组件 |
| **声音系统** | 声音播放、声音管理、声音池 |

| 要点和思考方向 |
|----------------|
| 通用层是前端框架的基础 |
| 实现模块化提高灵活性 |
| 提供统一接口便于使用 |
| 建立模板工程提高效率 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「网络」「资源」「数据」「UI」「声音」「生命周期」「模块化」 |
| **方法** | 让 AI 设计通用层架构、模块接口、生命周期管理 |
| **应用** | 客户端通用层、模块设计、模板工程 |
| **提示词范例** | 「游戏客户端需通用层：网络、资源、数据、UI、声音，请设计 C# 模块化架构，统一生命周期与接口，以及按需启用/替换模块的实现方案」 |
