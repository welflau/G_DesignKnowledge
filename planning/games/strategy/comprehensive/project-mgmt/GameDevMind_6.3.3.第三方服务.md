## 云服务商

| 维度 | 内容 |
|------|------|
| **作用** | 提供产品服务端运行的环境 |
| **应用场景** | 游戏服务器部署、数据存储、CDN加速等 |

| 服务商候选 | 说明 |
|------------|------|
| **AWS** | 全球领先的云服务商 |
| **阿里云** | 国内主流云服务商 |
| **腾讯云** | 国内主流云服务商 |
| **UCloud** | 国内云服务商 |
| **华为云** | 国内云服务商 |
| **网宿** | CDN和云服务商 |

| 问题 | 解决方向 |
|------|----------|
| **如何降低基础设施成本？** | 使用云服务商；云服务商选择需考虑价格、服务、地域覆盖等因素；关注云服务商的稳定性和技术支持能力 |

| 要点和思考方向 |
|----------------|
| 云服务商选择需要考虑价格、服务、地域覆盖等因素 |
| 需要关注云服务商的稳定性和技术支持能力 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「AWS」「阿里云」「腾讯云」「CDN」「部署」 |
| **方法** | 让 AI 设计云服务选型、部署架构、成本评估 |
| **应用** | 服务器部署、数据存储、CDN |
| **提示词范例** | 「游戏服务端需云服务（AWS、阿里云、腾讯云等），请设计选型考量（价格、地域、稳定性）、部署架构与 CDN 加速方案」 |

## 广告服务

| 维度 | 内容 |
|------|------|
| **作用** | 在游戏中集成广告，帮助变现 |
| **应用场景** | 游戏内广告展示，实现广告变现 |

| 问题 | 解决方向 |
|------|----------|
| **如何最大化广告收益？** | 使用广告聚合平台，整合多个广告源；优化广告展示策略；平衡用户体验和广告收益；关注广告质量和用户反馈 |

| 要点和思考方向 |
|----------------|
| 聚合平台可以整合多个广告源，提高填充率和收益 |

| 广告展示要合理，避免过度影响用户体验 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「广告聚合」「广告变现」「填充率」「用户体验」 |
| **方法** | 让 AI 设计广告接入、聚合策略、展示优化 |
| **应用** | 广告变现、多源聚合、收益优化 |
| **提示词范例** | 「游戏内广告需聚合多源、最大化收益，请设计广告聚合接入、展示策略与用户体验平衡要点」 |

## 运维服务商

| 维度 | 内容 |
|------|------|
| **作用** | 提供运维服务，支持资源管理和维护 |
| **应用场景** | 需要专业运维支持的产品 |

| 服务范围 | 说明 |
|----------|------|
| **资源管理** | 主机、数据库（备份、恢复）等资源管理 |
| **主机安全** | 破解、DDos攻击、安全漏洞等安全防护 |
| **全球加速** | 跨地区网络加速 |
| **问题排查** | 运维问题排查和解决 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决资源管理/安全问题/全球加速/运维问题排查？** | 使用运维服务商；选择有良好口碑和稳定服务的服务商 |

| 要点和思考方向 |
|----------------|
| **候选服务商**：有需要私聊 |
| 运维服务商可以提供专业的运维服务，降低运维成本 |
| 可以快速响应运维问题，提高服务稳定性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「运维」「主机」「数据库」「DDos」「全球加速」 |
| **方法** | 让 AI 设计运维服务选型、资源与安全方案 |
| **应用** | 资源管理、安全防护、问题排查 |
| **提示词范例** | 「运维需资源管理（主机、数据库）、安全防护（DDos、漏洞）、全球加速、问题排查，请设计运维服务商选型与对接要点」 |

## 游戏分发

| 维度 | 内容 |
|------|------|
| **作用** | 不上应用商店，也能让用户安装游戏 |
| **应用场景** | 绕过应用商店的分发需求 |

| 分发流程 | 说明 |
|----------|------|
| **封装** | 将游戏封装成可安装的包 |
| **签名** | 对游戏包进行签名 |
| **分发** | 通过分发平台分发游戏 |

| 服务商候选 | 说明 |
|------------|------|
| **第八区** | 游戏分发平台 |
| **乐变** | 游戏分发平台 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现游戏分发？** | 封装、签名、通过分发平台分发；注意合规性；关注分发平台的服务质量和稳定性 |

| 要点和思考方向 |
|----------------|
| 游戏分发可以绕过应用商店，但需要注意合规性 |
| 需要关注分发平台的服务质量和稳定性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「分发」「封装」「签名」「第八区」「乐变」 |
| **方法** | 让 AI 设计分发流程、平台选型、合规考量 |
| **应用** | 游戏分发、绕过应用商店、合规 |
| **提示词范例** | 「游戏分发需封装、签名、分发平台（第八区、乐变等），请设计分发流程、平台选型与合规性考量」 |

## 游戏热更新

| 维度 | 内容 |
|------|------|
| **作用** | 产品在运营时期快速应对变化，不用向各平台发布新包即可实现问题修复、功能开发或内容变更 |
| **应用场景** | 快速修复问题、更新内容、发布活动 |

| 服务商候选 | 说明 |
|------------|------|
| **GLTOP热更生产线** | 支持任意平台；热更资源一键生成与发布；有需要私聊 |
| **乐变热更** | 支持Android热更 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现快速更新？** | 使用游戏热更新服务；选择支持多平台的热更新服务；关注热更新的稳定性和兼容性 |

| 要点和思考方向 |
|----------------|
| 游戏热更新可以大大提高运营效率 |
| 需要选择支持多平台的热更新服务 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「热更新」「GLTOP」「乐变热更」「多平台」 |
| **方法** | 让 AI 设计热更服务选型、平台支持、稳定性考量 |
| **应用** | 热更服务接入、快速更新 |
| **提示词范例** | 「游戏热更新需支持多平台、一键生成与发布，请设计 GLTOP/乐变等热更服务选型、接入流程与兼容性考量」 |

## 第三方服务选择考虑

| 维度 | 内容 |
|------|------|
| **作用** | 选择和管理第三方服务，确保服务质量和成本控制 |
| **应用场景** | 第三方服务选型；服务商评估；服务管理 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的服务商？** | 根据业务需求选择；关注服务商的服务质量、稳定性和技术支持能力；考虑服务成本和长期维护成本 |
| **如何降低依赖风险？** | 建立服务商评估和切换机制；定期评估服务商；准备备用方案 |
| **如何控制服务成本？** | 考虑服务成本和长期维护成本；建立成本监控机制 |

| 要点和思考方向 |
|----------------|
| 第三方服务可以降低开发成本，提高开发效率 |
| 需要根据业务需求选择合适的服务商 |
| 需要建立服务商评估和切换机制，降低依赖风险 |
| 定期评估服务商，准备备用方案 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「服务商选型」「评估」「切换」「成本」「依赖风险」 |
| **方法** | 让 AI 设计选型框架、评估机制、降风险策略 |
| **应用** | 第三方选型、服务管理、成本控制 |
| **提示词范例** | 「第三方服务选型需考虑质量、稳定性、成本，请设计服务商评估框架、切换与备用方案、依赖风险降低策略」 |
