## 刚体

| 维度 | 内容 |
|------|------|
| **作用** | 表现质量、碰撞；模拟物理行为 |
| **应用场景** | 可移动对象；物理交互；受力碰撞 |
| **属性** | 质量；弹性；阻尼；角速度 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **性能** | 限制激活数量；静态碰撞体；物理 LOD；合并小物体 |
| **不稳定** | 调整时间步长；连续碰撞检测；限制速度；固定步长 |
| **质量设置** | 按大小材质设质量；避免差异过大；质量中心 |

| 要点和思考方向 |
|----------------|
| 刚体是物理模拟基础；合理设置属性；注意性能；静态用静态碰撞体 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明对象类型、性能、稳定性；如「大量可交互物体」「穿透问题」「Unity PhysX」 |
| **方法** | 静态用 Static Collider；限制激活数量；固定步长；连续碰撞检测 |
| **应用** | 刚体配置；性能优化；穿透修复 |
| **提示词范例** | 「Unity 场景有 100 个可推动的箱子，物理卡顿，请说明如何用静态碰撞体、物理 LOD、限制激活数量来优化」 |

## 碰撞体（Collider）

**是什么？在哪用？**
| 维度 | 内容 |
|------|------|
| **作用** | 产生物理阻挡、碰撞效果 |
| **应用场景** | 墙壁地面；碰撞检测；物理交互 |

| 类型 | 说明 |
|------|------|
| Box Collider | 盒形，性能好 |
| Sphere Collider | 球形，性能好 |
| Capsule Collider | 胶囊形，适合角色 |
| Mesh Collider | 网格，精确但性能差 |
| Compound Collider | 复合，组合多个简单碰撞体 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **碰撞体选择** | 简单形状用简单碰撞体；复杂用复合；避免 Mesh；按精度选 |
| **碰撞体大小** | 略大于等于模型；测试调整；可视化工具 |

| 要点和思考方向 |
|----------------|
| 简单碰撞体性能好；避免 Mesh，用复合替代 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明模型形状、精度、性能；如「角色」「复杂地形」「Mesh 卡顿」 |
| **方法** | 角色用 Capsule；复杂用 Compound；避免 Mesh Collider |
| **应用** | 碰撞体选型；复合碰撞体 |
| **提示词范例** | 「角色用 Box Collider 会卡墙角，请说明 Capsule 与 Box 的适用场景，以及如何用 Compound Collider 组合多个简单形状」 |

## 触发器（Trigger）

**是什么？在哪用？**
| 维度 | 内容 |
|------|------|
| **作用** | 检测物体接触或交叉 |
| **应用场景** | 触发事件；进入/离开区域；无阻挡检测 |
| **特点** | 不产生物理阻挡，只检测 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **触发时机** | 标志位防重复；进入/离开区分；时间间隔 |
| **性能** | 限制数量；空间分割；及时禁用 |

| 要点和思考方向 |
|----------------|
| 只检测不阻挡；注意重复触发 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明触发场景、重复问题；如「进入区域」「拾取」「多次触发」 |
| **方法** | 标志位防重复；OnTriggerEnter/Exit 区分；Layer 过滤 |
| **应用** | 区域检测；拾取逻辑 |
| **提示词范例** | 「玩家进入触发器区域应只触发一次奖励，但会重复触发，请用 C# 标志位或冷却时间实现防重复，并说明 OnTriggerEnter 与 Stay 的区别」 |

## 碰撞检测模式

**是什么？在哪用？**
| 维度 | 内容 |
|------|------|
| **作用** | 控制精度与性能 |
| **应用场景** | 按速度选模式；平衡性能与精度 |

| 模式 | 优点 | 缺点 | 应用 |
|------|------|------|------|
| Discrete | 性能好 | 快速时可能穿透 | 慢速物体 |
| Continuous | 不穿透 | 性能较低 | 快速物体 |
| Continuous dynamic | 最高精度 | 开销最大 | 快速物体 vs 静态网格 |

**会遇到哪些问题？用什么解决？**
| 问题 | 解决方向 |
|------|----------|
| **快速穿透** | 连续检测；限制速度；提高更新频率；连续动态 |
| **性能与精度** | 快速用连续；慢速用离散；按需选择 |

| 要点和思考方向 |
|----------------|
| 离散性能好可能穿透；连续精度高开销大；按速度选择 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明物体速度、穿透问题；如「子弹」「抛射物」「Discrete 穿透」 |
| **方法** | 快速用 Continuous；慢速用 Discrete；限制速度 |
| **应用** | 碰撞模式选择 |
| **提示词范例** | 「Unity 中 Rigidbody 的 Collision Detection 有 Discrete、Continuous、Continuous Dynamic，请说明三者区别和适用场景，以及快速物体 vs 静态网格应选哪种」 |

| **性能优化** | 空间分割；层次结构；粗检测过滤；减少检测 |

| 要点和思考方向 |
|----------------|
| AABB 常用粗检测；空间分割优化大量物体 |

| 类型 | AI Coding 指南 |

| **应用** | 碰撞优化；自定义物理 |
| **提示词范例** | 「1000 个物体两两碰撞检测性能差，请说明 AABB 粗检测、空间分割（四叉树/八叉树）、BVH 的流程和适用场景」 |

|----------------|

| 类型 | AI Coding 指南 |

| **提示词范例** | 「皮球落地应弹几下后停下，请说明 Bounciness 与阻尼的配合，以及如何避免反弹过高（能量守恒导致）」 |
