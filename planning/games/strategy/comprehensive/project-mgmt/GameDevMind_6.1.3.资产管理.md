## 数字资产

| 维度 | 内容 |
|------|------|
| **作用** | 游戏公司和项目中的重要数字资产，需要系统化管理 |
| **应用场景** | 游戏公司资产管理；游戏项目资产管理；数字资源管理 |

| 资产类型 | 说明 |
|----------|------|
| **版本仓库** | 代码仓库、账户、权限、数据备份 |
| **主机** | 物理服务器或虚拟机资源 |
| **云资源** | 云服务器、云存储、云数据库等云服务资源 |
| **域名** | 游戏服务域名、官网域名等 |
| **IP地址** | 服务器IP、公网IP等 |
| **端口** | 服务端口、数据库端口等 |
| **账号** | 各种服务账号、系统账号 |
| **证书** | SSL证书、安全证书等 |
| **邮箱** | 企业邮箱、服务邮箱等 |
| **AccessToken** | API访问令牌、第三方服务令牌 |
| **软件** | 安装包、授权许可等 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决资产分散问题？** | 建立资产清单和台账；使用管理平台统一管理；建立权限管理体系；定期审计和清理 |
| **如何管理大规模资产？** 资产规模可能从几TB到PB级别 | 根据资产规模选择合适的存储方案；使用分布式存储系统；实现自动化管理 |
| **如何保证资产安全？** | 建立权限控制机制；定期备份重要资产；进行安全审计 |

| 要点和思考方向 |
|----------------|
| 数字资产是游戏公司的重要财产，需要系统化管理 |
| 资产规模可能从几TB到PB级别，需要合适的存储和管理方案 |
| 需要建立资产清单，明确资产归属和使用情况 |
| 使用管理平台统一管理各类资产 |
| 建立权限管理体系，定期审计和清理 |
| 根据资产规模选择合适的存储和管理方案 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「资产清单」「台账」「版本仓库」「主机」「云资源」「域名」「证书」 |
| **方法** | 让 AI 设计资产清单结构、台账模板、管理流程 |
| **应用** | 资产清单、台账、权限体系 |
| **提示词范例** | 「游戏公司需管理数字资产：版本仓库、主机、云资源、域名、IP、账号、证书，请设计资产清单与台账结构、权限管理体系，以及 TB/PB 级资产的存储与管理方案」 |

## 成本管理

| 维度 | 内容 |
|------|------|
| **作用** | 优化资源使用，降低运营成本 |
| **应用场景** | 云资源成本控制；服务器资源优化；软件授权费用管理；运营成本控制；资源使用优化 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决资源浪费问题？** 闲置资源、过度配置 | 定期清理闲置资源；释放未使用的云资源；根据实际需求调整资源配置；使用更经济的方案 |
| **如何解决成本不透明问题？** | 建立成本监控机制；实时监控资源使用情况；分析费用构成；生成成本报告 |
| **如何控制成本持续增长？** | 建立成本控制机制；设置成本预算和预警；定期审计资源使用情况；及时调整资源配置 |

| 要点和思考方向 |
|----------------|
| 成本管理是资产管理的重要组成部分 |
| 需要定期审计资源使用情况，识别和回收无用资源 |
| 优化资源配置，避免过度配置造成的浪费 |
| 建立成本监控机制，及时发现成本异常 |
| 定期清理闲置资源，释放未使用的云资源 |
| 根据实际需求调整资源配置，使用更经济的方案 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「成本」「闲置资源」「成本监控」「预算」「预警」 |
| **方法** | 让 AI 设计成本监控、资源回收、优化策略 |
| **应用** | 成本监控、闲置清理、资源配置优化 |
| **提示词范例** | 「需成本管理：识别闲置资源、控制成本增长，请设计成本监控与预警机制、闲置资源识别与回收流程，以及云资源费用分析与优化策略」 |

## 资产安全

| 维度 | 内容 |
|------|------|
| **作用** | 保护数字资产安全，防止资产丢失、泄露、被攻击 |
| **应用场景** | 所有数字资产都需要安全保护；权限管理；数据备份；网络安全防护；安全审计 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决权限管理混乱问题？** | 建立清晰的权限管理体系；按角色分配权限；定期审计权限；遵循最小权限原则 |
| **如何解决数据丢失问题？** TB到PB级别的资产需要合适的备份方案 | 制定备份策略；定期备份重要资产；根据资产规模制定合适的备份方案；测试备份恢复流程 |
| **如何解决网络安全问题？** | 配置防火墙、安全组；限制访问；监控异常访问；从网络层、应用层、数据层多层防护 |
| **如何进行安全审计？** | 定期进行安全审计；审计权限使用情况；审计访问日志；发现和修复安全问题 |

| 要点和思考方向 |
|----------------|
| 资产安全是资产管理的核心要求 |
| 需要建立完善的权限管理体系，遵循最小权限原则 |
| 备份策略需要根据资产规模制定，TB到PB级别的资产需要合适的备份方案 |
| 网络安全需要从多个层面防护：网络层、应用层、数据层 |
| 定期进行安全审计，发现和修复安全问题 |
| 配置防火墙、安全组，限制访问，监控异常访问 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「权限」「角色」「备份」「防火墙」「安全组」「审计」 |
| **方法** | 让 AI 设计权限体系、备份策略、安全防护 |
| **应用** | 权限管理、备份、网络安全、安全审计 |
| **提示词范例** | 「需资产安全：权限、备份、网络安全，请设计角色与权限体系（最小权限）、TB/PB 级备份策略，以及防火墙/安全组配置与安全审计流程」 |

## 内容管理

| 维度 | 内容 |
|------|------|
| **作用** | 让团队可以简单的、快速的、有效的组织和分享内容（资料）；知识分享；信息共享 |
| **应用场景** | 团队文档管理；知识库；技术文档；项目资料；内容组织和分享 |

| 候选方案 | 说明 |
|----------|------|
| **Drupal** | 功能强大的内容管理框架 |
| **Wiki** | 适合知识库和文档协作 |
| **Wordpress** | 简单易用的内容管理系统 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决文档分散问题？** | 使用内容管理系统（CMS）统一管理；集中存储文档；提供统一访问入口 |
| **如何解决信息孤岛问题？** | 建立知识分享机制；促进团队成员信息共享；建立内容管理规范和流程 |
| **如何解决版本混乱问题？** | 实现版本管理功能；定期整理和归档；保持内容更新 |
| **如何提高内容可查找性？** | 实现内容分类和标签管理；提供强大的搜索功能；确保内容质量和可查找性 |

| 要点和思考方向 |
|----------------|
| Drupal：功能强大的内容管理框架 |
| Wiki：适合知识库和文档协作 |
| Wordpress：简单易用的内容管理系统 |
| 内容管理系统有助于团队知识沉淀和共享 |
| 需要建立内容管理规范和流程，确保内容质量和可查找性 |
| 使用CMS统一管理文档，解决文档分散问题 |
| 建立知识分享机制，促进信息共享 |
| 实现版本管理，定期整理和归档 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「CMS」「Drupal」「Wiki」「Wordpress」「知识库」「文档」 |
| **方法** | 让 AI 设计内容管理方案、知识库结构、协作流程 |
| **应用** | 文档管理、知识库、内容共享 |
| **提示词范例** | 「团队需内容管理：技术文档、项目资料、知识分享，请对比 Drupal/Wiki/Wordpress 的适用场景，设计文档分类与标签、版本管理，以及知识分享流程」 |

## 管理平台

| 维度 | 内容 |
|------|------|
| **作用** | 统一管理各类数字资产，提供可视化的管理界面 |
| **应用场景** | 资产清单管理；权限管理；成本监控；安全审计；资产管理统一视图 |

| 平台选择 | 说明 |
|----------|------|
| **自建管理平台** | 定制化开发，符合特定需求，开发成本高 |
| **第三方管理平台** | 快速部署，功能丰富，维护成本低 |

| 核心功能 | 说明 |
|----------|------|
| **资产查询** | 快速查询资产，多条件搜索，资产详情查看 |
| **权限管理** | 权限分配，权限审计，权限控制 |
| **成本分析** | 成本统计，成本趋势分析，成本预警 |
| **安全审计** | 访问日志，安全事件记录，审计报告 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择管理平台？** | 根据需求选择自建或第三方平台；考虑功能需求、成本、维护等因素；评估平台的扩展性和集成能力 |
| **如何整合各类资产信息？** | 整合各类资产信息；提供统一的管理视图；实现资产关联和数据同步 |
| **如何实现核心功能？** | 实现资产查询、权限管理、成本分析、安全审计功能 |
| **如何保证平台可用性？** | 实现高可用架构；定期数据备份；优化平台性能 |

| 要点和思考方向 |
|----------------|
| 管理平台是资产管理的核心工具 |
| 可以整合各类资产信息，提供统一的管理视图 |
| 支持资产查询、权限管理、成本分析、安全审计等功能 |
| 可以自建或使用第三方管理平台 |
| 根据需求选择合适的平台方案 |
| 实现核心功能，保证平台可用性和性能 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「管理平台」「资产查询」「权限」「成本分析」「安全审计」 |
| **方法** | 让 AI 设计平台功能、数据整合、自建 vs 第三方选型 |
| **应用** | 资产管理平台、统一视图、核心功能 |
| **提示词范例** | 「需资产管理平台：资产查询、权限管理、成本分析、安全审计，请设计平台核心功能与数据整合方案，以及自建与第三方平台的选型评估维度」 |
