## 运行环境

| 维度 | 内容 |
|------|------|
| **作用** | 可在不同运行环境中执行 |
| **环境类型** | Node.js（单线程、异步 I/O）；HTML（浏览器）；Electron、React Native 等 |

| 问题 | 解决方向 |
|------|----------|
| **单线程限制**：不适合 CPU 密集型 | cluster、Worker Threads；委托给其他服务；子进程 |
| **事件循环阻塞** | 避免长时间同步；异步 API；分解耗时操作；setImmediate/nextTick |

| 要点和思考方向 |
|----------------|
| Node.js 适合 I/O 密集型；注意事件循环阻塞；合理利用多进程、Worker Threads |

## 对象

| 维度 | 内容 |
|------|------|
| **赋值** | 引用传递 |
| **复制** | 需深拷贝才能创建副本 |
| **引用** | 多变量可能指向同一对象 |

| 问题 | 解决方向 |
|------|----------|
| **引用传递意外修改** | 深拷贝用 JSON.parse/stringify、lodash.cloneDeep、structuredClone；浅拷贝用 Object.assign、展开运算符 |
| **深拷贝性能** | 避免不必要深拷贝；浅拷贝；Immutable.js；优化算法 |
| **循环引用** | 支持循环引用的库；手动处理；WeakMap 跟踪 |

| 要点和思考方向 |
|----------------|
| 理解浅拷贝与深拷贝；注意性能和循环引用；不可变数据结构 |

## 异步编程

| 维度 | 内容 |
|------|------|
| **Promise** | 异步操作标准方式 |
| **async/await** | 基于 Promise 的语法糖 |
| **async 库** | 异步流程控制工具 |

| 问题 | 解决方向 |
|------|----------|
| **回调地狱** | Promise、async/await、async 库；提取命名函数；链式调用 |
| **Promise 错误处理** | .catch()；try-catch；全局错误处理；不忽略错误 |
| **Promise 未等待** | async 中用 await；Promise.all 并行；linter 检查 |
| **竞态条件** | Promise.race；AbortController；状态标志；锁 |

| 要点和思考方向 |
|----------------|
| 避免回调地狱；妥善处理错误；注意执行顺序和竞态条件 |

## 函数

| 维度 | 内容 |
|------|------|
| **作用** | 基本代码组织单元 |
| **特点** | 对象参数（避免顺序问题）；引用传递；函数是一等公民；闭包 |

| 问题 | 解决方向 |
|------|----------|
| **this 绑定** | 箭头函数；.bind()；self/that 保存引用；理解调用方式 |
| **闭包内存泄漏** | 及时释放；避免持有大对象；WeakMap；显式清理 |
| **参数验证** | 函数开始验证；TypeScript；JSDoc；默认参数 |

| 要点和思考方向 |
|----------------|
| 理解作用域和 this；对象参数提高可读性；注意闭包泄漏；箭头函数与普通函数区别 |

## 内存管理

| 维度 | 内容 |
|------|------|
| **作用** | 自动垃圾回收 |
| **特点** | 引用计数、标记清除；V8 分代回收 |

| 问题 | 解决方向 |
|------|----------|
| **内存泄漏** | 变量设为 null；关闭资源；避免循环引用；WeakMap；流式处理；清理事件监听器、定时器；慎用 global.gc() |
| **内存使用过多** | 对象池；避免大量临时对象；流式处理；限制缓存；定期清理 |
| **GC 暂停** | 减少分配；避免大对象；对象池；优化数据结构 |

| 要点和思考方向 |
|----------------|
| 及时释放资源；避免循环引用；流式处理；定期监控 |

## 优势

| 优势 | 说明 |
|------|------|
| 高性能 | 单线程事件循环，大量并发连接，I/O 表现出色 |
| 轻量级 | 安装包小，适合云部署 |
| JavaScript | 流行语言，开发人员多 |
| 社区支持 | 庞大社区，丰富开源模块 |
| 可伸缩性 | 事件循环便于水平扩展 |

| 要点和思考方向 |
|----------------|
| 适合 I/O 密集型；利用事件循环；丰富 npm 生态；注意单线程限制 |

## 劣势

| 问题 | 解决方向 |
|------|----------|
| **不适合 CPU 密集型** | 高性能语言独立服务；Worker Threads；子进程；cluster |
| **回调嵌套** | 异步流控制；Promise、async/await；async 库；提取命名函数 |
| **内存泄漏** | 参考"内存管理"部分 |
| **缺少标准库** | 成熟第三方库；注意依赖安全；定期更新 |
| **调试困难** | async/await；Node Inspector；详细日志；Sentry |

| 要点和思考方向 |
|----------------|
| 理解适用场景；避免回调地狱；注意内存泄漏；注意依赖管理 |

## JavaScript最佳实践

| 问题 | 解决方向 |
|------|----------|
| **代码质量**：动态类型易产生运行时错误 | TypeScript；ESLint；Prettier；单元测试 |
| **性能优化** | 避免不必要计算；对象池；优化循环递归；Web Workers |
| **安全性** | 验证清理输入；避免 eval、Function；HTTPS；定期更新依赖 |

| 要点和思考方向 |
|----------------|
| TypeScript 提高质量；遵循编码规范；注意性能；关注安全 |
