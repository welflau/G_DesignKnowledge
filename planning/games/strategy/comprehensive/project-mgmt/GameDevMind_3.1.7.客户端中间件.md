## 中间件概览

## GUI

| 维度 | 内容 |
|------|------|
| **作用** | UI界面开发框架 |
| **应用场景** | 所有需要UI的游戏项目、游戏界面开发、UI系统构建 |
| **做什么的？** | UI界面开发框架 |
| **在哪用？** | 所有需要UI的游戏项目 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的UI框架？** 不同UI框架有不同特点和适用场景 | NGUI：功能强大、性能好、需要购买，适合高性能UI项目；UGUI：官方支持、免费、功能完善，适合大多数Unity项目；根据项目需求和预算选择 |
| **UI框架性能优化** UI框架可能影响性能 | 优化UI渲染批次；使用UI图集；实现UI对象池；优化UI更新频率；使用UI合批 |

| 候选 | 说明 |
|------|------|
| **NGUI** | 第三方UI框架，功能强大、性能好、需要购买 |
| **UGUI** | Unity官方UI系统，官方支持、免费、功能完善 |

| 要点和思考方向 |
|----------------|
| 根据项目需求选择合适的UI框架 |
| 考虑UI框架的性能和维护成本 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「UGUI」「NGUI」「FairyGUI」「选型」「性能」 |
| **方法** | 让 AI 对比 UI 框架优缺点；或给出选型建议、迁移方案 |
| **应用** | UI 框架选型、性能评估、项目迁移 |
| **提示词范例** | 「Unity 游戏 UI 框架选型：UGUI、NGUI、FairyGUI，请对比性能、学习成本、生态，给出选型建议」 |

## 资源管理

| 维度 | 内容 |
|------|------|
| **作用** | 游戏资源的管理和加载框架 |
| **应用场景** | 所有需要资源管理的游戏项目、资源加载和释放、资源缓存管理 |
| **做什么的？** | 游戏资源的管理和加载框架 |
| **在哪用？** | 所有需要资源管理的游戏项目 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的资源管理框架？** 不同资源管理框架有不同特点 | LoxodonFramework：支持异步加载、资源池、依赖管理；Unity Addressables：官方支持、功能强大、支持热更新；根据项目需求选择 |
| **资源管理性能优化** 资源管理可能影响性能 | 实现资源预加载；使用资源池管理；优化资源加载流程；实现资源依赖管理；优化资源内存使用 |

| 候选 | 说明 |
|------|------|
| **LoxodonFramework** | 功能完善的资源管理框架，支持异步加载、资源池、依赖管理 |
| **Unity Addressables** | Unity官方资源管理系统，支持热更新 |

| 要点和思考方向 |
|----------------|
| 考虑资源管理框架的功能和性能 |
| 实现资源加载和释放的自动化管理 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「Addressables」「Loxodon」「异步加载」「热更新」 |
| **方法** | 让 AI 对比资源框架、设计加载流程；或生成异步加载封装、资源池实现 |
| **应用** | 资源管理选型、异步加载、热更新 |
| **提示词范例** | 「Unity 资源管理：Addressables 与 LoxodonFramework 对比，请说明异步加载、资源池、热更新支持，以及选型建议」 |

## AI行为

| 维度 | 内容 |
|------|------|
| **作用** | 创建AI行为的工具和框架 |
| **应用场景** | 需要AI控制的角色和NPC的游戏、NPC行为设计、敌人AI、智能角色 |
| **做什么的？** | 创建AI行为的工具和框架 |
| **在哪用？** | 需要AI控制的角色和NPC的游戏 |

| 问题 | 解决方向 |
|------|----------|
| **如何创建复杂的AI行为？** 复杂AI行为需要强大的工具支持 | 使用可视化脚本工具；使用状态机或行为树；实现分层AI系统；使用数据驱动的AI设计；优化AI性能 |
| **如何让非程序员也能创建AI行为？** 需要让策划和美术参与AI设计 | 编辑方式：数据配表、可视化脚本；使用可视化工具（如Playmaker）；提供友好的编辑界面；实现AI行为模板；建立AI设计规范 |

| 编辑方式 | 说明 |
|----------|------|
| **数据配表** | 通过配置表定义AI行为 |
| **可视化脚本** | 通过图形界面创建AI逻辑 |

| 候选 | 说明 |
|------|------|
| **Playmaker** | 多功能插件，基于图形工作流创建AI行为，无需编写代码 |
| **uScript** | 可视化脚本语言，需编写代码，有广泛的内置函数和节点库 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「Playmaker」「行为树」「状态机」「可视化」「策划友好」 |
| **方法** | 让 AI 对比 Playmaker/行为树/状态机；或设计 AI 行为结构、数据配表格式 |
| **应用** | NPC AI、敌人行为、可视化 AI 设计 |
| **提示词范例** | 「游戏 NPC AI 需要策划可配置，请对比 Playmaker 可视化与行为树配表两种方案，以及如何让非程序员参与 AI 设计」 |

## 日志

| 维度 | 内容 |
|------|------|
| **作用** | 日志记录和调试工具 |
| **应用场景** | 开发和调试阶段、问题排查、性能分析、运行时调试 |
| **做什么的？** | 日志记录和调试工具 |
| **在哪用？** | 开发和调试阶段 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的日志工具？** 不同日志工具有不同特点 | ConsolePro：过滤、搜索、保存到文件、日志级别、内存GC；UnityIngameDebugConsole：轻量级、过滤、运行时快捷键；根据项目需求选择 |
| **日志系统设计** 需要设计完善的日志系统 | 实现日志分级（Debug、Info、Warning、Error）；支持日志过滤和搜索；实现日志文件保存；优化日志性能；实现日志上报 |

| 候选 | 特性 |
|------|------|
| **ConsolePro** | 过滤日志、搜索、保存到文件、日志级别、内存垃圾收集器 |
| **UnityIngameDebugConsole** | 轻量级、过滤消息、运行时快捷键打开/关闭控制台 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「ConsolePro」「日志分级」「过滤」「运行时调试」 |
| **方法** | 让 AI 设计日志系统、生成日志封装；或对比 ConsolePro/UnityIngameDebugConsole |
| **应用** | 开发调试、问题排查、日志分级、运行时控制台 |
| **提示词范例** | 「Unity 需要日志系统：分级、过滤、保存到文件、运行时控制台，请对比 ConsolePro 与 UnityIngameDebugConsole，并给出封装建议」 |

## 动画

| 维度 | 内容 |
|------|------|
| **作用** | 动画制作和播放工具 |
| **应用场景** | 需要动画效果的游戏、UI动画、物体运动动画、2D骨骼动画 |
| **做什么的？** | 动画制作和播放工具 |
| **在哪用？** | 需要动画效果的游戏 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的动画工具？** 不同动画工具有不同特点和适用场景 | 曲线动画：HOTween、DOTween（推荐）；2D动画：Spine；根据动画类型选择 |
| **动画性能优化** 动画可能影响性能 | 优化动画计算性能；使用对象池管理动画对象；限制同时播放的动画数量；优化动画更新频率；使用GPU加速 |

| 类型 | 候选 |
|------|------|
| **曲线动画** | HOTween、DOTween |
| **2D动画** | Spine |

| 要点和思考方向 |
|----------------|
| 根据动画类型选择合适的工具 |
| 考虑动画性能和易用性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「DOTween」「Spine」「曲线动画」「2D 骨骼」 |
| **方法** | 让 AI 生成 DOTween 动画代码、Spine 集成示例；或对比 HOTween/DOTween、动画性能优化 |
| **应用** | UI 动画、物体运动、2D 骨骼动画 |
| **提示词范例** | 「Unity 需要 UI 缓动动画和 2D 骨骼动画，请对比 DOTween 与 Spine 的适用场景，并给出 DOTween 序列动画的 C# 示例」 |

## Json

| 维度 | 内容 |
|------|------|
| **作用** | JSON数据序列化和反序列化工具 |
| **应用场景** | 需要处理JSON数据的场景、配置文件读取、网络数据解析、数据存储 |
| **做什么的？** | JSON数据序列化和反序列化工具 |
| **在哪用？** | 需要处理JSON数据的场景 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的JSON库？** 不同JSON库有不同特点和性能 | SimpleJson：轻量级、简单易用、性能好；Newtonsoft.Json：功能完善、支持复杂类型；Unity JsonUtility：官方支持、性能好、功能基础；根据项目需求选择 |
| **JSON性能优化** JSON序列化和反序列化可能影响性能 | 使用高性能JSON库；优化JSON数据结构；使用对象池管理；实现JSON缓存；考虑使用二进制格式 |

| 候选 | 说明 |
|------|------|
| **SimpleJson** | 轻量级JSON库，简单易用、性能好 |
| **Newtonsoft.Json** | 功能强大的JSON库，支持复杂类型 |
| **Unity JsonUtility** | Unity官方JSON工具，功能基础 |

| 要点和思考方向 |
|----------------|
| 根据项目需求选择合适的JSON库 |
| 考虑JSON性能和易用性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「Newtonsoft」「JsonUtility」「SimpleJson」「性能」 |
| **方法** | 让 AI 对比 JSON 库、生成序列化代码；或说明性能差异、复杂类型支持 |
| **应用** | 配置读取、网络解析、数据存储 |
| **提示词范例** | 「Unity JSON 选型：JsonUtility、Newtonsoft.Json、SimpleJson，请对比性能、功能、复杂类型支持，给出配置表和网络数据解析的选型建议」 |
