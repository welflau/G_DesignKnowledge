## 功能开发

| 维度 | 内容 |
|------|------|
| **作用** | 开发运营所需的功能，支持产品运营活动 |
| **应用场景** | 拉新、社交等运营功能开发 |

| 目标市场 | 社交平台举例 |
|----------|--------------|
| **国内** | 微信 |
| **海外** | Facebook |

| 问题 | 解决方向 |
|------|----------|
| **如何实现拉新功能？** | 实现引导分享功能；提供分享奖励；降低分享门槛 |
| **如何实现社交功能？** | 接入社交平台的好友关系链；支持分享到社交平台；根据目标市场选择合适的社交平台 |

| 要点和思考方向 |
|----------------|
| 拉新和社交功能是运营的重要工具 |
| 需要根据目标市场选择合适的社交平台 |
| 分享功能要简单易用，降低用户分享门槛 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「拉新」「分享」「社交」「微信」「Facebook」 |
| **方法** | 让 AI 设计拉新/社交功能、分享流程、平台接入 |
| **应用** | 拉新功能、社交分享、平台适配 |
| **提示词范例** | 「运营需拉新与社交功能，请设计引导分享、分享奖励、好友关系链接入，以及国内（微信）与海外（Facebook）社交平台选型与接入要点」 |

## 渠道接入

| 维度 | 内容 |
|------|------|
| **作用** | 接入各种渠道平台，支持产品上架和分发 |
| **应用场景** | 产品上架、渠道推广 |

| 渠道类型 | 国内 | 海外 |
|----------|------|------|
| **App平台** | 华为、TapTap、应用宝、小米 | GooglePlay、AppStore |
| **公会** | 游戏公会渠道 | — |
| **短视频** | 抖音、快手 | — |
| **媒体** | — | 各种媒体渠道 |

| 问题 | 解决方向 |
|------|----------|
| **如何通过产品过审？** | 提前了解各平台过审要求；产品上架之前要保障达成这些要求 |
| **如何接入国内渠道？** | 接入App平台、公会、短视频等；支持多渠道并行；根据不同渠道特点进行适配 |
| **如何接入海外渠道？** | 接入App平台、媒体等；支持多渠道并行；根据不同渠道特点进行适配 |

| 要点和思考方向 |
|----------------|
| 渠道接入是产品上架的基础工作 |
| 不同渠道有不同的要求，需要提前了解 |
| 产品过审是关键环节，需要确保产品符合各平台要求 |
| 需要支持多渠道并行，提高产品覆盖面 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「渠道接入」「过审」「华为」「TapTap」「GooglePlay」「AppStore」 |
| **方法** | 让 AI 设计渠道接入方案、过审 checklist、多渠道适配 |
| **应用** | 渠道 SDK 接入、过审准备、多渠道出包 |
| **提示词范例** | 「产品需接入国内（华为、TapTap、应用宝、小米）与海外（GooglePlay、AppStore）渠道，请设计多渠道接入方案、各平台过审要求 checklist 与多渠道并行适配要点」 |

## 运营管理

| 维度 | 内容 |
|------|------|
| **作用** | 管理产品运营相关的版本、开发、区服等 |
| **应用场景** | 日常运营管理 |

| 管理维度 | 说明 |
|----------|------|
| **版本** | 版本维度 |
| **分支** | 分支维度 |
| **渠道** | 渠道维度 |

| 版本兼容性考虑 | 问题 |
|----------------|------|
| **客户端包体版本** | 线上是否同时存在多个客户端版本？什么时候需要打开强制更新？ |
| **客户端热更版本** | 热更版本数据是否能兼容线上的各个客户端包体版本？不同渠道的更新是否要同步或有区分？ |
| **服务器版本** | 服务器跟线上不同的客户端包体和热更的版本兼容性 |

| 区服管理功能 | 说明 |
|--------------|------|
| **开服** | 新服务器开启；支持灵活开服 |
| **合服** | 服务器合并；支持灵活合服 |
| **活动** | 区服活动管理 |

| 问题 | 解决方向 |
|------|----------|
| **如何管理版本？** | 建立清晰的版本管理策略；考虑版本兼容性；合理规划功能进入哪个分支、在哪个版本发布 |
| **如何管理开发？** | 建立运营反馈和开发计划的协同机制；提高响应速度 |
| **如何管理区服？** | 支持灵活的开服、合服和活动管理；建立区服管理机制 |

| 要点和思考方向 |
|----------------|
| 版本管理是运营管理的核心，需要建立清晰的版本管理策略 |
| 版本兼容性是要重点考虑的问题，避免版本混乱 |
| 需要建立运营和开发的协同机制，提高响应速度 |
| 区服管理要支持灵活的开服、合服和活动管理 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「版本管理」「版本兼容」「热更兼容」「开服」「合服」「区服活动」 |
| **方法** | 让 AI 设计版本策略、兼容性方案、区服管理 |
| **应用** | 版本与分支管理、客户端/热更/服务端兼容、开服合服 |
| **提示词范例** | 「运营管理需版本（包体、热更、服务端）兼容性、开服/合服/区服活动，请设计版本管理策略、多版本并存与强制更新时机、热更与渠道差异化，以及开服合服流程」 |

## 开发配合事项

| 维度 | 内容 |
|------|------|
| **作用** | 支持产品运营的各种开发事项 |
| **应用场景** | 日常运营支持 |

| 问题 | 解决方向 |
|------|----------|
| **如何对接第三方服务？** | 建立第三方接入规范和流程；统一第三方接入标准 |
| **如何提供快速出包环境？** | 建立自动化出包流程；支持快速出包；快速出包环境可以大大提高运营效率 |
| **如何进行产品维护？** | 建立产品维护机制；产品维护是持续性的工作 |
| **如何更新内容？** | 建立内容更新流程；内容更新是持续性的工作 |
| **如何更新活动？** | 建立活动更新机制；活动更新是持续性的工作 |

| 要点和思考方向 |
|----------------|
| 开发配合事项是运营的基础支撑 |
| 需要建立规范和流程，提高效率 |
| 快速出包环境可以大大提高运营效率 |
| 产品维护、内容更新、活动更新是持续性的工作 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「第三方接入」「快速出包」「产品维护」「内容更新」「活动更新」 |
| **方法** | 让 AI 设计接入规范、出包流水线、更新流程 |
| **应用** | 第三方对接、自动化出包、持续维护 |
| **提示词范例** | 「开发配合需第三方接入规范、快速出包环境、产品维护与内容/活动更新流程，请设计统一接入标准、自动化出包流水线与持续更新机制」 |

## 运营异常分析

| 维度 | 内容 |
|------|------|
| **作用** | 配合运营分析异常数据，定位问题 |
| **应用场景** | 运营数据异常时的排查 |

| 异常类型 | 可能原因 | 排查方向 |
|----------|----------|----------|
| **激活低** | 启动流程、下载安装等 | 配合检查游戏的启动环节，排查启动流程是否有问题 |
| **次留低** | 新手体验、游戏引导等 | 配合检查游戏进入和新手剧情、新手引导等是否有卡住的情况 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决激活低的问题？** | 配合检查游戏的启动环节；建立数据监控机制，及时发现问题 |
| **如何解决次留低的问题？** | 配合检查游戏进入和新手剧情、新手引导等；建立问题排查流程，快速定位和解决问题 |
| **如何建立异常分析机制？** | 建立数据监控机制；建立问题排查流程；开发和运营密切配合 |

| 要点和思考方向 |
|----------------|
| 运营异常分析需要开发和运营密切配合 |
| 激活低通常与启动流程、下载安装等有关 |
| 次留低通常与新手体验、游戏引导等有关 |
| 需要建立数据监控机制，及时发现问题 |
| 需要建立问题排查流程，快速定位和解决问题 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「激活低」「次留低」「启动流程」「新手引导」「数据监控」 |
| **方法** | 让 AI 设计异常分析流程、排查 checklist、监控机制 |
| **应用** | 激活/次留异常排查、问题定位、数据监控 |
| **提示词范例** | 「运营异常分析需排查激活低（启动、下载安装）与次留低（新手体验、引导），请设计数据监控机制、启动与新手指引排查 checklist，以及开发与运营协同的异常分析流程」 |
