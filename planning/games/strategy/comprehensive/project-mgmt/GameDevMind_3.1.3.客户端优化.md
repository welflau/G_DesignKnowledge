## 性能优化

| 维度 | 内容 |
|------|------|
| **作用** | 提升游戏运行性能，确保流畅的游戏体验 |
| **应用场景** | 所有游戏项目、对性能要求高的游戏、移动设备游戏、开放世界游戏 |
| **做什么的？** | 提升游戏运行性能，确保流畅的游戏体验 |
| **在哪用？** | 所有游戏项目，特别是对性能要求高的游戏 |

| 问题 | 解决方向 |
|------|----------|
| **如何确保游戏帧率稳定？** 游戏帧率不稳定会影响游戏体验 | 设定合理的帧率目标（至少20多FPS）；使用Profiler工具分析性能瓶颈；优化CPU和GPU负载；实现帧率监控和报警；优化渲染管线 |
| **如何优化 CPU 和 GPU 性能？** CPU和GPU是性能瓶颈的主要来源 | CPU：减少Draw Call、优化线程、使用合适指令集、优化数学计算、优化程序代码；GPU：减少三角面、优化贴图材质、优化Shader、控制视距和特效、优化渲染管线 |
| **如何做好性能保障和监控？** 需要持续监控性能，及时发现问题 | 在程序设计和实现时考虑性能优化；在数字内容制作前约定好规格；每日构建和测试版本，记录性能指标；使用Profiler工具持续监控；建立性能预算和检查清单 |

| 分类 | 要点 |
|------|------|
| **优化目标** | 至少 20 多 FPS 才能让画面看上去不难受；不同游戏类型对帧率要求不同 |
| **性能保障** | 事后发现问题代价高，做好日常保障；程序设计和实现时考虑性能；数字内容制作前约定规格；每日构建记录性能指标 |
| **CPU 优化** | Draw Call：合并可合并的模型，平衡模型合并和可见性裁剪；线程：检查线程等待、网络独立线程；指令集：使用合适指令集；数学：快速sqrt、化简或预计算；程序：空间换时间、预先算出结果 |
| **Graphics 优化** | 三角面：减少面数、可见性控制、LOD；贴图：平衡尺寸清晰度、mipmap、图集、2的幂、压缩；材质：shader复杂度、半透明数量；视距、特效粒子数；Shader：减少Sampler、减少数学函数、vs优先于ps |
| **IO 优化** | 不在 Update 中频繁操作 IO；检查频繁读写文件、频繁内存分配释放 |

| 要点和思考方向 |
|----------------|
| 至少 20 多 FPS 才能让画面看上去不难受 |
| 事后发现问题、解决问题，代价总是很高的，尽量做好日常保障 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明瓶颈类型、目标；如「CPU」「GPU」「Draw Call」「帧率」「Profiler」 |
| **方法** | 让 AI 分析 Profiler 数据、给出优化建议；或生成优化代码（合并 Draw Call、LOD、对象池等） |
| **应用** | 性能瓶颈定位、优化方案设计、Profiler 解读、渲染管线优化 |
| **提示词范例** | 「Unity Profiler 显示 Rendering 占用 15ms，Draw Call 800+，请给出 Draw Call 合并、批处理、LOD 的优化方案和实现思路」 |

## 网络优化

| 维度 | 内容 |
|------|------|
| **作用** | 优化网络通信性能，减少延迟和流量消耗 |
| **应用场景** | 所有需要网络通信的游戏、在线游戏、多人游戏 |
| **做什么的？** | 优化网络通信性能，减少延迟和流量消耗 |
| **在哪用？** | 所有需要网络通信的游戏 |

| 问题 | 解决方向 |
|------|----------|
| **如何减少网络延迟？** 网络延迟影响游戏体验 | 使用多线程处理网络，避免让UI卡顿；优化网络消息大小；使用消息压缩；优化网络协议；使用CDN加速 |
| **如何减少网络流量消耗？** 网络流量消耗影响用户体验和运营成本 | 使用消息压缩；减少不必要的网络请求；使用增量更新；优化消息格式；实现消息缓存 |
| **如何选择消息压缩方案？** 不同压缩方案有不同优缺点 | Protobuf：压缩更高效，开发效率稍低；JSON+gzip：开发便利，压缩和执行效率相对较低；根据项目需求选择 |
| **网络异常处理** 网络异常需要妥善处理 | 实现网络重连机制；处理网络超时；实现消息队列；优化网络错误提示 |

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Protobuf** | 压缩更高效 | 开发效率稍低 |
| **JSON+gzip** | 开发很便利 | 压缩效率和执行效率相对较低 |

| 要点和思考方向 |
|----------------|
| 多线程：在独立线程中工作，避免让 UI 卡顿 |
| 考虑网络异常处理 |
| 优化网络协议和消息格式 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「延迟」「流量」「压缩」「消息格式」「多线程」 |
| **方法** | 让 AI 对比 Protobuf/JSON+gzip、设计消息压缩方案；或生成网络线程封装、重连逻辑 |
| **应用** | 网络协议设计、消息压缩选型、网络异常处理、流量优化 |
| **提示词范例** | 「游戏网络消息需要压缩，请对比 Protobuf 和 JSON+gzip 的优缺点，并给出 C# 下 Protobuf 序列化/反序列化的示例代码」 |

## 加载优化

| 维度 | 内容 |
|------|------|
| **作用** | 提高数据加载速度，让玩家尽快开始体验游戏 |
| **应用场景** | 游戏启动、场景切换、资源加载、关卡加载 |
| **做什么的？** | 提高数据加载速度，让玩家尽快开始体验游戏 |
| **在哪用？** | 游戏启动、场景切换、资源加载等场景 |

| 问题 | 解决方向 |
|------|----------|
| **如何提高加载速度？** 加载速度慢影响游戏体验 | 使用数据序列化，加载到内存直接使用数据；实现异步加载；优化资源格式；使用资源压缩；实现资源预加载 |
| **如何优化加载体验？** 加载过程需要给用户良好体验 | 实现加载进度显示；使用延迟后台加载，非关键资源在后台异步加载；实现加载动画；优化加载界面；实现加载提示 |
| **如何管理加载资源？** 大量资源需要管理 | 实现资源加载优先级；实现资源依赖管理；实现资源缓存机制；优化资源加载流程；实现资源预加载策略 |
| **加载性能优化** 加载可能影响游戏性能 | 使用多线程加载；优化IO操作；实现加载限流；优化资源解压；使用流式加载 |

| 要点和思考方向 |
|----------------|
| 数据序列化：加载到内存直接使用数据，不需要再对数据进行解析、组织等操作 |
| 延迟后台加载：非关键资源可以在后台异步加载 |
| 考虑加载体验优化 |
| 实现资源加载管理 |
| 优化加载性能 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「启动加载」「场景切换」「序列化」「异步」「进度」 |
| **方法** | 让 AI 设计加载流程、生成异步加载代码；或设计二进制序列化格式、资源优先级队列 |
| **应用** | 启动优化、场景加载、资源预加载、加载进度 UI |
| **提示词范例** | 「Unity 场景切换加载慢，请给出异步加载场景、显示加载进度、延迟后台加载非关键资源的实现方案和 C# 代码示例」 |

## 内存优化

| 维度 | 内容 |
|------|------|
| **作用** | 优化内存使用，提高设备适配性；更少的内存使用将获得更大的设备适配性 |
| **应用场景** | 所有游戏项目、移动设备游戏、内存受限的设备 |
| **做什么的？** | 优化内存使用，提高设备适配性 |
| **在哪用？** | 所有游戏项目，特别是移动设备游戏 |

| 问题 | 解决方向 |
|------|----------|
| **如何做好资源内存预算？** 资源占用大量内存，需要合理规划 | 举例：贴图、模型、音效、游戏对象；方法：在游戏或系统设计前期，程序、美术共同对各种资源的使用量做个初步规划；建立资源内存预算表；定期检查；优化资源内存占用 |
| **如何做好系统内存预算？** 系统内存需要合理分配 | 尽量为各个系统预先分配好内存；建立系统内存预算表；优化系统内存使用；实现内存池管理；定期检查系统内存使用 |
| **如何做好内存监控？** 需要持续监控内存使用情况 | 在开发过程中监控内存变化；前期可能超预算，后期寻找可优化内容；使用Profiler工具持续记录；实现内存监控系统；建立内存报警机制 |
| **内存泄漏检测** 内存泄漏会导致内存持续增长 | 使用内存分析工具检测泄漏；实现对象引用计数；检查资源释放；定期进行内存泄漏测试；建立内存泄漏检查流程 |
| **内存碎片优化** 内存碎片影响内存使用效率 | 使用内存池减少碎片；优化内存分配策略；实现内存整理；使用对象池管理；优化资源加载和释放 |

| 要点和思考方向 |
|----------------|
| 做好资源内存预算：贴图、模型、音效、游戏对象；程序美术共同规划 |
| 做好系统内存预算：尽量为各个系统预先分配好内存 |
| 做好内存监控：开发过程中监控变化，前期可能超预算后期优化，使用 Profiler 持续记录 |
| 考虑内存泄漏检测 |
| 优化内存碎片 |
| 建立内存优化策略 |

| 类型 | AI Coding 指南 |
