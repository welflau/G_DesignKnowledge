## 内容合规

| 维度 | 内容 |
|------|------|
| **作用** | 确保产品内容符合各地区法律法规和政策要求 |
| **应用场景** | 产品在不同地区运营时的内容审核和合规 |

| 内容类型 | 审核范围 |
|----------|----------|
| **文本** | 聊天内容、游戏内文本等 |
| **图片** | 游戏画面、UI图片等 |
| **音频** | 游戏音效、语音等 |
| **视频** | 游戏视频、宣传视频等 |

| 地区 | 合规要求 |
|------|----------|
| **中国大陆** | 聊天敏感词过滤；规范游戏内名词使用；画面（如血的表现）需符合要求 |
| **美国** | 政治正确；避免种族歧视等政治不正确的内容 |

| 敏感内容审核范围 | 涉黄/涉政/暴恐/违禁/辱骂/广告/未成年/仇恨言论/个人信息保护/私下交易等 |

| 问题 | 解决方向 |
|------|----------|
| **如何处理不同类型的内容？** | 建立不同类型内容的审核机制；使用内容审核工具和服务 |
| **如何满足不同地区的合规要求？** | 针对不同地区建立相应的合规策略；使用地区特定的内容审核规则 |
| **如何审核敏感内容？** | 建立完善的敏感内容审核机制；敏感词过滤、内容审核是重要的合规手段 |

| 要点和思考方向 |
|----------------|
| 内容合规是产品运营的基础要求，必须严格遵守 |
| 不同地区有不同的合规要求，需要针对性地处理 |
| 敏感词过滤、内容审核是重要的合规手段 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「内容审核」「敏感词」「涉黄涉政」「文本」「图片」「音频」 |
| **方法** | 让 AI 设计审核机制、敏感词库、地区规则 |
| **应用** | 内容审核、敏感词过滤、多地区合规 |
| **提示词范例** | 「内容合规需审核文本/图片/音频/视频，请设计敏感词过滤、涉黄涉政暴恐等审核机制，以及中国大陆与美国等地区的差异化合规策略」 |

## 地方性政策

| 维度 | 内容 |
|------|------|
| **作用** | 了解并遵守各地区法律法规和政策要求，确保产品合规 |
| **应用场景** | 产品在不同地区运营；合规规划；合规检查 |

| 地区 | 主要政策 |
|------|----------|
| **欧盟（GDPR）** | Terrorist Content Regulation：用户数据需要存储在当地；需遵守数据保护法规 |
| **马来西亚** | 《通信和多媒体法案》 |
| **法国** | 《阿维亚法》 |
| **美国** | 《儿童网上隐私保护法》、《儿童在线保护法》、《儿童互联网保护法》、《通信内容端正法》；加州（CCPA）：加州消费者隐私法案 |
| **德国** | 《网络执行法》 |
| **澳大利亚** | Violent Material Act |
| **日本** | 《青少年网络环境整治法》 |
| **新加坡** | 《防止网络虚假和操纵法案》、《广播法》、《远程赌博法》、《计算机滥用和网络安全方案》、《新闻法》、《内部安全法》、《煽动法》、《网络行为法》、《不良出版物法》 |

| 问题 | 解决方向 |
|------|----------|
| **如何应对不同地区的政策要求？** | 了解并遵守各地区相关政策；建立合规检查清单 |
| **如何应对GDPR要求？** | 遵守GDPR数据保护法规；用户数据需要存储在当地 |
| **如何应对儿童保护要求？** | 遵守各地区儿童保护法规；建立儿童保护机制 |

| 要点和思考方向 |
|----------------|
| 不同地区的政策要求差异很大，需要深入了解 |
| GDPR对数据保护要求严格，需要特别注意 |
| 儿童保护是很多地区的重点，需要严格遵守 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「GDPR」「CCPA」「儿童保护」「数据本地化」 |
| **方法** | 让 AI 设计合规 checklist、数据保护方案、政策适配 |
| **应用** | 多地区政策适配、数据保护、儿童保护 |
| **提示词范例** | 「产品多地区运营需应对 GDPR、CCPA、儿童保护等政策，请设计各地区合规检查清单、数据本地化与存储要求、儿童保护机制」 |

| 要点和思考方向 |
|----------------|
| 中国大陆的合规要求比较严格，需要提前准备 |

| 实名认证是游戏运营的基本要求 |

| 类型 | AI Coding 指南 |

| **方法** | 让 AI 设计合规流程、备案 checklist |
| **应用** | 中国大陆合规、备案与认证 |
| **提示词范例** | 「中国大陆合规需等保认证、实名认证、公安备案、ICP 备案，请设计合规流程与各项手续完成 checklist」 |

## 平台合规

| 维度 | 内容 |
|------|------|
| **作用** | 确保产品符合各应用商店和平台的合规要求 |
| **应用场景** | 产品上架应用商店时的合规审核 |

| 平台 | 审核要求 |
|------|----------|
| **苹果** | App Store审核要求 |
| **谷歌** | Google Play审核要求 |
| **任天堂** | trc/tcr/lotcheck审核要求 |

| 问题 | 解决方向 |
|------|----------|
| **如何满足不同平台的审核要求？** | 了解各平台的审核要求；不同平台需分别准备 |
| **如何准备隐私条款？** | 提供完善的隐私政策；确保隐私政策符合各平台要求 |
| **如何满足技术合规要求？** | 需要支持IPv6等技术合规要求 |
| **如何确保内容合规？** | 内容需符合平台要求；建立平台合规检查清单 |

| 要点和思考方向 |
|----------------|
| 平台合规是产品上架的前提条件 |
| 不同平台有不同的审核要求，需要分别准备 |
| 隐私条款是重要的合规内容，需要认真准备 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「App Store」「Google Play」「隐私条款」「IPv6」 |
| **方法** | 让 AI 设计平台审核 checklist、隐私政策、技术合规 |
| **应用** | 应用商店上架、隐私政策、技术合规 |
| **提示词范例** | 「平台合规需满足 App Store、Google Play 等审核要求，请设计各平台审核 checklist、隐私政策要点与 IPv6 等技术合规」 |

| 要点和思考方向 |
|----------------|
| 合规服务商可以提供专业的合规服务，降低合规成本 |
| 需要根据业务需求选择合适的服务商 |

| 类型 | AI Coding 指南 |
|------|----------------|

| **方法** | 让 AI 设计服务商选型、接入方案 |

## 产品合规策略

| 维度 | 内容 |
|------|------|
| **作用** | 制定和实施产品合规策略，确保产品持续合规 |
| **应用场景** | 产品合规规划；合规体系建设；合规管理 |

| 合规体系包含 | 内容审核、数据保护、隐私保护等 |

| 问题 | 解决方向 |
|------|----------|
| **如何建立合规体系？** | 建立完善的合规体系；建立完善的合规管理制度 |
| **如何应对不同地区的合规要求？** | 针对不同地区建立相应的合规策略 |
| **如何建立合规检查机制？** | 建立合规检查机制；定期检查合规情况；建立合规检查清单 |
| **如何建立应急响应机制？** | 建立应急响应机制；建立合规问题处理流程 |
| **如何持续更新合规策略？** | 合规是持续性的工作；关注法规变化；及时更新合规策略 |

| 要点和思考方向 |
|----------------|
| 产品合规是产品运营的基础，必须高度重视 |
| 需要建立完善的合规体系，包括内容审核、数据保护、隐私保护等 |
| 不同地区有不同的合规要求，需要针对性地处理 |
| 合规是持续性的工作，需要持续关注和更新 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「合规体系」「内容审核」「数据保护」「隐私保护」「检查机制」「应急响应」 |
| **方法** | 让 AI 设计合规体系、检查清单、应急流程 |
| **应用** | 合规体系建设、持续合规管理 |
| **提示词范例** | 「产品合规策略需建立合规体系（内容审核、数据保护、隐私保护）、合规检查机制、应急响应、法规跟踪，请设计合规管理制度与持续更新流程」 |
