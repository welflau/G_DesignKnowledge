## 帐号系统

| 维度 | 内容 |
|------|------|
| **作用** | 管理用户账号，建立用户与产品的关联 |
| **应用场景** | 用户注册、登录、身份验证 |

| 登录方式 | 说明 |
|----------|------|
| **账号登录** | 用户名、邮箱、手机（短信验证）登录 |
| **游客登录** | 临时账号，方便快速体验；需引导绑定账号 |
| **第三方登录** | 国内：微信；海外：Facebook、Google |

| 问题 | 解决方向 |
|------|----------|
| **如何降低账号系统开发成本？** | 使用第三方账号服务；缩短开发周期；快速获得稳定的账号系统 |
| **如何满足实名认证要求？** | 接入实名认证服务；国内运营需要实名认证，需提前规划 |
| **如何实现账号功能？** | 实现注册、登录、密码找回等完整功能；支持多种登录方式；第三方登录可降低注册门槛 |

| 要点和思考方向 |
|----------------|
| 账号系统是产品的基础功能，需要稳定可靠 |
| 支持多种登录方式可以提升用户体验 |
| 第三方登录可以降低注册门槛，提高转化率 |
| 国内运营需要实名认证，需要提前规划 |
| 游客登录可以降低体验门槛，但需要引导绑定账号 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「账号登录」「游客」「第三方登录」「微信」「Facebook」「实名认证」 |
| **方法** | 让 AI 设计账号体系、登录流程、第三方接入 |
| **应用** | 注册登录、多种登录方式、实名认证 |
| **提示词范例** | 「账号系统需支持账号/游客/第三方登录（国内微信、海外 Facebook/Google）、实名认证，请设计登录流程、游客绑定引导与第三方 SDK 接入要点」 |

## 账号管理后台

| 维度 | 内容 |
|------|------|
| **作用** | 提供账号管理功能，支持运营人员查询和管理账号 |
| **应用场景** | 账号查询、问题排查、订单补单 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现账号查询功能？** | 后台提供完备的查询帐号功能；支持多种查询方式（账号、手机、邮箱等），快速定位账号 |
| **如何实现补单功能？** | 实现补单功能，处理支付异常；建立补单审核机制 |
| **如何实现日志记录？** | 记录关键操作日志；日志记录要详细，包括登录、支付等关键操作 |
| **如何实现账号管理功能？** | 支持账号封禁、解封等管理功能 |

| 要点和思考方向 |
|----------------|
| 管理后台是运营的重要工具，需要功能完善 |
| 账号查询功能要支持多种查询方式（账号、手机、邮箱等） |
| 补单功能是支付异常处理的重要工具 |
| 日志记录要详细，包括登录、支付等关键操作 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「账号查询」「补单」「操作日志」「封禁」 |
| **方法** | 让 AI 设计后台功能、查询与补单流程、日志结构 |
| **应用** | 账号查询、补单、操作审计 |
| **提示词范例** | 「账号管理后台需账号查询（账号/手机/邮箱）、补单、操作日志、封禁解封，请设计查询接口、补单审核流程与日志记录结构」 |

## 支付系统

| 维度 | 内容 |
|------|------|
| **作用** | 处理用户支付，实现产品变现 |
| **应用场景** | 游戏内购、道具购买、会员订阅等 |

| 市场 | 支付方式 |
|------|----------|
| **国内** | 应用商店内购、微信支付、支付宝 |
| **海外** | 应用商店内购（App Store、Google Play）、信用卡、MyCard |

| 问题 | 解决方向 |
|------|----------|
| **如何降低支付接入成本？** | 使用第三方支付服务；使用支付聚合服务，统一接入多个支付渠道 |
| **如何选择支付方式？** | 根据目标市场选择合适的支付方式；提供便捷的支付体验 |
| **如何考虑支付通道费用？** | 综合考虑成本和用户体验；支付通道费用是重要成本，需合理选择 |
| **如何处理支付异常？** | 处理掉单、重复支付、支付超时、支付失败；建立支付监控和告警机制 |

| 要点和思考方向 |
|----------------|
| 支付系统是产品变现的关键，需要稳定可靠 |
| 需要支持多种支付方式，满足不同用户需求 |
| 支付通道费用是重要成本，需要合理选择 |
| 需要处理支付异常情况，如掉单、重复支付等 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「内购」「微信支付」「支付宝」「信用卡」「掉单」「重复支付」 |
| **方法** | 让 AI 设计支付流程、异常处理、监控告警 |
| **应用** | 支付接入、异常处理、通道费用 |
| **提示词范例** | 「支付系统需支持国内（应用商店、微信、支付宝）与海外（App Store、Google Play、信用卡）支付，请设计支付流程、掉单/重复支付/超时处理与监控告警机制」 |

| 类型 | AI Coding 指南 |

## 账号与支付的安全考虑

| 维度 | 内容 |
|------|------|
| **作用** | 确保账号和支付系统的安全性，保护用户隐私和资金安全 |
| **应用场景** | 账号系统安全；支付系统安全；数据安全保护 |

| 问题 | 解决方向 |
|------|----------|
| **如何保护用户数据安全？** | 采用加密传输（HTTPS/TLS）；保护用户隐私和资金安全 |
| **如何建立风控机制？** | 建立完善的风控机制；实现异常检测和风险控制 |
| **如何进行安全审计？** | 定期进行安全审计；及时发现和修复漏洞 |
| **如何建立应急响应机制？** | 建立应急响应机制；建立安全事件处理流程 |
| **如何确保合规？** | 遵守相关法律法规；保护用户权益；建立合规检查机制 |

| 要点和思考方向 |
|----------------|
| 账号和支付涉及用户隐私和资金安全，需要高度重视 |
| 需要采用加密传输，保护用户数据安全 |
| 需要建立完善的风控机制，防范欺诈和盗刷 |
| 需要定期进行安全审计，及时发现和修复漏洞 |
| 需要建立应急响应机制，快速处理安全问题 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「HTTPS」「风控」「安全审计」「应急响应」「合规」 |
| **方法** | 让 AI 设计安全架构、风控机制、合规框架 |
| **应用** | 数据加密、风控、审计与应急 |
| **提示词范例** | 「账号与支付安全需加密传输（HTTPS/TLS）、风控机制、安全审计、应急响应、合规，请设计安全架构、异常检测与风险控制、审计与应急流程」 |
