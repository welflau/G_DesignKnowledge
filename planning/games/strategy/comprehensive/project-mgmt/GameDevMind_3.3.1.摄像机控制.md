## 旋转轴

| 旋转轴 | 应用 | 限制/注意 |
|--------|------|-----------|
| **Pitch（俯仰角）** | 控制视角的上下角度 | 通常有上下角度限制，避免视角翻转 |
| **Yaw（偏航角）** | 控制视角的左右角度 | 可 360 度旋转，或限制在特定范围内 |
| **Roll（翻滚角）** | 很少使用，主要用于特殊效果（飞机翻滚、眩晕效果） | 在角色摄像机中很少使用 |

| 问题 | 解决方向 |
|------|----------|
| **镜头移动不流畅** 镜头移动时出现抖动、卡顿 | 使用平滑插值（Lerp、Slerp）；添加加速度和阻尼；使用缓动函数（Easing）优化运动曲线 |
| **镜头操作体验差** 操作手感不好，响应不灵敏或过于敏感 | 调整镜头灵敏度，适配不同输入设备；添加加速度和阻力；支持灵敏度设置让玩家自定义 |
| **视角翻转问题** 镜头旋转时可能出现视角翻转导致眩晕 | 限制 Pitch 角度范围（如 -90 度到 90 度）；使用平滑过渡；添加视角约束 |

| 要点和思考方向 |
|----------------|
| 第一人称时：基于角色自身的轴来转动，镜头与角色绑定 |
| 第三人称时：以角色为 Look at 的目标点来转动，镜头围绕角色旋转 |
| 操作镜头运动时，让镜头有一定的加速度和阻力，提升镜头运动体验 |
| 使用平滑插值和缓动函数，实现流畅的镜头移动 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明平台、需求；如「Lerp」「Slerp」「Pitch」「Yaw」「阻尼」「缓动」 |
| **方法** | 让 AI 设计平滑插值、角度限制、灵敏度与阻尼 |
| **应用** | 镜头平滑、视角限制、操作手感 |
| **提示词范例** | 「Unity 第三人称镜头需平滑跟随，请给出 Lerp/Slerp 的 C# 实现，以及 Pitch 上下角度限制、防止视角翻转的代码」 |

## 镜头振动

| 维度 | 内容 |
|------|------|
| **作用** | 实现镜头震动效果，增强游戏表现力 |
| **应用场景** | 爆炸、撞击等冲击效果；角色受伤、死亡等状态变化；环境震动（地震、爆炸等） |

| 问题 | 解决方向 |
|------|----------|
| **震动效果不自然** 震动效果过于生硬，不符合物理规律 | 使用衰减函数让震动逐渐减弱；添加随机性；根据事件强度调整震动幅度和频率 |
| **震动影响操作** 震动过于强烈，影响玩家操作 | 控制震动幅度；提供震动开关；在关键时刻（如瞄准）减少或关闭震动 |

| 要点和思考方向 |
|----------------|
| 镜头震动要符合物理规律，使用衰减和随机性 |
| 震动幅度要适中，避免影响玩家操作 |
| 根据游戏类型和场景调整震动效果 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、强度；如「震动」「衰减」「随机」「Perlin」 |
| **方法** | 让 AI 设计震动曲线、衰减、强度与频率控制 |
| **应用** | 爆炸/撞击震动、受伤/死亡震动 |
| **提示词范例** | 「Unity 需要镜头震动：爆炸时强震动、逐渐衰减，请给出基于 Perlin 噪声或正弦波的 C# 震动实现，支持幅度和持续时间参数」 |

## 约束

| 维度 | 内容 |
|------|------|
| **作用** | 将摄像机的运动限定在指定空间中，防止"穿帮"、避免不好的视角 |
| **应用场景** | 限制摄像机移动范围；防止摄像机穿透场景；避免不合理的视角 |

| 问题 | 解决方向 |
|------|----------|
| **约束范围设置困难** 如何设置合适的约束范围 | 使用碰撞体或触发器定义约束区域；根据场景和游戏类型调整；提供可视化工具方便调试 |
| **约束与自由度的平衡** 约束过严影响自由度，约束过松容易穿帮 | 根据游戏类型和场景需求调整；使用软约束（轻微超出会回弹）；关键区域用硬约束 |

| 要点和思考方向 |
|----------------|
| 约束要防止"穿帮"、避免不好的视角 |
| 使用碰撞检测和范围限制实现约束 |
| 平衡约束和自由度，提供良好的游戏体验 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明约束类型；如「碰撞」「范围」「软约束」「硬约束」 |
| **方法** | 让 AI 设计约束区域、碰撞检测、回弹逻辑 |
| **应用** | 防止穿帮、限制视角范围 |
| **提示词范例** | 「第三人称镜头需防止穿透墙壁，请给出 Unity 中从角色向摄像机方向做 Raycast 检测、发生穿透时平滑拉近摄像机的 C# 实现」 |

## 分屏/画中画

| 维度 | 内容 |
|------|------|
| **作用** | 支持多个摄像机同时渲染，实现分屏或画中画效果 |
| **应用场景** | 多人同屏游戏、赛车后视镜、监控画面、小地图等 |

| 问题 | 解决方向 |
|------|----------|
| **性能问题** 每个分屏摄像机都要走一遍渲染管线，性能开销大 | 优化不同屏幕中的内容；降低分屏摄像机分辨率；使用 LOD、遮挡剔除；限制分屏数量 |
| **画面同步问题** 多个摄像机画面不同步 | 确保所有摄像机使用相同的更新逻辑；使用统一的渲染时机；处理网络延迟（多人游戏） |

| 要点和思考方向 |
|----------------|
| 每个分屏摄像机都要走一遍渲染管线，要考虑不同屏幕中的内容的优化 |
| 合理使用分屏，避免性能问题 |
| 确保多个摄像机画面同步 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明用途、数量；如「分屏」「多摄像机」「后视镜」「小地图」 |
| **方法** | 让 AI 设计多摄像机布局、Viewport、性能优化 |
| **应用** | 多人同屏、后视镜、小地图 |
| **提示词范例** | 「Unity 双人同屏需两个摄像机分屏，请说明 Viewport Rect 配置、每个摄像机渲染范围，以及如何降低分屏摄像机的分辨率以优化性能」 |

## 第一人称视角

| 维度 | 内容 |
|------|------|
| **作用** | 以角色视角观察游戏世界，提供沉浸式的游戏体验 |
| **应用场景** | FPS 游戏、部分 RPG 游戏、探索类游戏、VR 游戏 |

| 问题 | 解决方向 |
|------|----------|
| **镜头与角色绑定** 如何让镜头完美跟随角色移动和旋转 | 将摄像机作为角色的子对象；使用平滑插值避免抖动；处理角色移动时的镜头补偿（如头部摆动） |
| **视野范围设置** FOV 设置不当影响游戏体验 | 根据游戏类型设置 FOV（FPS 通常 60-90 度）；支持 FOV 动态调整（如瞄准时缩小）；考虑不同屏幕比例 |
| **武器/手部模型处理** 武器和手部模型的显示和动画 | 使用独立的摄像机层渲染武器模型；处理武器动画和镜头移动的同步；优化武器模型渲染性能 |
| **眩晕问题** 部分玩家可能产生眩晕 | 提供视野范围调整选项；添加视野稳定功能；提供第三人称视角选项（如适合） |

| 要点和思考方向 |
|----------------|
| 相对简单，跟角色绑定，跟着角色动就行 |
| 注意镜头与角色移动的同步，避免抖动 |
| 合理设置 FOV，提供良好的游戏体验 |
| 考虑眩晕问题，提供可调节选项 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「第一人称」「FOV」「武器层」「眩晕」 |
| **方法** | 让 AI 设计 FPS 摄像机、武器分层渲染、FOV 动态调整 |
| **应用** | FPS 镜头、武器显示、FOV 设置 |
| **提示词范例** | 「Unity FPS 需武器在独立层渲染、瞄准时缩小 FOV，请给出摄像机层级、Culling Mask 配置，以及 FOV 平滑过渡的 C# 实现」 |

## 镜头位置

| 维度 | 内容 |
|------|------|
| **作用** | 设置摄像机相对于角色的位置，影响视角效果 |
| **应用场景** | 所有第三人称视角游戏 |

| 问题 | 解决方向 |
|------|----------|
| **镜头高度设置** 镜头高度不合适影响视角和游戏体验 | 根据游戏类型设置合适的镜头高度；支持动态调整（如跳跃时）；提供镜头高度设置选项 |
| **镜头距离设置** 镜头距离不合适，视野过窄或过宽 | 根据游戏类型设置合适的镜头距离；支持动态调整（如瞄准时拉近）；考虑不同场景的需求 |

| 要点和思考方向 |
|----------------|
| 镜头高度和距离要根据游戏类型和场景调整 |
| 支持动态调整，适应不同游戏状态 |
| 提供玩家自定义选项 |

## 镜头失序（穿透场景）

| 维度 | 内容 |
|------|------|
| **作用** | 防止摄像机穿透场景或角色，避免看到不该看到的内容 |
| **应用场景** | 所有第三人称视角游戏 |

| 问题 | 解决方向 |
|------|----------|
| **穿透检测** 摄像机会穿透场景或其它角色 | 限制镜头的运动范围（上下、左右、前后）；摄像机碰撞检测，发生穿插后平滑调整位置；距离限制，距离太近时角色透明化；使用碰撞体防止穿透 |
| **平滑移动** 摄像机位置调整时出现突然跳跃 | 使用平滑插值（Lerp）移动摄像机；添加缓动函数；控制移动速度 |

| 要点和思考方向 |
|----------------|
| 使用碰撞检测和范围限制防止穿透 |
| 摄像机位置调整要平滑，避免突然跳跃 |
| 距离过近时可以考虑角色透明化 |

## 遮挡处理

| 维度 | 内容 |
|------|------|
| **作用** | 处理角色和摄像机之间被场景物体遮挡的问题 |
| **应用场景** | 所有第三人称视角游戏 |

| 问题 | 解决方向 |
|------|----------|
| **遮挡检测** 角色和摄像机之间可能被场景物体遮挡 | 射线检测：从摄像机向角色发射射线检测遮挡物；遮挡物半透明；调整摄像机位置；角色高亮 |
| **半透明效果** 遮挡物半透明后视觉效果不自然 | 使用合适的透明度（通常 30-50%）；添加淡入淡出效果；只对特定物体使用半透明（如墙壁、柱子） |

| 要点和思考方向 |
|----------------|
| 使用射线检测识别遮挡物 |
| 遮挡物半透明是常用解决方案 |
| 调整摄像机位置也可以避免遮挡 |
| 注意半透明效果的视觉质量 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「第三人称」「LookAt」「穿透」「遮挡」「半透明」 |
| **方法** | 让 AI 设计跟随逻辑、穿透检测、遮挡半透明 |
| **应用** | 第三人称镜头、穿透处理、遮挡处理 |
| **提示词范例** | 「Unity 第三人称：摄像机到角色被墙遮挡时，墙需半透明，请给出 Raycast 检测、动态修改材质透明度的 C# 实现，以及穿透时拉近摄像机的逻辑」 |

## 镜头跟随

| 维度 | 内容 |
|------|------|
| **作用** | 实现摄像机平滑跟随角色移动，提供流畅的视角体验 |
| **应用场景** | 所有第三人称视角游戏 |

| 问题 | 解决方向 |
|------|----------|
| **跟随延迟** 摄像机跟随过于僵硬或延迟 | 使用平滑插值（Lerp、Slerp）；添加惯性和阻尼；根据角色移动速度调整跟随速度 |
| **快速移动时的跟随** 角色快速移动时摄像机跟不上或过于敏感 | 考虑摄像机随运动的惯性和阻尼；根据移动速度动态调整跟随参数；使用预测算法 |
| **眩晕问题** 镜头跟随可能导致玩家眩晕 | 减少不必要的镜头晃动；使用稳定的跟随算法；提供镜头稳定选项 |

| 要点和思考方向 |
|----------------|
| 要考虑摄像机随着运动的惯性和阻尼 |
| 不能让用户有眩晕感 |
| 使用平滑插值和缓动函数，实现流畅跟随 |
| 根据游戏类型调整跟随参数 |

## 多目标跟随

| 维度 | 内容 |
|------|------|
| **作用** | 多人游戏时，让多个目标保持"同框"，确保所有玩家都在视野内 |
| **应用场景** | 多人同屏游戏、合作游戏、竞技游戏 |

| 问题 | 解决方向 |
|------|----------|
| **视野范围计算** 如何计算合适的视野范围包含所有目标 | 计算所有目标的边界框；根据边界框调整摄像机位置和 FOV；添加边距避免目标在屏幕边缘 |
| **目标距离过远** 无法同时包含在视野内 | 限制摄像机距离；使用分屏或画中画；动态调整 FOV（如合适） |

| 要点和思考方向 |
|----------------|
| 多人游戏时，要让多个目标保持"同框" |
| 动态调整摄像机位置和 FOV |
| 考虑目标距离过远的情况 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明目标数量、场景；如「多目标」「同框」「边界框」「FOV 动态」 |
