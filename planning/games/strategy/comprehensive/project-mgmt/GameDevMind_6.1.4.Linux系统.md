<h2 align="center">Linux系统</h2>
<p>

## 常用指令

| 维度 | 内容 |
|------|------|
| **作用** | Linux系统操作的基础指令，用于系统管理和维护 |
| **应用场景** | 远程服务器管理；系统维护；文件操作；进程监控；软件安装 |

| 常用指令 | 说明 |
|----------|------|
| **ssh** | 连接进入主机，对系统或游戏后端进行维护 |
| **sudo** | 用管理权限执行指令 |
| **scp** | 复制文件（包括远程复制），用于往主机上传、部署数据 |
| **vi** | 文本编辑工具 |
| **cron** | 设置定时任务，如定时重启服务、定时清理日志 |
| **top** | 查看进程和资源使用情况 |
| **apt-get** | 安装软件（Ubuntu/Debian系统） |

| 问题 | 解决方向 |
|------|----------|
| **如何解决ssh安全问题？** 默认端口22容易被扫描攻击 | 修改SSH访问端口；配置防火墙规则；禁用密码认证，使用密钥认证；限制SSH访问IP |
| **如何实现ssh免密访问？** | 生成SSH密钥对（ssh-keygen）；将公钥配置到服务器（~/.ssh/authorized_keys）；配置SSH客户端使用私钥 |
| **如何解决权限问题？** | 使用sudo以管理权限执行指令；配置sudo权限（/etc/sudoers）；遵循最小权限原则 |
| **如何使用cron定时任务？** | 编辑crontab（crontab -e）；设置定时任务规则；测试定时任务执行 |
| **如何使用top监控系统？** | 使用top查看进程和资源使用情况；使用htop等增强工具；分析系统性能瓶颈 |

| 要点和思考方向 |
|----------------|
| 这些指令是Linux系统操作的基础，需要熟练掌握 |
| ssh是远程管理服务器的核心工具，安全配置很重要 |
| cron定时任务可以自动化很多运维工作 |
| top可以实时监控系统资源使用情况 |
| 掌握常用指令的基本用法和参数 |
| 注意ssh安全配置，使用密钥认证 |
| 合理使用sudo，遵循最小权限原则 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「ssh」「scp」「cron」「top」「免密」「密钥」 |
| **方法** | 让 AI 生成配置、脚本、安全加固方案 |
| **应用** | SSH 配置、定时任务、监控脚本 |
| **提示词范例** | 「需配置 Linux 服务器：SSH 免密、修改 22 端口、cron 定时重启服务，请给出 ssh-keygen 与 authorized_keys 配置步骤、crontab 示例，以及 top/htop 监控脚本」 |

## 目录结构

| 维度 | 内容 |
|------|------|
| **作用** | Linux系统的目录结构，用于组织和管理系统文件 |
| **应用场景** | 系统配置管理；软件安装管理；日志和缓存管理；系统维护 |

| 重要目录 | 说明 |
|----------|------|
| **/etc** | 系统级别的配置文件，如网络配置、用户账户、系统服务等 |
| **/opt** | 第三方软件包安装目录，如/opt/mysql、/opt/java |
| **/var** | 可变数据，如日志文件、邮件队列、缓存文件等，需定期监视和管理 |

| 问题 | 解决方向 |
|------|----------|
| **如何管理/etc配置文件？** | 修改前需要备份；使用版本控制管理配置；测试配置修改效果；记录配置修改历史 |
| **如何管理/opt第三方软件？** | 在/opt目录安装第三方软件；按软件包组织目录结构；便于管理和维护 |
| **如何管理/var可变数据？** | 定期清理日志和缓存文件；配置日志轮转机制；监控磁盘使用情况；设置磁盘使用告警 |

| 要点和思考方向 |
|----------------|
| 了解Linux目录结构有助于理解系统组织方式 |
| /etc目录存放配置文件，修改前需要备份 |
| /opt目录适合安装第三方软件，便于管理 |
| /var目录需要定期清理，避免占用过多磁盘空间 |
| 理解各目录的作用和使用场景 |
| 建立配置文件备份和版本控制机制 |
| 定期清理/var目录，防止磁盘空间不足 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「/etc」「/opt」「/var」「目录」「配置」「日志轮转」 |
| **方法** | 让 AI 设计目录规范、备份流程、清理脚本 |
| **应用** | 配置管理、软件安装、日志管理 |
| **提示词范例** | 「Linux 需管理 /etc 配置、/opt 第三方软件、/var 日志，请设计配置备份与版本控制流程、日志轮转（logrotate）配置，以及磁盘使用告警脚本」 |

## 网络使用与问题排查

| 维度 | 内容 |
|------|------|
| **作用** | 网络工具用于测试网络连通性、排查网络问题、测试接口 |
| **应用场景** | 网络连通性测试；网络问题排查；DNS解析查询；HTTP接口测试；网络配置管理 |

| 常用工具 | 说明 |
|----------|------|
| **ping** | 测试域名/IP是否通、速度、是否有丢包 |
| **traceroute** | 追踪数据包从源到目的地的路径 |
| **ifconfig** | 查看本机网络配置 |
| **nslookup** | 查询DNS解析信息 |
| **dig** | 从DNS域名服务器查询主机地址信息 |
| **curl** | 下载文件、测试HTTP接口 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决网络连通性问题？** | 使用ping测试连通性；使用traceroute追踪路径；分析网络延迟和丢包；检查防火墙和路由配置 |
| **如何解决DNS解析问题？** | 使用nslookup或dig查询DNS；检查DNS服务器配置；验证DNS解析结果 |
| **如何解决网络配置问题？** | 使用ifconfig查看和配置网络接口；检查网络配置文件；验证IP地址和子网掩码 |
| **如何测试HTTP接口？** | 使用curl测试HTTP接口；测试GET、POST、PUT、DELETE等请求；查看响应状态码和内容 |

| 要点和思考方向 |
|----------------|
| 网络排查是运维中的常见工作，需要掌握这些工具 |
| ping是最基础的网络测试工具 |
| traceroute可以定位网络路径中的问题节点 |
| curl不仅可以下载文件，还可以测试HTTP接口，非常实用 |
| 掌握网络工具的基本用法和参数 |
| 按照从简单到复杂的顺序排查网络问题 |
| 结合多个工具综合分析网络问题 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「ping」「traceroute」「dig」「curl」「DNS」「网络排查」 |
| **方法** | 让 AI 生成排查脚本、测试命令、诊断流程 |
| **应用** | 网络连通性、DNS 解析、HTTP 接口测试 |
| **提示词范例** | 「需排查网络：连通性、DNS、HTTP 接口，请给出 ping/traceroute/dig/curl 的排查脚本，以及从简单到复杂的网络问题诊断流程」 |

## 文件管理与操作

| 维度 | 内容 |
|------|------|
| **作用** | 文件管理工具用于文件操作、文件关联、文本查找 |
| **应用场景** | 文件移动和重命名；文件关联和链接；日志文件查找；文本内容搜索 |

| 常用工具 | 说明 |
|----------|------|
| **link（ln）** | 关联文件，在目录中关联另一位置的文件/目录，避免复制 |
| **mv** | 移动文件 |
| **grep** | 文本查找，在日志文件中找信息 |

| 链接类型 | 说明 |
|----------|------|
| **符号链接（软链接）** | 指向文件路径；可跨文件系统；删除源文件后失效 |
| **硬链接** | 指向文件inode；不能跨文件系统；删除源文件后仍有效 |

| 问题 | 解决方向 |
|------|----------|
| **如何解决文件同步问题？** | 使用ln创建符号链接或硬链接；根据需求选择合适的链接类型 |
| **如何使用mv移动文件？** | 使用mv移动文件或目录；使用mv重命名文件 |
| **如何解决日志查找问题？** | 使用grep进行文本搜索；支持正则表达式和多文件搜索；结合find、awk等命令使用 |

| 要点和思考方向 |
|----------------|
| link可以创建文件关联，避免重复存储 |
| mv可以移动或重命名文件 |
| grep是日志分析的重要工具，支持正则表达式，功能强大 |
| 理解符号链接和硬链接的区别和使用场景 |
| 掌握grep的正则表达式用法 |
| 结合多个命令提高文件操作效率 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「ln」「符号链接」「硬链接」「grep」「mv」「日志查找」 |
| **方法** | 让 AI 生成 ln/grep 命令、日志分析脚本 |
| **应用** | 文件关联、日志搜索、批量操作 |
| **提示词范例** | 「需在日志中查找错误、创建软件多版本符号链接，请给出 grep 正则示例、ln 软链接/硬链接使用场景，以及 find+grep 组合的日志分析脚本」 |

## 环境变量

| 维度 | 内容 |
|------|------|
| **作用** | 配置系统环境、应用程序运行环境 |
| **应用场景** | 设置PATH；设置JAVA_HOME等开发环境变量；设置数据库连接等应用环境变量 |

| 常用环境变量 | 说明 |
|--------------|------|
| **PATH** | 系统命令搜索路径 |
| **JAVA_HOME** | Java开发环境路径 |
| **数据库连接** | 数据库相关环境变量 |

| 问题 | 解决方向 |
|------|----------|
| **如何设置环境变量？** 用户级别：~/.bashrc；系统级别：/etc/profile | 使用export命令设置；编辑配置文件实现永久设置 |
| **如何查看环境变量？** | echo $VARIABLE查看单个；env/printenv查看所有 |
| **如何修改环境变量？** | 临时：export VARIABLE=new_value；永久：编辑配置文件后source使生效 |
| **如何处理环境变量冲突？** 用户级别优先于系统级别；后加载覆盖先加载 | 理解优先级；检查配置文件加载顺序 |

| 要点和思考方向 |
|----------------|
| 环境变量影响系统和应用程序的行为 |
| 需要了解如何设置、查看、修改环境变量 |
| 环境变量可以在用户级别或系统级别设置 |
| 理解用户级别和系统级别环境变量的区别 |
| 掌握环境变量的设置、查看、修改方法 |
| 注意环境变量的优先级和冲突问题 |
| 使用source命令使配置文件生效 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「PATH」「JAVA_HOME」「环境变量」「.bashrc」「/etc/profile」 |

| **应用** | 环境配置、多版本切换、应用部署 |
| **提示词范例** | 「需配置 Java、数据库连接等环境变量，请给出 ~/.bashrc 与 /etc/profile 的配置示例、用户级与系统级优先级说明，以及 source 生效与冲突处理要点」 |
