## 汇编语言

| 维度 | 内容 |
|------|------|
| **作用** | 直接操作 CPU 和内存，对硬件完全控制 |
| **应用场景** | 运算量大的极致优化；C++ 项目；系统编程、驱动、嵌入式 |
| **知识点** | 寄存器（CPU 高速存储）；指令（基本操作）；寻址（访问内存方式） |

| 问题 | 解决方向 |
|------|----------|
| **学习曲线陡峭** | 从简单指令开始；结合 C/C++ 理解；编译器生成参考；必要时使用 |
| **平台依赖性强** | 内联汇编；多平台实现；条件编译；优先高级语言优化 |
| **维护困难** | 详细注释；关键路径使用；内联汇编；提供高级语言参考 |

| 要点和思考方向 |
|----------------|
| 用于性能关键路径极致优化；结合高级语言使用 |

## LUA

| 维度 | 内容 |
|------|------|
| **作用** | 轻量级、可嵌入脚本语言，适合游戏逻辑 |
| **应用场景** | 热更脚本；游戏逻辑、配置；AI 脚本；配置解析 |

### 对象

| 维度 | 内容 |
|------|------|
| **作用** | 用表（table）实现面向对象 |
| **: 与 . 区别** | `:` 自动传 self，`.` 不传；`obj:method()` 等价 `obj.method(obj)` |

| 问题 | 解决方向 |
|------|----------|
| **: 和 . 混淆** | 对象方法用 `:`，静态用 `.`；定义时对应语法 |
| **性能问题** | 关键路径用原生代码；LuaJIT；C/C++ 绑定 |
| **调试困难** | ZeroBrane Studio；日志；断点；详细错误信息 |

### 表

| 维度 | 内容 |
|------|------|
| **作用** | 唯一数据结构，表示数组、字典、对象 |
| **应用场景** | 所有需数据结构的地方 |

| 问题 | 解决方向 |
|------|----------|
| **表性能** | 预分配大小；避免频繁操作；数组部分优于哈希部分 |
| **索引混淆** | 数组从 1 开始；C/C++ 交互时索引转换；统一 0 或 1 |

### 协程

| 维度 | 内容 |
|------|------|
| **作用** | 协作式多任务，实现异步逻辑 |
| **应用场景** | 状态机；异步逻辑；生成器模式 |

| 问题 | 解决方向 |
|------|----------|
| **生命周期管理** | 明确创建销毁时机；协程管理器；避免泄漏 |

| 要点和思考方向 |
|----------------|
| 合理配置 JVM；利用 Spring、Hibernate 等生态；理解 JVM 原理 |

## GO

**是什么？在哪用？**
- **作用**：Google开发的系统编程语言，专注于简洁、高效、并发
- **应用场景**：
  - 服务端开发
  - 高性能后端服务、微服务架构、云原生应用
  - 网络服务、API服务
  - 命令行工具开发

### 特性

**是什么？在哪用？**
- **作用**：Go语言的核心特性，提供强大的并发和内存管理能力
- **应用场景**：所有Go程序

### 并发编程

| 维度 | 内容 |
|------|------|
| **作用** | 轻量级并发模型 |
| **组件** | goroutine（轻量级线程）；channel（通信同步）；条件变量；同步锁 |

| 问题 | 解决方向 |
|------|----------|
| **goroutine 泄漏** | context 控制生命周期；确保正常退出；sync.WaitGroup；监控数量 |
| **channel 阻塞** | 带缓冲 channel；select 处理多 channel；超时机制；避免死锁 |

| 要点和思考方向 |
|----------------|
| "通过通信共享内存"；注意 goroutine 泄漏和 channel 阻塞 |

### GC（垃圾回收）

| 维度 | 内容 |
|------|------|
| **作用** | 自动内存管理，减少泄漏风险 |
| **应用场景** | 所有 Go 程序 |

| 问题 | 解决方向 |
|------|----------|
| **GC 延迟**（STW） | 控制对象创建；对象池；GOGC 参数；sync.Pool |
| **内存使用过多** | 减少分配；值类型；及时释放；pprof 分析 |

| 要点和思考方向 |
|----------------|
| 对象池和 sync.Pool；监控 GC 性能 |

### 优势

**是什么？在哪用？**
- **作用**：Go语言的优势使其适合某些特定的应用场景
- **应用场景**：高并发服务、微服务、云原生应用

| 优势 | 说明 |
|------|------|
| 并发性能 | goroutine 轻量，可创建大量并发 |
| 编译速度 | 快速编译 |
| 部署简单 | 单一可执行文件 |
| 类型安全 | 静态类型，编译时检查 |
| 简洁语法 | 易学易用 |

| 要点和思考方向 |
|----------------|
| 适合高并发服务端；编译快、部署简单；适合微服务和云原生 |
