## 复杂度

## 时间复杂度

| 维度 | 内容 |
|------|------|
| **作用** | 衡量算法执行时间随输入规模增长的变化趋势 |
| **应用场景** | 算法设计与选择；性能评估与优化；系统设计权衡 |
| **应用** | 根据需求权衡时间换空间或空间换时间 |

## 应用举例

| 场景 | 方案 | 权衡 |
|------|------|------|
| 预计算 vs 实时计算 | 有限数列且性能要求高时，预存到数组 | 用空间换时间 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **复杂度分析不准确** | 考虑常数因子；实际测试；分析最坏/平均/最好；考虑数据特征 |
| **空间复杂度忽略** | 同时分析时空；按资源限制选算法；考虑内存受限 |

| 要点和思考方向 |
|----------------|
| 复杂度帮助选算法；时空权衡；理论需实际验证；注意常数因子 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数据规模、最坏情况、内存限制；如「100 万玩家排序」「移动端内存紧张」 |
| **方法** | 要求分析最坏/平均；提供时空权衡；考虑常数因子和缓存 |
| **应用** | 分析算法复杂度；选合适算法；优化空间使用 |
| **提示词范例** | 「这段 O(n²) 的排序在 10000 条数据时卡顿，请改为 O(n log n) 算法并说明选择理由，C++」 |

## 排序

| 维度 | 内容 |
|------|------|
| **作用** | 将数据按规则重新排列 |
| **应用场景** | 数据展示（排行榜）；优化查找；预处理；算法辅助 |

## 方法

| 算法 | 特点 | 复杂度 |
|------|------|--------|
| 冒泡排序 | 简单，效率低 | O(n²) |
| 选择排序 | 每次选最小/最大 | O(n²) |
| 插入排序 | 适合部分有序 | O(n²)，最好 O(n) |
| 快速排序 | 平均性能好，广泛使用 | 平均 O(n log n)，最坏 O(n²) |
| 归并排序 | 稳定，适合大规模 | O(n log n) |
| 堆排序 | 使用堆结构 | O(n log n) |
| 计数排序 | 适合小范围整数 | O(n+k) |

## 应用举例

| 场景 | 需求/方案 |
|------|-----------|
| 游戏排行榜 | 对玩家分数排序，常用快排、归并 |
| 碰撞检测 | 按位置排序后只检测相邻物体 |
| 搜索算法 | 启发式搜索需对队列排序 |
| AI 决策 | 按评估值排序后选最高决策 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **算法选择不当** | 按规模选；小规模用插入；大规模用快排/归并；考虑部分有序 |
| **稳定性要求** | 归并/插入；比较函数加原始位置；稳定实现 |
| **性能问题** | 高效算法；增量排序；并行；标准库 |

| 要点和思考方向 |
|----------------|
| 按规模和特征选算法；注意稳定性；小规模可能简单算法更快 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数据规模、有序性、稳定性需求；如「排行榜 Top 100」「需稳定排序」「部分有序」 |
| **方法** | 小规模考虑插入；大规模用快排/归并；稳定性要求选归并 |
| **应用** | 选排序算法；实现稳定排序；优化增量排序 |
| **提示词范例** | 「游戏排行榜需对 1000 个玩家按分数排序，分数相同时按时间先后，请用稳定排序实现，C#」 |

## 查找

| 维度 | 内容 |
|------|------|
| **作用** | 在数据集合中定位特定元素 |

## 方法

| 方法 | 特点 | 复杂度 |
|------|------|--------|
| 二分查找 | 有序数组 | O(log n) |
| 散列表查找 | 哈希表 | 平均 O(1) |
| 树结构查找 | 二叉搜索树等 | O(log n) |
| 线性查找 | 顺序遍历 | O(n) |

## 应用举例

| 场景 | 说明 |
|------|------|
| 玩家匹配 | 快速查找匹配玩家 |
| 资源管理 | 查找游戏资源 |
| 缓存系统 | 查找缓存 |
| 配置查找 | 查找配置项 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **查找效率低** | 二分（有序）；哈希表；树结构；建立索引 |
| **数据需排序** | 哈希表；自平衡树；增量排序 |
| **内存开销** | 小规模用线性；紧凑结构；权衡内存与性能 |

| 要点和思考方向 |
|----------------|
| 按数据特征选方法；有序用二分；考虑维护成本 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数据特征、查找频率、键类型；如「有序配置表」「按 ID 查找」「需范围查询」 |
| **方法** | 有序用二分；无序用哈希；范围查询用树 |
| **应用** | 选查找方法；实现二分查找；设计索引结构 |
| **提示词范例** | 「配置表按 ID 有序，需频繁查找，请用二分查找实现 GetConfig(id)，并处理 ID 不存在的情况」 |

## 深度优先搜索（DFS）

| 维度 | 内容 |
|------|------|
| **作用** | 沿深度遍历，尽可能深地搜索分支 |
| **应用场景** | 探索所有路径；回溯；拓扑排序；连通性检测 |
| **特点** | 栈（递归）实现；可能非最短路径 |

| 问题 | 解决方向 |
|------|----------|
| **栈溢出** | 迭代实现；限制深度；显式栈 |
| **非最短路径** | 需最短路径用 BFS；深度限制；IDS |

| 要点和思考方向 |
|----------------|
| 适合探索所有路径；注意栈溢出；适合回溯 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明图规模、目标、深度；如「状态空间搜索」「需遍历所有状态」「深度可能 20+」 |
| **方法** | 深度大要求迭代+栈；回溯用 DFS；提供剪枝策略 |
| **应用** | 将递归 DFS 改为迭代；设计回溯剪枝；生成遍历代码 |
| **提示词范例** | 「用 DFS 遍历技能树所有节点收集可学技能，树深度约 5 层，请用迭代+栈避免递归栈溢出」 |

## 广度优先搜索（BFS）

| 维度 | 内容 |
|------|------|
| **作用** | 沿宽度遍历树的节点 |
| **应用场景** | 最短路径；层次遍历；连通性检测 |
| **特点** | 队列实现；能找到最短路径 |

| 问题 | 解决方向 |
|------|----------|
| **内存开销** | 双向 BFS；限制范围；IDBFS |
| **性能问题** | A*；并行 BFS；优化队列 |

| 要点和思考方向 |
|----------------|
| 适合最短路径；注意内存；适合无权图 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明图类型、路径需求、规模；如「网格寻路」「需最短路径」「100x100」 |
| **方法** | 无权图用 BFS；有权图用 A* 或 Dijkstra；大图考虑双向 BFS |
| **应用** | 实现 BFS 寻路；优化队列；生成 A* 变体 |
| **提示词范例** | 「网格地图 50x50，障碍物用 0 表示，请用 BFS 实现从起点到终点的最短路径，返回路径坐标列表」 |

## 填充算法（Hole-Filling）

| 维度 | 内容 |
|------|------|
| **作用** | 从内点开始，由内向外填充至边界 |
| **应用场景** | 图像处理；游戏机制；地图生成；区域标记 |

## 算法

| 算法 | 说明 |
|------|------|
| Lazy flood fill | 延迟填充，优化性能 |
| 洪水填充 | 从起点向四周扩散 |
| 边界填充 | 沿边界填充 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 消除游戏 | 消除同色区域 |
| 寻路 | 标记可达区域 |
| 地图生成 | 生成连续地形 |
| 画笔功能 | 填充工具 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **性能问题** | Lazy flood fill；限制范围；扫描线；分块 |
| **栈溢出** | 迭代实现；队列替代递归；限制深度 |
| **边界检测** | 精确算法；预处理；抗锯齿 |

| 要点和思考方向 |
|----------------|
| 注意性能；迭代避免栈溢出；注意边界检测 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明填充规模、性能要求、边界；如「消除游戏 10x10」「每帧可能多次」「需抗锯齿」 |
| **方法** | 大区域用 Lazy flood fill；迭代+队列避免栈溢出；提供扫描线优化 |
| **应用** | 实现洪水填充；优化填充性能；处理边界情况 |
| **提示词范例** | 「消除游戏需填充相连同色块，网格 8x8，请用迭代+队列实现 flood fill，避免递归栈溢出」 |

## 分治法

| 维度 | 内容 |
|------|------|
| **作用** | 分解为子问题，递归解决后合并 |
| **应用场景** | 可分解为相似子问题；解可合并；适合并行 |
| **应用** | 快排、归并、大整数乘法 |

| 问题 | 解决方向 |
|------|----------|
| **子问题重叠** | 动态规划；缓存子问题；合并相似子问题 |
| **合并复杂度** | 优化合并算法；简化逻辑；并行合并 |

| 要点和思考方向 |
|----------------|
| 注意子问题重叠；优化合并过程 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明问题可分解性、子问题关系、并行需求；如「归并排序」「大数组分块处理」 |
| **方法** | 识别子问题重叠则用 DP；优化合并；提供并行分治 |
| **应用** | 设计分治策略；优化合并；生成分治代码 |
| **提示词范例** | 「将这段 O(n²) 的最近点对暴力算法改为分治法 O(n log n)，并说明合并步骤」 |

## 动态规划法

| **应用场景** | 最优子结构；子问题重叠；需最优解 |

| **空间复杂度高** | 滚动数组；只保存必要状态；记忆化搜索 |
| **边界条件** | 仔细定义初始状态；边界检查；测试验证 |

| 要点和思考方向 |
|----------------|
| 注意空间优化；仔细定义状态转移；注意边界 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明问题类型、状态定义、空间限制；如「背包问题」「最长公共子序列」「O(n) 空间」 |
| **方法** | 要求定义状态和转移；滚动数组优化；提供边界条件 |
| **应用** | 设计 DP 状态；实现空间优化；生成 DP 解法 |
| **提示词范例** | 「0-1 背包，n 个物品，容量 W，请用 DP 实现，并用滚动数组将空间优化到 O(W)」 |

| 要点和思考方向 |
|----------------|
| 需证明正确性；不一定全局最优；适合快速近似 |

| 类型 | AI Coding 指南 |
|------|----------------|

| **方法** | 要求分析贪心性质；提供反例验证；说明与 DP 关系 |
