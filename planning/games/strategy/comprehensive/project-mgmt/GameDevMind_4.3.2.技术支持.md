## 分布式编译

| 维度 | 内容 |
|------|------|
| **作用** | 通过分布式编译来提高代码编译速度，缩短开发反馈周期 |
| **应用场景** | 大型项目编译、多平台项目编译、提高编译效率 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现分布式编译** 需要提高编译速度 | 将编译任务分发到多台机器并行处理；对大型 C++/多平台项目尤为关键；选择合适的工具；配置分布式编译环境 |

| 候选工具 | 特点 |
|----------|------|
| **IncrediBuild** | 商业分布式编译工具；支持多种构建系统；提高编译速度 |
| **yadcc** | 开源分布式编译工具；支持多种编译器；提高编译效率 |

| 要点和思考方向 |
|----------------|
| 分布式编译是提高开发效率的重要手段 |
| 对大型项目和多平台项目尤为重要 |
| 选择合适的工具和配置 |
| 优化编译性能减少等待时间 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「分布式编译」「IncrediBuild」「yadcc」「并行」「C++」 |
| **方法** | 让 AI 对比工具、设计配置、给出接入方案 |
| **应用** | 分布式编译选型、环境配置、构建集成 |
| **提示词范例** | 「C++ 游戏项目编译慢，需分布式编译，请对比 IncrediBuild 与 yadcc 的适用场景、配置复杂度，以及与 Visual Studio/MSBuild 的集成步骤」 |

## 代码辅助工具

| 维度 | 内容 |
|------|------|
| **作用** | 通过工具自动生成或批量修改代码，减少机械性工作与手误 |
| **应用场景** | 代码生成、批量修改、减少重复工作 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现代码辅助** 需要减少机械性工作和手误 | 为 Method 批量生成注入代码（日志、监控、权限校验等）；批量修改类成员的作用域；选择合适的工具；实现代码生成规则 |

| 典型能力 | 说明 |
|----------|------|
| **代码生成** | 为 Method 批量生成注入代码；批量修改类成员的作用域 |
| **候选** | NRefactory 等代码分析/生成框架 |

| 要点和思考方向 |
|----------------|
| 代码辅助工具是提高开发效率的重要手段 |
| 减少机械性工作和手误 |
| 选择合适的工具和框架 |
| 建立代码生成规范 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「代码生成」「批量注入」「日志」「监控」「NRefactory」「Roslyn」 |
| **方法** | 让 AI 设计代码生成规则、AST 分析、注入逻辑 |
| **应用** | 代码生成、批量修改、AOP 注入 |
| **提示词范例** | 「需为 C# Method 批量生成日志、监控、权限校验等注入代码，请用 Roslyn 或 NRefactory 设计代码分析/生成流程，以及批量修改类成员作用域的实现思路」 |

## 技术选型辅助

| 维度 | 内容 |
|------|------|
| **作用** | 在关键技术选型上提供调研、验证与决策支持 |
| **应用场景** | 技术选型、技术决策、技术评估 |

| 问题 | 解决方向 |
|------|----------|
| **如何进行技术选型** 需要在关键技术选型上做出正确决策 | 建立技术选型流程；实现技术评估方法；做出正确决策 |

| 常见议题 | 说明 |
|----------|------|
| **引擎选择** | 自研/Unity/Unreal 等 |
| **UI 中间件选型** | 评估 UI 中间件、功能需求、性能需求 |
| **植被实现方案** | SpeedTree、自研植被系统等 |

| 要点和思考方向 |
|----------------|
| 技术选型是项目成功的关键 |
| 需要充分的调研和验证 |
| 综合考虑特性、需求、成本和风险 |
| 建立技术选型流程和规范 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「引擎」「UI 中间件」「植被」「选型」「评估」 |
| **方法** | 让 AI 做技术调研、对比分析、给出选型建议 |
| **应用** | 技术选型、方案评估、决策支持 |
| **提示词范例** | 「项目需技术选型：引擎（自研/Unity/Unreal）、UI 中间件、植被方案（SpeedTree/自研），请给出评估维度、对比表及选型决策流程建议」 |

## 技术支援

| 维度 | 内容 |
|------|------|
| **作用** | 为项目组在特定技术难点上提供支持 |
| **应用场景** | 技术难点攻关、技术问题解决、技术方案实现 |

| 问题 | 解决方向 |
|------|----------|
| **如何提供技术支援** 需要解决项目中的技术难点 | 识别技术难点；提供技术方案；实现技术方案；优化技术方案 |

| 举例 | 说明 |
|------|------|
| **算法实现** | 寻路、匹配、推荐等 |
| **性能优化方案** | 渲染优化、内存优化等 |

| 要点和思考方向 |
|----------------|
| 技术支援是项目推进的重要保障 |
| 需要快速识别和解决技术难点 |
| 提供可行的技术方案 |
| 持续优化技术方案 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「寻路」「匹配」「推荐」「渲染优化」「内存优化」「算法」 |
| **方法** | 让 AI 设计算法、实现方案、优化策略 |
| **应用** | 算法实现、性能优化、技术方案 |
| **提示词范例** | 「游戏需实现寻路/匹配/推荐算法，请设计 C# 实现方案与接口，以及渲染/内存优化的常见策略与排查流程」 |

## 技术美术（TA）

| 维度 | 内容 |
|------|------|
| **作用** | 连接程序与美术，在「效果与性能」之间寻找平衡 |
| **应用场景** | 视觉效果实现、性能优化、美术与程序协作 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现技术美术工作** 需要在效果和性能之间找到平衡 | 建立技术美术工作流程；实现效果和性能平衡；优化技术美术方案 |

| 典型工作方向 | 说明 |
|--------------|------|
| **光照** | 光照方案选型与烘焙流程（静态光照、实时光照、混合光照） |
| **材质** | 毛皮、布料等特殊材质的实现与优化 |
| **效果** | 卡通渲染等风格化渲染方案 |
| **性能与效果评估与平衡** | 为不同平台/档位制定表现与性能的平衡策略 |

| 要点和思考方向 |
|----------------|
| 技术美术是连接程序和美术的桥梁 |
| 需要在效果和性能之间找到平衡 |
| 为不同平台制定合适的策略 |
| 持续优化技术美术方案 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「光照」「材质」「卡通渲染」「性能」「效果平衡」 |
| **方法** | 让 AI 设计光照方案、材质实现、性能与效果平衡策略 |
| **应用** | 光照烘焙、特殊材质、风格化渲染、平台档位策略 |
| **提示词范例** | 「TA 需实现：光照方案（静态/实时/混合）、毛皮/布料材质、卡通渲染，请给出 Unity/Unreal 的实现要点及不同平台档位的效果与性能平衡策略」 |

## DevOps 搭建

| 维度 | 内容 |
|------|------|
| **作用** | 为项目搭建完整的开发运维（DevOps）体系，让开发、测试、发布尽可能自动化与可观测 |
| **应用场景** | 开发流程自动化、测试流程自动化、发布流程自动化、运维监控 |

| 问题 | 解决方向 |
|------|----------|
| **如何搭建 DevOps 体系** 需要搭建完整的 DevOps 体系 | 设计 DevOps 架构；实现自动化流程；实现监控系统；实现可观测性 |
| **如何实现自动化** 需要实现各种自动化流程 | 测试自动化、出包自动化、热更自动化；将自动化流程沉淀为可配置的流水线，而不是一次性脚本 |

| 典型自动化方向 | 说明 |
|----------------|------|
| **测试自动化** | 自动化测试、测试报告 |
| **出包自动化** | 自动出包、多渠道支持 |
| **热更自动化** | 自动热更、版本管理 |

| 要点和思考方向 |
|----------------|
| DevOps 是提高开发效率的重要手段 |
| 实现完整的自动化流程 |
| 建立可配置的流水线系统 |
| 实现监控和可观测性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「CI/CD」「流水线」「测试」「出包」「热更」「监控」 |
| **方法** | 让 AI 设计 DevOps 架构、流水线配置、监控方案 |
| **应用** | CI/CD 搭建、自动化流水线、可观测性 |
| **提示词范例** | 「需搭建 DevOps：测试、出包、热更自动化，请设计可配置流水线架构（非一次性脚本），以及 Jenkins/GitLab CI 的配置要点与监控告警方案」 |
