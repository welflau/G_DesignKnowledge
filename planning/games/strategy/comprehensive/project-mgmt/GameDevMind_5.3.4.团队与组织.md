## 部门分工

| 维度 | 内容 |
|------|------|
| **作用** | 定义不同部门的职责和分工，确保团队协作高效 |
| **应用场景** | 所有需要多部门协作的游戏项目；团队组建时；需要明确职责时 |
| **做什么的？** | 定义不同部门的职责和分工，确保团队协作高效 |
| **在哪用？** | 所有需要多部门协作的游戏项目 |

| 问题 | 解决方向 |
|------|----------|
| **如何合理划分部门职责？** | 根据工作内容划分；根据技能要求划分；考虑协作关系；明确各部门的职责范围 |
| **如何避免职责重叠或遗漏？** | 建立清晰的职责边界；定期审查职责划分；明确协作接口；建立沟通机制 |
| **如何促进部门间的协作？** | 建立协作流程；明确协作接口；定期沟通会议；建立共同目标 |

| 要点和思考方向 |
|----------------|
| 部门分工是团队协作的基础 |
| 需要合理划分职责，避免重叠或遗漏 |
| 建立清晰的协作机制促进部门间协作 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「部门分工」「职责」「程序」「策划」「美术」「QA」「协作」 |
| **方法** | 让 AI 协助设计部门职责、协作接口、沟通机制 |
| **应用** | 部门职责划分、协作流程、职责边界 |
| **提示词范例** | 「游戏团队需部门分工：程序、策划、美术、QA、制作人、运维、运营，请协助设计各部门职责边界、协作接口，以及避免重叠或遗漏的职责矩阵」 |

## 岗位与角色

| 建立能力模型评估人员能力 |
| 制定培养计划培养团队成员 |

## 程序

| 类型 | 内容 |
|------|------|
| **职责** | 根据需求，保质保量完成系统或功能 |
| **要求** | 技术基本功越扎实越好；能良好的沟通（与策划、美术等协作） |

## 高级程序

| 类型 | 内容 |
|------|------|
| **职责** | 实现复杂系统；解决技术难题 |
| **要求** | 有技术分析和设计能力；有良好的沟通能力 |

## 主程序

| 类型 | 内容 |
|------|------|
| **职责** | 做好统筹规划；能解决技术难题；能把大的工作目标分解成小的任务部件；沟通协调；了解人，用好人；保障质量和进度；能做好过程保障；能做好结果保障；对团队有积极的带动作用 |
| **要求** | 需要具备技术能力、管理能力、沟通能力等多方面的综合能力 |

| **应用场景** | 人员招聘；职业规划；团队建设；技术选型 |
| **做什么的？** | 定义技术人员的专业方向和发展路径 |
| **在哪用？** | 人员招聘、职业规划、团队建设等场景 |

| 问题 | 解决方向 |

| **如何培养专业人才？** | 提供专业培训；安排专业项目；建立学习路径；提供实践机会 |

| 要点和思考方向 |

| 技术方向帮助技术人员明确专业方向和发展路径 |
| 需要根据工作内容和技能要求划分技术方向 |
| 提供专业培训和实践机会培养专业人才 |
| 平衡专业化和通用性，培养T型人才 |

## 客户端

| 方向 | 说明 | 细分 |
|------|------|------|
| **GamePlay** | 跟产品和体验相关性更大，直接影响玩家的游戏体验 | 系统：跟（UI）系统交互相关的功能开发；3C（Character Camera Control）：跟角色互动玩法和体验相关的功能开发 |
| **引擎** | 关注技术更多，关注底层技术和性能优化 | General：通用引擎功能；图形：图形渲染相关；工具流水线：工具和流水线开发 |

## 服务端

| 方向 | 说明 |
|------|------|
| **业务功能开发** | 实现游戏的服务端业务逻辑 |
| **架构** | 设计和维护服务端架构 |

## 人员招募

| 维度 | 内容 |
|------|------|
| **作用** | 招聘合适的技术人员，组建高效的开发团队 |
| **应用场景** | 团队组建；人员补充；团队扩张 |
| **做什么的？** | 招聘合适的技术人员，组建高效的开发团队 |
| **在哪用？** | 团队组建和人员补充 |

| 评估维度 | 说明 |
|----------|------|
| **笔试** | 看逻辑思维能力；程序或算法考察编程能力和算法基础 |
| **面试-技术知识** | 看储备的弹药量，考察技术知识的广度和深度 |
| **面试-做过的工作** | 看走过的坑、积累的经验，通过实际项目经验了解候选人能力 |
| **面试-解决过的难题** | 看处理问题的能力、探究问题的深度 |
| **面试-其它闲聊** | 看脾气秉性、团队协作 |

| 问题 | 解决方向 |
|------|----------|
| **如何评估候选人的能力？** | 多维度评估；结合笔试和面试；参考实际项目经验 |
| **如何判断候选人是否适合团队？** | 评估技术能力匹配度；评估性格和团队文化匹配度；评估协作能力；评估学习能力 |
| **如何吸引优秀人才？** | 提供有竞争力的薪酬；提供良好的发展机会；营造良好的团队文化；提供有挑战性的项目 |

| 要点和思考方向 |
|----------------|
| 人员招募是组建高效团队的基础 |
| 通过笔试和面试多维度评估候选人能力 |
| 评估技术能力、项目经验、问题解决能力、性格和协作能力 |
| 判断候选人是否适合团队，考虑技术匹配度和文化匹配度 |
| 提供有竞争力的薪酬和发展机会吸引优秀人才 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「招聘」「笔试」「面试」「评估」「候选人」 |
| **方法** | 让 AI 协助设计笔试题目、面试问题、评估维度 |
| **应用** | 招聘流程、能力评估、文化匹配 |
| **提示词范例** | 「需招募游戏程序，请协助设计笔试（逻辑/算法）、面试问题（技术知识/做过的工作/解决过的难题），以及技术能力与团队文化匹配的评估维度」 |
