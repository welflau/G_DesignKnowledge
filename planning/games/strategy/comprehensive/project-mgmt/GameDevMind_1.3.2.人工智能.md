## 有限状态机（FSM）

| 维度 | 内容 |
|------|------|
| **作用** | 用有限状态和转换描述对象行为 |
| **应用场景** | NPC 行为；游戏状态；动画状态；简单 AI |
| **组成** | 状态（行走、攻击、防御）；转移条件（如血量<50%→逃跑）；动作（如攻击时用武器） |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **状态数量爆炸** | 分层状态机、行为树 |
| **状态转换逻辑复杂** | 明确条件；状态转换表；避免循环依赖 |

| 要点和思考方向 |
|----------------|
| 适合简单行为；复杂用行为树；条件清晰 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明状态数量、转换逻辑、复杂度；如「NPC 巡逻/攻击/逃跑」「3 状态」「条件基于血量」 |
| **方法** | 状态少用 FSM；复杂用分层 FSM 或行为树；状态转换表 |
| **应用** | 实现 FSM；生成状态转换表；迁移到行为树 |
| **提示词范例** | 「NPC 有巡逻、追击、攻击、逃跑 4 状态，血量<30% 逃跑，发现玩家追击，距离<2 攻击，请用 FSM 实现并给出状态转换表」 |

## 寻路

|------|------|
| **应用** | 物体/角色自动寻找合理路径 |
| **应用场景** | NPC 寻路；玩家挂机；单位路径规划 |

|------|----------|
| **动态障碍物** | RVO、ORCA；定期重算路径 |

| 要点和思考方向 |
|----------------|

|------|----------------|

| **应用** | 实现寻路；路径平滑；避障逻辑 |
| **提示词范例** | 「Unity 中多个 NPC 同时寻路到目标，会互相碰撞，请用 RVO 或 ORCA 实现局部避障，并说明与 A* 的配合方式」 |

## A*

| 维度 | 内容 |
|------|------|
| **作用** | 寻找图中最短路径，保证最优 |
| **应用场景** | 需最优路径；网格/导航网格寻路 |
| **算法特点** | 贪心+估价函数；曼哈顿/欧几里得距离；高效 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **性能问题** | 分层寻路；预计算；限制搜索范围 |
| **启发函数选择** | 按地图特点选择；平衡效率与准确性 |

| 要点和思考方向 |
|----------------|
| 不需最优可用 JPS、Theta*；启发函数需可采纳性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明地图类型、精度需求、性能；如「网格寻路」「需最优路径」「实时 60fps」 |
| **方法** | 最优用 A*；不需最优用 JPS/Theta*；启发函数需可采纳且一致 |
| **应用** | 实现 A*；选择启发函数；实现 JPS |
| **提示词范例** | 「2D 网格寻路，8 方向移动，允许对角线，请实现 A* 并说明曼哈顿距离与欧几里得距离作为启发函数的区别和选择」 |
