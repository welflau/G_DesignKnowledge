<h2 align="center">角色</h2>
<p>
对于有角色的游戏，其中一个最重要的体验感就是来自于角色，包括角色的表现、动作、行为、控制等等。角色系统涉及基本特性、物理表现、角色控制、战斗系统等多个方面。

## 基本特性

| 维度 | 内容 |
|------|------|
| **作用** | 角色的基础属性和能力 |
| **应用场景** | 所有有角色的游戏、角色设计、游戏平衡性设计 |

| 问题 | 解决方向 |
|------|----------|
| **如何设计角色属性** 需要设计合理的角色属性系统 | 属性：速度（影响移动能力、平衡速度差异）、力量（影响攻击力、平衡战斗）、敏捷/重量（影响反应速度、移动灵活性）；建立属性体系；实现属性计算与成长 |
| **如何设计角色能力** 需要设计角色的特殊能力 | 设计独特的能力系统；实现能力激活机制；平衡能力强度；实现能力系统、冷却、效果 |

| 属性 | 说明 |
|------|------|
| **速度** | 角色移动速度，影响移动能力、平衡不同角色速度差异 |
| **力量** | 角色力量属性，影响攻击力或物理能力，平衡战斗系统 |
| **敏捷/重量** | 影响角色反应速度、移动灵活性，平衡角色特性 |

| 要点和思考方向 |
|----------------|
| 属性：速度、力量、敏捷/重量 |
| 能力：角色的特殊能力 |
| 属性设计要考虑游戏平衡性 |
| 能力设计要符合角色定位 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明游戏类型、平衡需求；如「属性」「速度」「力量」「敏捷」「能力」「冷却」 |
| **方法** | 让 AI 设计属性体系、能力系统、成长与平衡 |
| **应用** | 属性设计、能力系统、数值平衡 |
| **提示词范例** | 「角色需属性：速度、力量、敏捷，以及技能能力带冷却，请设计 C# 属性类、能力激活与冷却逻辑，以及属性对移动和攻击的影响计算」 |

## 物理表现

| 维度 | 内容 |
|------|------|
| **作用** | 角色的物理交互和表现 |
| **应用场景** | 需要物理交互的游戏、角色与场景交互、角色物理反馈 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现角色的物理表现** 需要实现真实的物理表现 | 基本物理要求：能被阻挡（碰撞检测、响应、边界）、不能被穿透（连续碰撞检测、穿透修复）、贴着地形走（地形检测、角色对齐、坡度处理）、要有物理但不能"躺倒"（物理约束、姿态控制）；使用物理引擎；实现物理控制器 |
| **如何处理角色与场景的交互** 需要处理角色与场景的物理交互 | 实现场景碰撞检测；实现交互响应；实现物理反馈；优化交互性能 |
| **特殊物理效果** 需要实现特殊的物理效果 | 布娃娃：死亡后、被炸飞；软体：服饰、非人类角色 |

| 基本物理要求 | 说明 |
|--------------|------|
| **能被阻挡** | 角色可以被障碍物阻挡，实现碰撞检测、响应、边界处理 |
| **不能被穿透** | 角色不能穿透其他物体，实现连续碰撞检测、穿透修复 |
| **贴着地形走** | 角色要贴合地形移动，实现地形检测、角色对齐、坡度处理 |
| **要有物理但不能"躺倒"** | 角色要有物理但保持站立，实现物理约束、姿态控制 |

| 特殊物理效果 | 应用举例 |
|--------------|----------|
| **布娃娃** | 死亡后、被炸飞 |
| **软体** | 服饰、非人类角色 |

| 要点和思考方向 |
|----------------|
| 能被阻挡、不能被穿透、贴着地形走、要有物理但不能"躺倒" |
| 布娃娃：死亡后、被炸飞 |
| 软体：服饰、非人类角色 |
| 物理表现要平衡真实性和游戏性 |
| 优化物理性能避免影响游戏流畅度 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「CharacterController」「Rigidbody」「碰撞」「地形」「布娃娃」 |
| **方法** | 让 AI 设计物理控制器、碰撞、地形贴合、布娃娃切换 |
| **应用** | 角色物理、地形贴合、布娃娃实现 |
| **提示词范例** | 「Unity 角色需贴着地形走、不穿透、死亡后切换布娃娃，请对比 CharacterController 与 Rigidbody 的取舍，以及如何实现死亡时切换到 Ragdoll」 |

## 角色控制

| 维度 | 内容 |
|------|------|
| **作用** | 角色的控制方式和辅助功能 |
| **应用场景** | 所有需要角色控制的游戏、玩家控制、AI 控制 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现玩家控制** 需要实现玩家对角色的控制 | 实现输入处理、控制响应、控制反馈；实现输入系统、控制映射、控制辅助 |
| **如何实现 AI 控制** 需要实现 AI 对角色的控制 | 寻路：使用 A*、NavMesh 等寻路算法，路径优化，动态避障；策略：使用状态机或行为树，实现决策逻辑；实现 AI 系统、行为树或状态机 |
| **如何处理第一人称和第三人称** 需要支持不同的视角控制 | 第一人称：第一人称摄像机、视角控制、武器显示；第三人称：第三人称摄像机、视角跟随、视角切换；实现视角切换、平滑过渡 |
| **控制辅助功能** 需要实现控制辅助功能 | 自动目标锁定：目标检测、锁定、锁定切换；动画控制：动画驱动、响应；辅助瞄准、自动移动、操作简化 |

| 控制类型 | 说明 |
|----------|------|
| **玩家控制** | 玩家直接控制角色，实现输入处理、控制响应、控制反馈 |
| **AI 控制（寻路、策略）** | 寻路（A*、NavMesh）、策略（状态机、行为树） |
| **动画控制** | 动画驱动控制 |

| 要点和思考方向 |
|----------------|
| 控制类型：玩家控制、AI 控制（寻路、策略）、动画控制 |
| 控制辅助：自动目标锁定 |
| 第一人称、第三人称视角控制 |
| 控制要流畅响应玩家输入 |
| AI 控制要自然合理 |
| 支持多种控制方式提高游戏体验 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明控制类型；如「输入」「寻路」「NavMesh」「A*」「行为树」「状态机」「目标锁定」 |
| **方法** | 让 AI 设计输入映射、寻路、AI 决策、目标锁定 |
| **应用** | 玩家控制、AI 控制、寻路、目标锁定 |
| **提示词范例** | 「Unity NPC 需 NavMesh 寻路和状态机决策，请给出 NavMeshAgent 寻路、到达目标后切换状态的 C# 实现，以及如何实现追击和攻击的状态转换」 |

## 战斗系统

| 维度 | 内容 |
|------|------|
| **作用** | 角色的战斗框架和体验 |
| **应用场景** | 有战斗系统的游戏、战斗玩法设计、战斗体验优化 |

| 问题 | 解决方向 |
|------|----------|
| **如何设计战斗框架** 需要设计合理的战斗框架 | 操作决策：在战斗中如何获得更优决策，举例追击、攻击、放技能；行为策略：战场态势观察和反应，举例 BOSS、NPC、玩家；基本功能：与决策无关的基础操作，举例行走、动画、寻路；设计战斗流程；实现战斗模块 |
| **如何提供战斗决策支持** 需要为玩家或 AI 提供战斗决策支持 | 实现决策系统、决策提示、自动决策 |
| **如何平衡战斗体验** 需要平衡战斗体验 | 平衡性：数值平衡、机制平衡、体验平衡；实现平衡性测试、数据收集、平衡调整 |
| **战斗体验优化** 需要优化战斗体验 | 动画系统、特效系统、镜头处理、音效、震动反馈、后处理；实现体验优化、反馈系统 |

| **行为策略** | 战场态势观察和针对态势做出自身的反应 | BOSS、NPC、玩家 |
| **基本功能** | 各种跟决策判断无关的基础操作 | 行走、动画、寻路 |

| **镜头处理** | 镜头跟随、镜头切换 |
