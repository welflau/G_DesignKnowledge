## 线性表

## 顺序/数组（Array）

| 维度 | 内容 |
|------|------|
| **作用** | 连续内存的线性结构，支持随机访问 |
| **应用场景** | 一次性分配、分配后少变动；需随机访问；固定大小集合 |
| **优势** | 随机访问 O(1)；实现简单；内存连续，缓存友好 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **中间插入删除效率低**（O(n)） | 频繁中间操作用链表；仅末尾操作用数组；标记删除 |
| **尺寸有限** | 动态数组；reserve() 预分配；预估最大容量 |
| **内存浪费** | 合理预估；动态数组按需扩展；定期整理 |

| 要点和思考方向 |
|----------------|
| 适合随机访问；预分配提高性能；注意内存浪费 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明访问模式、大小预估、语言；如「需随机访问玩家列表」「最多 1000 个元素」 |
| **方法** | 要求 reserve() 预分配；说明插入删除频率；提供 STL/语言对应容器 |
| **应用** | 选数组 vs 动态数组；生成预分配代码；优化扩容策略 |
| **提示词范例** | 「这段代码用 List 存储 500 个敌人用于随机访问，请改为数组或 vector，并预分配容量」 |

## 链表（Linked List）

| 维度 | 内容 |
|------|------|
| **作用** | 指针连接的节点序列，支持动态插入删除 |
| **应用场景** | 频繁变化；频繁中间插入删除；大小不确定 |
| **优势** | 动态伸缩；中间删除 O(1)；按需分配 |
| **举例** | GC 内存管理 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **额外存储开销** | 小对象用数组；对象池；紧凑节点结构 |
| **无法快速定位**（O(n)） | 频繁查找用哈希表；维护索引；跳表 |
| **实现复杂** | 标准库 list；智能指针；边界检查和断言 |
| **缓存不友好** | 关键路径用数组；内存池；数组链表混合 |

| 要点和思考方向 |
|----------------|
| 适合频繁插入删除；注意内存和缓存；使用标准库；关键路径考虑数组 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明操作模式、查找频率、性能要求；如「频繁头部插入删除」「需按 ID 查找」 |
| **方法** | 频繁查找推荐哈希表；中间操作用链表；热路径考虑数组 |
| **应用** | 选链表 vs 数组；生成跳表/索引方案；优化缓存局部性 |
| **提示词范例** | 「这个技能冷却列表需频繁头部插入删除，但也要按 ID 查找，请推荐结构（链表+哈希？）并说明理由」 |

## 栈（Stack）

| 维度 | 内容 |
|------|------|
| **作用** | 后进先出（LIFO）的线性结构 |
| **应用场景** | 后进先出；递归非递归实现；表达式求值；遍历树状结构 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 函数调用栈 | 调用栈 |
| 表达式求值 | 表达式 |
| 撤销操作 | 撤销 |
| 括号匹配 | 括号匹配 |
| DFS | 深度优先搜索 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **栈溢出** | 限制深度；迭代替代递归；增加栈空间；堆栈手动管理 |
| **性能问题** | 数组实现；预分配；对象池 |

| 要点和思考方向 |
|----------------|
| 注意栈溢出；使用数组实现提高性能 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明深度、递归/迭代、语言；如「表达式嵌套 10 层」「DFS 遍历 1000 节点」 |
| **方法** | 深度大要求迭代；数组实现栈；预分配栈容量 |
| **应用** | 将递归改为迭代+栈；生成栈实现；检查溢出风险 |
| **提示词范例** | 「将这段递归遍历场景树的代码改为迭代+栈实现，避免深层场景栈溢出，C++」 |

## 队列（Queue）

| 维度 | 内容 |
|------|------|
| **作用** | 先进先出（FIFO）的线性结构 |
| **应用场景** | 先进先出；消息管理；任务调度；缓冲区 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 任务调度 | 任务队列 |
| 消息队列 | 消息 |
| BFS | 广度优先搜索 |
| 打印队列 | 打印 |
| 网络缓冲 | 数据包缓冲 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **队列溢出** | 循环队列；动态扩展；阻塞队列；丢弃旧数据 |
| **性能问题** | 数组实现循环队列；预分配；批量处理；无锁队列 |
| **优先级需求** | 优先队列；堆实现；多队列按优先级 |

| 要点和思考方向 |
|----------------|
| 注意溢出；需优先级用优先队列；多线程用线程安全队列 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明吞吐量、优先级、多线程；如「每秒 1000 消息」「需按优先级处理」「多生产者」 |
| **方法** | 高吞吐用循环队列；优先级用堆；多线程用并发队列 |
| **应用** | 设计消息队列；实现优先队列；生成线程安全队列 |
| **提示词范例** | 「网络层收到的数据包需按优先级（紧急>普通）处理，请用优先队列实现，支持多线程入队」 |

## 树（Tree）

| 维度 | 内容 |
|------|------|
| **作用** | 分割/分类数据，层次化结构 |
| **应用场景** | 场景静态物体空间管理；层次化组织；快速查找；BVH 层次包围盒 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 场景可见性 | 判断物体是否在视野内 |
| 碰撞检测 | 物体间碰撞检测 |
| 动画骨骼 | 骨骼层次结构 |
| 文件系统 | 目录树 |
| 数据库索引 | B 树、B+ 树 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **树不平衡** | 自平衡树（AVL、红黑树）；定期重平衡；B 树 |
| **内存开销** | 数组实现完全二叉树；紧凑节点；树池 |
| **更新复杂** | 标准库；延迟更新；不可变树 |

| 要点和思考方向 |
|----------------|
| 注意平衡；考虑内存；使用标准库 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明用途、数据量、更新频率；如「场景物体空间划分」「BVH 碰撞」「静态/动态」 |
| **方法** | 静态用一次构建；动态需重平衡；提供八叉树/BVH/四叉树选择 |
| **应用** | 设计空间划分结构；生成树构建代码；选合适树类型 |
| **提示词范例** | 「为 1000 个静态场景物体做碰撞检测，请推荐空间结构（八叉树/BVH）并给出构建伪代码」 |

## 二叉树

| 维度 | 内容 |
|------|------|
| **作用** | 每节点最多两子节点的树 |
| **应用场景** | 搜索、排序；表达式树；决策树；BST |

| 问题 | 解决方向 |
|------|----------|
| **退化为链表** | 自平衡 BST（AVL、红黑树）；随机化插入；旋转保持平衡 |

| 要点和思考方向 |
|----------------|
| 注意平衡；自平衡避免退化 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数据分布、有序需求、退化风险；如「玩家 ID 有序插入」「可能退化为链表」 |
| **方法** | 要求 AVL/红黑树；避免顺序插入退化；提供标准库 map/set |
| **应用** | 将普通 BST 改为自平衡；生成旋转逻辑；选 map vs unordered_map |
| **提示词范例** | 「这段用普通 BST 存玩家 ID，按注册顺序插入会退化，请改为红黑树或 std::map，并说明选择」 |

## 多路查找树

| 维度 | 内容 |
|------|------|
| **作用** | 每节点可有多个子节点的树 |
| **应用场景** | 文件系统；数据库索引（B 树、B+ 树）；字典树（Trie） |

| 问题 | 解决方向 |
|------|----------|
| **节点分裂** | 成熟 B 树实现；合理节点大小；批量操作 |

| 要点和思考方向 |
|----------------|
| 适合大量数据索引；B 树适合数据库；Trie 适合字符串匹配 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明数据规模、查询类型、场景；如「百万级字符串前缀匹配」「数据库索引」 |
| **方法** | 前缀匹配用 Trie；范围查询用 B+ 树；提供实现选择 |
| **应用** | 设计 Trie 实现；选 B 树 vs B+ 树；生成字典树代码 |
| **提示词范例** | 「需要快速匹配技能名前缀（如输入 'fire' 匹配 'fireball'），请用 Trie 实现并给出查找接口」 |

## 堆（Heap）

| 维度 | 内容 |
|------|------|
| **作用** | 特殊完全二叉树，满足堆性质（父节点大于/小于子节点） |
| **应用场景** | 优先队列；堆排序；任务调度；Top K |

| 问题 | 解决方向 |
|------|----------|
| **堆化开销** | 批量操作；斐波那契堆；预分配 |
| **不支持随机访问** | 需查找用其他结构；哈希表辅助；索引堆 |

| 要点和思考方向 |
|----------------|
| 用数组实现；注意堆化开销 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明优先级语义、操作频率、Top K 需求；如「任务调度」「每帧取最高优先级」 |
| **方法** | 要求数组实现；批量操作优化；提供 std::priority_queue 或手写堆 |
| **应用** | 实现优先队列；Top K 问题；生成堆排序 |
| **提示词范例** | 「AI 行为需每帧取优先级最高的待执行行为，请用堆实现，支持动态增删，C++」 |

## 哈希表（Hash）

| 维度 | 内容 |
|------|------|
| **作用** | 基于哈希函数，快速插入、查找、删除（平均 O(1)） |
| **应用场景** | 快速查找；键值映射；去重；缓存 |
| **特点** | 关键字映射到位置实现快速访问 |

## 应用

| 场景 | 说明 |
|------|------|
| 文件变化检查 | 检查文件是否修改 |
| ID 映射 | 通过 ID 查找对象 |
| 缓存系统 | 查找缓存 |
| 去重 | 判断元素是否存在 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **哈希冲突** | 开放寻址；链地址法；再哈希；负载因子 |
| **哈希函数选择** | 分布均匀；MD5/SHA 简化版；根据数据特点；标准库 |
| **动态扩容** | 预分配；渐进式扩容；负载因子阈值；批量操作 |
| **内存开销** | 紧凑实现；合理初始容量；开放寻址减少指针 |

| 要点和思考方向 |
|----------------|
| 注意冲突；选好哈希函数；合理负载因子 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明键类型、冲突容忍、扩容需求；如「玩家 ID 为键」「高并发插入」 |
| **方法** | 要求合理初始容量；负载因子阈值；链地址 vs 开放寻址 |
| **应用** | 设计哈希表；选哈希函数；生成 LRU 缓存 |
| **提示词范例** | 「用哈希表实现玩家 ID 到对象的映射，预计 10000 玩家，请给出 C++ unordered_map 的 reserve 和负载因子建议」 |

## 图（Graph）

| 维度 | 内容 |
|------|------|
| **作用** | 节点和边组成，表示节点间关系 |
| **应用场景** | 状态管理；网络拓扑；路径规划；依赖关系 |
| **特点** | 边描述连通性、权重等 |

## 应用举例

| 场景 | 说明 |
|------|------|
| 角色状态 | 状态间转换 |
| 游戏状态 | 菜单、游戏中、暂停等 |
| 寻路 | 地图路径查找 |
| 社交网络 | 用户关系 |
| 依赖关系 | 任务依赖 |

**会遇到哪些问题？用什么解决？**

| 问题 | 解决方向 |
|------|----------|
| **遍历效率** | DFS/BFS 按场景选择；A*；并行；缓存结果 |
| **存储方式** | 稠密图邻接矩阵、稀疏图邻接表；压缩；图数据库 |
| **内存开销** | 稀疏矩阵；图压缩；分块加载；图数据库 |
| **算法复杂度** | 近似算法；并行；优化；图算法库 |

| 要点和思考方向 |
|----------------|
| 按稠密程度选存储；注意遍历效率；大规模用图数据库 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明图规模、遍历需求、存储约束；如「寻路网格 1000x1000」「需 A*」「稀疏/稠密」 |
| **方法** | 稠密用邻接矩阵；稀疏用邻接表；寻路用 A*；提供 DFS/BFS 选择 |
| **应用** | 设计图存储；实现寻路；生成图遍历 |
| **提示词范例** | 「地图是 500x500 网格，需实现 A* 寻路，请说明用邻接表还是网格直接遍历，并给出开放列表数据结构建议」 |
