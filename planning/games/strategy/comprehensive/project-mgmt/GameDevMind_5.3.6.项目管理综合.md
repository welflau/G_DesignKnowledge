## 敏捷开发

| 维度 | 内容 |
|------|------|
| **作用** | 一种以快速响应变化、持续交付价值为核心的开发方法 |
| **应用场景** | 需要快速迭代的项目；需要适应变化的项目；游戏开发；需求变化频繁的项目 |
| **做什么的？** | 一种以快速响应变化、持续交付价值为核心的开发方法 |
| **在哪用？** | 需要快速迭代、适应变化的项目，特别是游戏开发 |

| 问题 | 解决方向 |
|------|----------|
| **如何快速响应变化？** | 短周期迭代；持续集成和交付；快速反馈机制；灵活的计划调整 |
| **如何持续交付价值？** 价值驱动：以价值为导向，优先交付高价值功能，而非按计划执行 | 以价值为导向；优先交付高价值功能；持续交付可用的产品 |
| **如何提高团队协作效率？** 参考 [5.3.3.SCRUM](./5.3.3.SCRUM.md)；结对编程：两名开发者共同编写，提高代码质量和知识共享 | 使用SCRUM等敏捷框架；结对编程提高代码质量；持续沟通和协作；自组织团队 |

| 要点和思考方向 |
|----------------|
| 敏捷开发以快速响应变化、持续交付价值为核心 |
| 以价值为导向，优先交付高价值的功能 |
| 使用SCRUM等敏捷框架提高团队协作效率 |
| 结对编程提高代码质量和知识共享 |
| 短周期迭代，快速反馈 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「敏捷」「价值驱动」「SCRUM」「结对编程」「迭代」 |
| **方法** | 让 AI 协助设计敏捷实践、价值优先级、协作机制 |
| **应用** | 敏捷实践、价值排序、结对编程 |
| **提示词范例** | 「游戏项目需敏捷开发，以价值为导向优先交付高价值功能，请协助设计价值评估与优先级排序流程，以及结对编程与 SCRUM 的实践要点」 |

## PMP（项目管理专业）

| 维度 | 内容 |
|------|------|
| **作用** | 一套系统化的项目管理知识体系和方法 |
| **应用场景** | 大型项目；多项目环境；需要严格管理的项目；大型组织 |
| **做什么的？** | 一套系统化的项目管理知识体系和方法 |
| **在哪用？** | 大型项目、多项目环境、需要严格管理的项目 |

| 管理领域 | 说明 |
|----------|------|
| **相关方/合同管理** | 签订合同、监督合同履行、结束合同 |
| **采购管理** | 确定外部产品、服务或结果，有效管理 |
| **风险管理** | 识别、评估和应对风险 |
| **资源管理** | 确定人员、设备和材料，有效利用 |
| **沟通管理** | 确定信息需求，有效传递 |
| **整合/结算管理** | 记录项目成果，评估项目，确定改进点 |
| **范围管理** | 确定目标和边界，不超出范围 |
| **时间/进度管理** | 确定时间计划，按时完成 |
| **成本管理** | 确定预算，在预算范围内完成 |
| **质量管理** | 确保符合质量标准 |

| 项目管理过程 | 说明 |
|--------------|------|
| **开始** | 确定范围、目标、预算和资源 |
| **规划** | 制定项目计划（时间、成本、质量等） |
| **执行** | 转化为具体行动，团队协作完成任务 |
| **监控和控制** | 监督进展，对变化做出反应，确保按计划完成 |
| **关闭** | 完成项目，归档项目文件 |

| 问题 | 解决方向 |
|------|----------|
| **如何系统化管理项目？** 更适用于组织很大、项目很多的环境 | 使用PMP知识体系；系统化管理各个领域；建立管理流程 |
| **如何应对复杂的项目环境？** | 遵循项目管理过程；系统化规划和管理；持续监控和控制 |
| **如何确保项目成功？** | 系统化管理各个领域；遵循项目管理过程；持续监控和控制；及时应对变化 |

| 要点和思考方向 |
|----------------|
| PMP是一套系统化的项目管理知识体系和方法 |
| 更适用于组织很大、项目很多的环境 |
| 涵盖多个管理领域：范围、时间、成本、质量、资源、沟通、风险、采购、合同、相关方等 |
| 遵循项目管理过程：开始、规划、执行、监控和控制、关闭 |
| 系统化管理确保项目成功 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「PMP」「范围」「时间」「成本」「风险」「资源」 |
| **方法** | 让 AI 协助设计项目管理流程、领域管理、过程模板 |
| **应用** | 项目规划、风险管理、资源管理 |
| **提示词范例** | 「大型项目需系统化管理，请协助设计 PMP 风格的项目启动清单（范围、目标、预算、资源）、规划模板（时间、成本、质量），以及监控与控制的检查要点」 |

## 管理工具

| 维度 | 内容 |
|------|------|
| **作用** | 用于支持项目管理的软件工具 |
| **应用场景** | 所有需要项目管理的场景；任务跟踪；进度管理；团队协作 |
| **做什么的？** | 用于支持项目管理的软件工具 |
| **在哪用？** | 所有需要项目管理的场景 |

| 候选工具 | 说明 |
|----------|------|
| **飞书** | 多维表格：灵活的数据管理和可视化 |
| **JIRA** | 功能丰富，适合大型团队 |
| **Rally** | 敏捷项目管理工具 |
| **Gitlab** | 简单轻量，适合小型团队 |
| **IBM Rational** | 企业级项目管理工具 |

| 工具功能 | 说明 |
|----------|------|
| **泳道** | 可视化不同角色或团队的工作流程 |
| **里程碑** | 标记项目关键节点，跟踪进度 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的项目管理工具？** | 评估团队规模和需求；考虑工具的功能和易用性；考虑工具的集成能力；考虑工具的成本 |
| **如何让工具真正发挥作用？** | 建立工具使用规范；培训团队成员；持续使用和维护；根据需求调整配置 |
| **如何平衡工具的复杂性和易用性？** | 根据团队规模选择；从小功能开始使用；逐步扩展功能；保持简单易用 |

| 要点和思考方向 |
|----------------|
| 管理工具用于支持项目管理 |
| 根据团队规模和需求选择合适的工具 |
| 建立工具使用规范，让工具真正发挥作用 |
| 使用泳道可视化工作流程，使用里程碑跟踪进度 |
| 平衡工具的复杂性和易用性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「飞书」「JIRA」「泳道」「里程碑」「任务跟踪」 |
| **方法** | 让 AI 协助设计工具选型、配置方案、使用规范 |
| **应用** | 工具选型、泳道设计、里程碑设置 |
| **提示词范例** | 「团队需项目管理工具，请对比飞书、JIRA、GitLab 的适用场景，协助设计泳道与里程碑配置，以及工具使用规范与培训要点」 |

## OKR框架

| 维度 | 内容 |
|------|------|
| **作用** | 目标与关键结果框架，用于设定和跟踪目标 |
| **应用场景** | 需要明确目标和衡量结果的场景；从个人到组织都可以使用；需要目标对齐时；需要跟踪目标达成时 |
| **做什么的？** | 目标与关键结果框架，用于设定和跟踪目标 |
| **在哪用？** | 需要明确目标和衡量结果的场景，从个人到组织都可以使用 |

| OKR三问 | 说明 |
|---------|------|
| **我想去哪里？** | 设定清晰、有挑战性的目标 |
| **我怎么知道我到达了那里？** | 设定可衡量的关键结果，用于判断目标是否达成 |
| **我怎么到达那里？** | 制定执行计划，持续跟踪和调整 |

| 问题 | 解决方向 |
|------|----------|
| **如何设定有效的目标？** | 设定清晰、有挑战性的目标；目标要可达成但有一定挑战；目标要明确和具体；目标要可衡量 |
| **如何衡量目标达成情况？** | 设定可衡量的关键结果；关键结果要具体和可量化；定期跟踪关键结果；根据关键结果判断目标达成情况 |
| **如何确保目标对齐？** | 确保组织目标和个人目标对齐；确保团队目标对齐；定期回顾和调整目标；持续跟踪和调整 |

| 要点和思考方向 |
|----------------|
| OKR是目标与关键结果框架 |
| 目标要清晰、有挑战性（我想去哪里？） |
| 关键结果要可衡量，用于判断目标是否达成（我怎么知道我到达了那里？） |
| 制定执行计划，持续跟踪和调整（我怎么到达那里？） |
| 确保目标对齐，从个人到组织 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「OKR」「目标」「关键结果」「对齐」「可衡量」 |
| **方法** | 让 AI 协助设计 OKR 模板、关键结果、对齐流程 |
| **应用** | 目标设定、关键结果、目标对齐 |
| **提示词范例** | 「团队需 OKR 框架，请协助设计 OKR 三问（去哪里/怎么知道到达/怎么到达）的填写模板、关键结果的可衡量标准，以及组织与个人目标对齐的回顾流程」 |
