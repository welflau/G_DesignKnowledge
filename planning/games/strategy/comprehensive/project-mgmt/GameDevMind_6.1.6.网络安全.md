## 为什么要关注网络安全？

| 维度 | 内容 |
|------|------|
| **作用** | 保护游戏服务器和业务免受网络攻击，避免损失和被敲诈 |
| **应用场景** | 游戏服务器防护；业务安全保护；数据安全保护；用户数据保护 |
| **重要性** | 产品数据好时可能被"灰产"盯上，需提前做好防备 |

| 问题 | 解决方向 |
|------|----------|
| **如何应对服务器被入侵？** 可能被变成肉鸡挖矿或做DDos攻击 | 定期更新系统和工具，修复漏洞；使用密钥认证，设置IP白名单；监控异常访问，设置安全告警 |
| **如何应对业务被攻击？** 如DDos攻击后勒索保护费 | 提前做好防护，隐藏真实IP；使用高防服务抵御攻击；建立应急响应机制，快速应对攻击 |

| 要点和思考方向 |
|----------------|
| 网络安全不是可选项，而是必需品 |
| 随着业务发展，被攻击的风险会越来越大 |
| 需要提前做好防护，而不是等到被攻击后再处理 |
| 攻击者有自己的"产业"模式，攻击是有成本的，也会考虑目标的价值 |
| 建立完善的安全防护体系 |
| 定期进行安全审计和漏洞修复 |
| 建立应急响应机制，快速应对攻击 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「安全防护」「漏洞」「密钥」「IP白名单」「应急响应」 |
| **方法** | 让 AI 设计安全体系、审计清单、应急流程 |
| **应用** | 安全防护、漏洞修复、应急响应 |
| **提示词范例** | 「游戏服务器需安全防护，请设计系统漏洞修复与更新流程、密钥认证与 IP 白名单配置，以及应急响应与异常访问告警机制」 |

## DDos攻击

| 维度 | 内容 |
|------|------|
| **作用** | 用大规模流量攻击游戏服务器，导致服务无法访问 |
| **攻击方式** | 用大规模流量（几十G甚至几百上千G）访问主机，运营商为保网络稳定会封堵IP，业务停转，攻击者勒索 |
| **应用场景** | 游戏服务器攻击；业务勒索；竞争对手恶意攻击 |

| 问题 | 解决方向 |
|------|----------|
| **如何预防DDos攻击？** | 使用负载均衡、反向代理等方式隐藏真实IP；避免直接暴露服务器IP；建立安全防护体系 |
| **如何应对DDos攻击？** 不要接受勒索给钱 | 快速迁移环境，更换IP恢复服务；使用多链路攻击分流；使用高防IP服务抵御攻击 |
| **如何理解攻击者的行为模式？** 攻击者有产业模式，攻击有成本，会评估目标价值 | 理解攻击者的产业模式和成本；根据业务价值评估攻击风险；制定相应的防护策略 |

| 要点和思考方向 |
|----------------|
| DDos攻击是最常见的网络攻击方式之一 |
| 预防比应对更重要，隐藏真实IP是关键 |
| 高防服务是有效的解决方案，但成本较高 |
| 需要建立应急响应机制，能够快速应对攻击 |
| 不要接受勒索，这会助长攻击者气焰 |
| 理解攻击者的行为模式，制定相应策略 |
| 建立完善的防护和应急响应机制 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「DDoS」「高防IP」「负载均衡」「隐藏IP」「多链路」 |
| **方法** | 让 AI 设计防护架构、应急迁移、高防选型 |
| **应用** | DDoS 防护、IP 隐藏、应急恢复 |
| **提示词范例** | 「游戏需防 DDoS，请设计负载均衡与反向代理隐藏真实 IP 的架构、高防 IP 与多链路分流方案，以及攻击后快速迁移与更换 IP 的应急流程」 |

## DNS劫持

| 维度 | 内容 |
|------|------|
| **作用** | 劫持域名解析，将用户引导到恶意网站 |
| **攻击方式** | 劫持域名解析回包，将目标域名指向恶意IP，造成无法正常使用、弹广告、挂马、搜集账号密码、DDoS等危害 |
| **应用场景** | 域名解析攻击；用户数据窃取；恶意代码植入 |

| 问题 | 解决方向 |
|------|----------|
| **如何应对DNS劫持？** | 使用HTTPDNS服务避免DNS劫持；使用HTTPS加密通信；定期检查DNS解析是否正确 |
| **如何检测DNS劫持？** | 定期检查DNS解析是否正确；监控DNS解析异常；使用多个DNS服务器验证 |
| **如何保护移动端应用？** 移动端特别容易受DNS劫持 | 移动端应用特别需要使用HTTPDNS；避免运营商Local DNS劫持 |

| 要点和思考方向 |
|----------------|
| DNS劫持是常见的网络攻击手段，影响面广 |
| HTTPDNS可以有效避免DNS劫持问题 |
| 需要定期检查DNS解析是否正确 |
| 移动端应用特别需要使用HTTPDNS |
| 使用HTTPDNS服务提高解析安全性 |
| 定期检查DNS解析，监控异常情况 |
| 移动端应用必须使用HTTPDNS避免劫持 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「HTTPDNS」「DNS劫持」「HTTPS」「解析检查」 |
| **方法** | 让 AI 设计 HTTPDNS 接入、检测脚本、移动端方案 |
| **应用** | DNS 劫持防护、解析校验、移动端安全 |
| **提示词范例** | 「游戏移动端需防 DNS 劫持，请设计 HTTPDNS 接入方案、DNS 解析正确性检查脚本，以及 HTTPS 与解析异常监控要点」 |

## 暴力破解

| 维度 | 内容 |
|------|------|
| **作用** | 通过暴力破解密码等方式控制服务器 |
| **攻击目的** | 控制主机后植入木马、当肉机、当矿机等 |
| **应用场景** | 服务器入侵；数据窃取；资源滥用 |

| 问题 | 解决方向 |
|------|----------|
| **如何应对暴力破解攻击？** | 定期更新系统和工具，修复漏洞；关闭不必要的端口，只开放必要的服务；遵循最小权限原则 |
| **如何增强安全防护？** | 提升密码复杂度，使用强密码；使用密钥认证替代密码登录；设置IP白名单限制访问 |
| **如何监控和检测暴力破解？** | 监控登录失败次数；设置登录失败锁定机制；记录异常登录日志，设置告警 |

| 要点和思考方向 |
|----------------|
| 暴力破解是常见的服务器入侵手段 |
| 需要定期更新系统和工具，修复漏洞 |
| 最小权限原则：只开放必要的端口和服务 |
| 使用密钥认证比密码更安全 |
| IP白名单可以有效限制访问来源 |
| 定期更新系统和工具，修复安全漏洞 |
| 使用密钥认证和IP白名单增强安全 |
| 监控异常登录，及时响应安全威胁 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「暴力破解」「端口」「密钥」「密码复杂度」「IP白名单」 |
| **方法** | 让 AI 设计端口管理、认证加固、监控告警 |
| **应用** | 端口加固、密钥认证、登录监控 |
| **提示词范例** | 「需防暴力破解，请设计端口开放与关闭策略（最小权限）、密钥认证与密码复杂度要求、IP 白名单配置，以及登录失败锁定与异常告警」 |

## 服务商选择

| 维度 | 内容 |
|------|------|
| **作用** | 提供网络安全防护服务，如高防IP、DDoS防护、安全监控等 |
| **应用场景** | 需要专业安全防护的游戏服务；DDoS攻击防护；安全监控和告警；应急响应支持 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择服务商？** | 对比不同服务商的价格和服务；选择有良好口碑和案例的服务商；根据业务规模和需求选择合适的方案 |
| **如何评估服务商能力？** 各家云服务商都有安全服务，价格差别较大 | 评估服务商的技术能力和服务能力；建议选择有7x24小时技术支持的服务商 |
| **如何控制成本？** 安全服务是持续性的，需考虑长期成本 | 根据业务规模和预算选择合适的服务商；考虑长期成本，评估总体拥有成本；平衡成本和防护需求 |

| 要点和思考方向 |
|----------------|
| 各家云服务商都有相应的安全服务，不过价格差别会比较大 |
| 需要根据业务规模和预算选择合适的服务商 |
| 安全服务是持续性的，需要考虑长期成本 |
| 建议选择有7x24小时技术支持的服务商 |
| 仔细对比不同服务商的价格和服务 |
| 选择有良好口碑和案例的服务商 |
| 考虑长期成本和技术支持能力 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「高防」「服务商」「DDoS防护」「7x24」「成本」 |
| **方法** | 让 AI 设计选型评估、成本对比、方案建议 |
| **应用** | 服务商选型、成本评估、方案对比 |
| **提示词范例** | 「需选择高防/DDoS 防护服务商，请设计选型评估维度（防护能力、价格、7x24 支持、案例），以及长期成本与业务规模的匹配建议」 |
