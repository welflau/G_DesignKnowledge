## Lambda 表达式

| 维度 | 内容 |
|------|------|
| **作用** | 匿名函数，可作参数传递，简化代码 |
| **应用场景** | 函数式风格；临时函数；事件处理、回调；集合操作（排序、筛选、映射） |
| **特点** | 简化代码；"随用随写" |

### 应用举例

| 场景 | 说明 |
|------|------|
| 事件处理 | 玩家与游戏对象交互 |
| 排序 | 敌人列表、道具列表比较函数 |
| 筛选过滤 | 按条件筛选元素 |
| 映射 | 元素类型或格式转换 |
| 协程 | 异步操作、资源加载、动画 |

**会遇到哪些问题？用什么解决？**
| 问题 | 解决方向 |
|------|----------|
| **可读性** | 复杂逻辑提取为命名函数；有意义变量名；限制复杂度；注释 |
| **性能** | 关键路径谨慎使用；内联函数；避免循环中创建；函数指针/委托 |
| **调试困难** | 命名函数替代；调试器支持；日志；提取为独立函数 |
| **闭包陷阱** | 理解捕获机制；注意循环变量；显式传值；值捕获优于引用捕获 |

| 要点和思考方向 |
|----------------|
| 注意可读性；关键路径谨慎使用；理解闭包；复杂逻辑提取命名函数 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明用途（排序/筛选/回调）、语言、闭包捕获；如「C# 事件回调」「循环中 lambda 捕获变量」 |
| **方法** | 复杂逻辑要求提取命名函数；注意闭包捕获；提供热路径标识 |
| **应用** | 将复杂 lambda 重构为命名函数；检查闭包陷阱；生成可读的集合操作 |
| **提示词范例** | 「这段 C# 的 OrderBy 中 lambda 超过 5 行，请提取为命名方法并注意闭包是否捕获了循环变量」 |

## 并行编程

| 维度 | 内容 |
|------|------|
| **作用** | 多任务同时执行，提高性能 |
| **应用场景** | 多核利用；CPU 密集型；提高响应性；大规模数据处理 |

### 应用举例

| 场景 | 说明 |
|------|------|
| 多线程处理 | AI、物理、渲染；减少延迟、提高帧率 |
| 并行计算 | 粒子系统、寻路算法 |
| 任务并行 | 独立任务分配不同线程 |

**会遇到哪些问题？用什么解决？**
| 问题 | 解决方向 |
|------|----------|
| **线程同步** | 锁、原子操作、无锁结构；mutex；读写锁；条件变量；避免死锁 |
| **负载均衡** | 工作窃取；动态分配；线程池；监控负载 |
| **数据竞争** | 同步保护；Thread Local；消息传递；不可变结构 |
| **性能开销** | 线程池；减少切换；无锁结构；线程数≈CPU 核心数 |
| **调试困难** | 线程安全结构；ThreadSanitizer；日志；单元测试 |

| 要点和思考方向 |
|----------------|
| 注意同步和负载均衡；避免数据竞争和死锁；合理线程数；优先线程安全结构 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明任务类型、共享数据、平台；如「粒子系统并行」「多线程访问共享配置」 |
| **方法** | 要求锁/原子/无锁选择；提供数据竞争分析；说明线程池配置 |
| **应用** | 将串行改为并行；检查数据竞争；生成线程安全封装 |
| **提示词范例** | 「将这段粒子更新逻辑改为多线程并行，注意共享的 Transform 数组，请用合适同步方式并避免死锁」 |

## 异步开发

| 维度 | 内容 |
|------|------|
| **作用** | 任务执行中可进行其他操作，提高响应性和资源利用率 |
| **应用场景** | I/O 密集型；资源加载；提高响应性；避免阻塞主线程 |

### 应用举例

| 场景 | 说明 |
|------|------|
| 加载资源 | 避免阻塞主线程 |
| 网络通信 | 提高响应性 |
| 文件操作 | 不阻塞程序 |
| 数据库操作 | 提高并发 |

**会遇到哪些问题？用什么解决？**
| 问题 | 解决方向 |
|------|----------|
| **回调地狱** | Promise、async/await；提取命名函数；async 库；链式调用 |
| **错误处理** | try-catch、Promise.catch；全局处理；正确传播；Error Boundary |
| **竞态条件** | Promise.race；AbortController、CancellationToken；状态标志；锁 |
| **死锁风险** | 避免 .Result、.Wait()；ConfigureAwait(false)；保持异步链；Task.Run |
| **性能考虑** | 关键路径谨慎；协程；批量处理；异步 I/O |

| 要点和思考方向 |
|----------------|
| 注意错误处理和竞态；避免回调地狱；注意死锁；理解执行顺序和生命周期 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明 I/O 类型、取消需求、语言；如「资源加载需取消」「C# async 死锁」 |
| **方法** | 要求 async/await 或 Promise；CancellationToken/AbortController；避免 .Result/.Wait() |
| **应用** | 将回调改为 async；添加取消支持；解决竞态条件 |
| **提示词范例** | 「这段 Unity 资源加载用回调嵌套，请改为 async/await 或 UniTask，支持取消，并处理加载失败」 |

## 正则表达式

| 维度 | 内容 |
|------|------|
| **作用** | 模式匹配字符串，处理字符串操作 |
| **应用场景** | 验证校验；搜索替换；数据提取解析；文本处理 |
| **匹配类型** | 匹配字符；匹配位置（开头、结尾、边界） |
| **应用** | 校验；切割；提取；替换 |

### 应用举例

检查表情符号；判断是否数字；解析代码；验证邮箱/URL；提取日志关键信息

**会遇到哪些问题？用什么解决？**
| 问题 | 解决方向 |
|------|----------|
| **性能**：回溯过多 | 优化避免回溯；非贪婪匹配；避免嵌套量词；预编译；简单匹配用字符串方法 |
| **可读性** | 注释、命名组；拆分为多个简单正则；构建工具；测试用例 |
| **错误处理** | 测试工具验证；详细错误信息；try-catch；单元测试 |
| **安全性**：ReDoS | 避免指数回溯；限制复杂度；验证用户输入；超时机制 |

| 要点和思考方向 |
|----------------|
| 注意性能和可读性；避免回溯；复杂正则注释或拆分；避免 ReDoS |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明匹配目标、输入来源、性能要求；如「用户输入校验」「日志解析」「防 ReDoS」 |
| **方法** | 要求避免嵌套量词；非贪婪匹配；命名组提高可读性；提供测试用例 |
| **应用** | 生成/优化正则；添加 ReDoS 防护；拆分为多个简单正则 |
| **提示词范例** | 「写一个验证玩家昵称的正则，支持中文英文数字 2-16 字符，避免 ReDoS，并添加命名捕获组」 |

## 编译

| 维度 | 内容 |
|------|------|
| **作用** | 将高级语言转换为机器代码或字节码 |
| **应用场景** | C/C++、Java、Go、Rust 等；需优化性能的场景 |

### 编译阶段

| 阶段 | 说明 |
|------|------|
| 词法分析 | 分解为 tokens |
| 语法分析 | 构建 AST |
| 语义分析 | 类型检查等 |
| 中间代码生成 | 生成 IR |
| 优化 | 提高执行效率 |
| 目标代码生成 | 生成机器代码或字节码 |

**会遇到哪些问题？用什么解决？**
| 问题 | 解决方向 |
|------|----------|
| **编译错误** | IDE 语法检查；理解错误信息；静态分析；编码规范 |
| **编译性能** | 增量、并行、预编译头；distcc；优化头文件；模块化；ccache |
| **跨平台兼容性** | 标准库；多平台测试；条件编译；CMake；注意类型大小、字节序 |
| **链接错误** | 检查路径和顺序；确保依赖；符号导出导入；链接器选项 |
| **优化问题** | 理解优化选项；volatile；内存屏障；调试时关闭优化 |

| 要点和思考方向 |
|----------------|
| 使用工具优化编译时间；注意跨平台；理解优化行为；现代构建工具 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明语言、构建系统、错误类型；如「C++ 链接错误」「CMake 跨平台」「编译慢」 |
| **方法** | 提供完整错误信息；要求增量/并行编译；说明目标平台 |
| **应用** | 解释编译链接错误；优化 CMake/Makefile；生成预编译头配置 |
| **提示词范例** | 「这段 C++ 项目报 undefined reference，请分析可能原因（链接顺序、库路径、符号导出）并给出修复方案」 |
