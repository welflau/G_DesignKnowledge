## 图片制作

| 维度 | 内容 |
|------|------|
| **作用** | 生产 UI、图标、场景贴图等 2D 资源 |
| **应用场景** | UI 界面制作、图标设计、场景贴图制作、2D 游戏资源 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的图片格式** 需要根据用途选择合适的图片格式 | 根据用途选择格式；考虑透明通道需求；考虑压缩率需求 |
| **如何适配不同平台** 需要适配不同平台的分辨率和规格 | 定义不同平台的分辨率标准；实现多分辨率资源、自动缩放；考虑不同设备的像素密度；使用资源管理系统 |

| 图片格式 | 特点 | 适用 |
|----------|------|------|
| **PNG** | 无损压缩，支持透明通道 | UI 和图标 |
| **JPEG** | 有损压缩，不支持透明 | 照片和背景 |
| **WEBP** | 现代格式，压缩率高，支持透明 | 多种场景 |

| 要点和思考方向 |
|----------------|
| 工具举例：Adobe Photoshop 等 |
| 与技术沟通约定：图片规格（分辨率、长宽比，适配不同平台）；图片格式（PNG/JPEG/WEBP 等）；命名规范和导出目录结构 |
| 图片制作要考虑性能和兼容性 |
| 建立清晰的规范提高协作效率 |

## PCG（程序化内容生成）

| 维度 | 内容 |
|------|------|
| **作用** | 用规则和算法批量生成内容，减少纯手工劳动 |
| **应用场景** | 地形生成、植被布置、道路生成、地牢/地图布局、大量重复内容生成 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现可控的随机生成** 需要实现可控的随机生成 | 随机但可控（Seed + 规则），可复现；使用 Seed 控制随机性；定义生成规则；实现验证系统 |
| **如何应用到不同场景** 需要应用到不同的生成场景 | 设计通用生成框架；实现参数化配置；优化生成性能 |

| 适用场景 | 说明 |
|----------|------|
| **地形** | 地形算法、地形编辑、地形导出 |
| **植被布置** | 植被分布算法、植被类型选择、植被优化 |
| **道路** | 道路生成算法、道路连接、道路优化 |
| **地牢/地图布局** | 布局算法、布局验证、布局优化 |

| 要点和思考方向 |
|----------------|
| 随机但可控（Seed + 规则），可复现 |
| 适合：地形、植被布置、道路、地牢/地图布局等 |
| PCG 要平衡随机性和可控性 |
| 实现可复现性便于调试和优化 |

| **如何实现快速检索** 需要实现快速检索功能 | 按标签、类型、作者、时间等快速搜索；实现多条件搜索、智能搜索 |
| **如何实现版本管理** 需要实现版本管理和备份 | 自动版本管理与备份，支持回滚历史版本；实现版本管理、自动备份、版本回滚、版本对比 |
| **如何实现资源预览** 需要实现资源在线预览 | 常见格式（图片、模型、音频、视频等）的在线预览 |
