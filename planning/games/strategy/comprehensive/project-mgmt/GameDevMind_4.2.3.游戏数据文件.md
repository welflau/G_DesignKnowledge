## 3D 建模数据

| 维度 | 内容 |
|------|------|
| **作用** | 描述 3D 模型在引擎中的几何与材质信息 |
| **应用场景** | 3D 模型导入、模型渲染、模型编辑 |

| 问题 | 解决方向 |
|------|----------|
| **如何管理 3D 模型数据** 需要管理复杂的 3D 模型数据 | 网格数据（顶点位置、法线、顶点色、UV、骨骼权重、三角面索引）；材质数据（颜色、金属度、粗糙度等）；对象数据（平滑组）；设计数据格式；实现数据管理 |
| **如何选择数据格式** 需要选择合适的 3D 模型数据格式 | 评估格式特性、兼容性；选择合适格式 |

| 编辑期数据格式 | 特点 |
|----------------|------|
| **fbx** | 最常用的建模数据格式；二进制存储；可包含多种建模数据；可兼容多种 DCC 编辑器 |
| **collada** | XML 格式的建模数据文件；可读性好；跨平台 |
| **ma / max** | Maya / 3ds Max 编辑器的源文件格式；保留编辑信息 |

| 数据内容 | 说明 |
|----------|------|
| **网格数据** | 顶点位置、法线、顶点色、UV、骨骼权重、三角面索引 |
| **材质数据** | 材质属性（颜色、金属度、粗糙度等） |
| **对象数据** | 平滑组（对三角面分组，法线平滑） |

| 要点和思考方向 |
|----------------|
| 3D 建模数据是游戏资源的重要组成部分 |
| 选择合适的格式平衡性能和兼容性 |
| 优化数据格式提高加载性能 |
| 管理数据版本保证兼容性 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「FBX」「网格」「顶点」「材质」「UV」「骨骼权重」 |
| **方法** | 让 AI 设计模型数据结构、格式转换、导入流程 |
| **应用** | 模型导入、格式转换、数据优化 |
| **提示词范例** | 「游戏需导入 FBX 模型，请设计 C# 数据结构存储网格（顶点、法线、UV、骨骼权重）、材质，以及 FBX 到引擎内部格式的转换流程」 |

## 图片数据

| 维度 | 内容 |
|------|------|
| **作用** | 为模型、UI、特效等提供纹理与贴图 |
| **应用场景** | 模型贴图、UI 纹理、特效贴图、背景图片 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择图片格式** 需要选择合适的图片格式 | 评估格式特性、应用场景；选择合适格式 |
| **如何应用图片数据** 需要根据应用场景选择合适的格式 | 有损 vs 无损、有压缩 vs 无压缩、有 Alpha vs 无 Alpha、像素位数；高度场需要无损格式 |

| 常见格式 | 特点 | 适用 |
|----------|------|------|
| **TGA** | 32 位，有 8 位 Alpha 通道；无压缩 | 高质量贴图 |
| **PNG** | 压缩格式，有透明色；无损压缩 | UI 和图标 |
| **JPG** | 压缩格式，无半透明；有损压缩 | 照片和背景 |
| **DXT** | 多种 GPU 友好压缩格式 | 运行时使用 |

| 要点和思考方向 |
|----------------|
| 图片数据是游戏视觉表现的基础 |
| 根据应用场景选择合适的格式 |
| 平衡图片质量和文件大小 |
| 优化图片加载性能 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「TGA」「PNG」「DXT」「贴图」「Alpha」「压缩」 |
| **方法** | 让 AI 设计图片格式选型、转换流程、压缩策略 |
| **应用** | 贴图格式、格式转换、压缩优化 |
| **提示词范例** | 「游戏贴图需支持 TGA/PNG 源文件，运行时转为 DXT，请设计 C# 贴图导入与压缩流程，以及 UI 图标用 PNG、3D 贴图用 DXT 的选型策略」 |

## 声音数据

| 维度 | 内容 |
|------|------|
| **作用** | 为游戏提供音效与背景音乐 |
| **应用场景** | 背景音乐、音效、语音、环境音 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择音频格式** 需要选择合适的音频格式 | 评估格式特性、授权问题；选择合适格式 |
| **如何应用音频数据** 需要根据应用场景选择合适的格式和码率 | 不同设备、不同类型音频（BGM、语音、短音效）适合不同格式和码率：BGM 使用压缩格式；语音使用中等码率；短音效使用高质量格式 |

| 常见格式 | 特点 | 适用 |
|----------|------|------|
| **WAV** | 无压缩；音质好；体积较大 | 短音效 |
| **OGG** | 开源免费；有损压缩；体积小 | BGM 和语音 |
| **MP3** | 有损压缩；广泛支持；注意授权问题 | BGM |

| 要点和思考方向 |
|----------------|
| 声音数据是游戏体验的重要组成部分 |
| 根据应用场景选择合适的格式和码率 |
| 注意音频格式的授权问题 |
| 优化音频加载和播放性能 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「WAV」「OGG」「MP3」「BGM」「音效」「码率」 |
| **方法** | 让 AI 设计音频格式选型、码率策略、加载流程 |
| **应用** | 音频格式、码率选型、授权评估 |
| **提示词范例** | 「游戏音频：BGM 用 OGG、短音效用 WAV、语音用中等码率，请设计 C# 音频加载流程及不同场景的格式与码率选型表」 |

## Blob（Binary Large Object）

| 维度 | 内容 |
|------|------|
| **作用** | 作为大数据对象，用于存储各类对象的数据块 |
| **应用场景** | 大数据存储、快速加载、内存映射 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现 Blob 存储** 需要实现高效的 Blob 存储 | 加载数据后是连续的内存分布；数据结构在内存中的直接映射（结构体布局与文件布局一致）；设计 Blob 格式；实现内存映射 |
| **如何实现 Blob 读写** 需要实现高效的 Blob 读写流程 | 读：序列化（Blob→编辑/运行时数据）、分配（直接加载到内存）；写：序列化（编辑/运行时数据→Blob）、分配（直接存入 Blob） |

| 特性 | 说明 |
|------|------|
| **连续内存** | 加载数据后是连续的内存分布 |
| **直接映射** | 结构体布局与文件布局一致 |

| 要点和思考方向 |
|----------------|
| Blob 是高效数据存储的重要方式 |
| 实现连续内存分布提高访问性能 |
| 实现直接映射减少转换开销 |
| 优化读写流程提高性能 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明需求；如「Blob」「连续内存」「直接映射」「内存映射」 |
| **方法** | 让 AI 设计 Blob 格式、内存布局、mmap 方案 |
| **应用** | Blob 存储、零拷贝加载、大文件处理 |
| **提示词范例** | 「游戏需 Blob 存储配置，加载后连续内存、结构体与文件布局一致，请设计 C# Blob 格式与读写流程，以及内存映射（mmap）的跨平台实现要点」 |
