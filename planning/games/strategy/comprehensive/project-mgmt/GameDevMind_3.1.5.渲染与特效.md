## 渲染技术概览

## 全局光照（Global Illumination）

| 维度 | 内容 |
|------|------|
| **作用** | 模拟真实世界的光照效果，包括直接光照和间接光照 |
| **应用场景** | 需要真实感光照的3D游戏场景、写实风格游戏、开放世界游戏 |
| **做什么的？** | 模拟真实世界的光照效果，包括直接光照和间接光照 |
| **在哪用？** | 需要真实感光照的3D游戏场景 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现全局光照？** 全局光照计算复杂，性能开销大 | 使用预计算全局光照（Baked GI）、实时全局光照（Real-time GI）；使用环境光遮蔽（Ambient Occlusion）；使用 Light Probe 和 Reflection Probe；优化性能 |
| **全局光照性能优化** 全局光照可能影响渲染性能 | 使用预计算减少实时计算；限制全局光照范围；使用简化算法；优化光照贴图分辨率；实现动态开关 |

| 要点和思考方向 |
|----------------|
| 环境光遮蔽（Ambient Occlusion）：可增加画面光影真实感，模拟物体之间相互遮挡产生的阴影效果 |
| 考虑全局光照性能优化 |
| 使用预计算和实时混合方案 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「Baked GI」「Real-time GI」「AO」「Light Probe」 |
| **方法** | 让 AI 设计光照方案、生成 Light Probe 配置；或对比预计算与实时 GI、优化建议 |
| **应用** | 全局光照方案、AO 实现、光照贴图优化 |
| **提示词范例** | 「Unity URP 需要增加 Ambient Occlusion 增强画面真实感，请说明 SSAO 实现原理和 Shader 关键步骤」 |

## HDR

| 维度 | 内容 |
|------|------|
| **作用** | High Dynamic Range（高动态范围），让画面获得更大范围的颜色和对比度 |
| **应用场景** | 需要更丰富色彩表现和层次感的游戏场景、写实风格游戏、需要高对比度的场景 |
| **做什么的？** | High Dynamic Range（高动态范围），让画面获得更大范围的颜色和对比度 |
| **在哪用？** | 需要更丰富色彩表现和层次感的游戏场景 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现HDR？** 需要实现HDR渲染管线 | 使用HDR渲染目标；实现色调映射（Tone Mapping）；支持HDR显示设备；优化HDR性能；处理HDR到LDR的转换 |
| **HDR性能优化** HDR可能影响渲染性能 | 优化HDR渲染流程；使用合适的HDR格式；限制HDR范围；优化色调映射算法 |

| 要点和思考方向 |
|----------------|
| 让画面获得更大范围的颜色和对比度，让画面更有层次、更逼真 |
| 考虑HDR性能优化 |
| 实现合适的色调映射 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「HDR」「Tone Mapping」「颜色范围」 |
| **方法** | 让 AI 设计 HDR 管线、生成 Tone Mapping Shader；或说明 HDR 到 LDR 转换、ACES 曲线 |
| **应用** | HDR 渲染、色调映射、色彩增强 |
| **提示词范例** | 「Unity 后处理需要 HDR + Tone Mapping，请给出 URP 中 HDR 渲染目标配置和 ACES 色调映射 Shader 实现思路」 |

## 描边效果

| 维度 | 内容 |
|------|------|
| **作用** | 为物体添加描边，用于表现物体被选中或突出显示 |
| **应用场景** | 需要高亮显示物体的场景、选中物体、技能范围、UI高亮 |
| **做什么的？** | 为物体添加描边，用于表现物体被选中或突出显示 |
| **在哪用？** | 需要高亮显示物体的场景，如选中物体、技能范围等 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现描边效果？** 需要实现描边渲染效果 | 取得深度纹理的颜色；计算深度差值；深度差值小于某值表示边缘；否则绘制颜色纹理；使用法线扩展方法或后处理描边 |
| **如何控制描边宽度？** 需要控制描边宽度以适应不同场景 | 调整深度差值阈值；使用法线扩展距离；支持动态调整；优化描边性能 |
| **描边性能优化** 描边可能影响渲染性能 | 使用简化的描边算法；限制描边对象数量；优化描边 Shader；使用后处理描边 |

| 基本原理 |
|----------|
| 取得深度纹理的颜色；计算深度差值（1像素宽度描边）；深度差值小于某值表示边缘；否则绘制颜色纹理 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「深度描边」「法线扩展」「后处理」「选中高亮」 |
| **方法** | 让 AI 生成深度/法线描边 Shader；或对比两种描边方案、控制描边宽度 |
| **应用** | 选中高亮、技能范围、UI 描边 |
| **提示词范例** | 「需要为 3D 物体添加描边效果用于选中高亮，请给出基于深度差值的后处理描边 Shader（HLSL/Unity），并说明如何控制描边宽度」 |

## Bloom

| 维度 | 内容 |
|------|------|
| **作用** | 使图像增加光晕效果，使其显得更加有灵气 |
| **应用场景** | 需要发光效果的游戏场景、魔法效果、霓虹灯、光源效果 |
| **做什么的？** | 使图像增加光晕效果，使其显得更加有灵气 |
| **在哪用？** | 需要发光效果的游戏场景，如魔法效果、霓虹灯等 |

| 问题 | 解决方向 |
|------|----------|
| **如何实现Bloom效果？** 需要实现Bloom后处理效果 | 提取高亮区域；对高亮区域进行模糊处理；将模糊结果和原始图像相加；使用高斯模糊或双线性模糊；优化 Bloom 性能 |
| **Bloom性能优化** Bloom可能影响渲染性能 | 降低 Bloom 分辨率；使用多级模糊；限制 Bloom 范围；优化模糊算法；使用 GPU 加速 |

| 要点和思考方向 |
|----------------|
| 基本原理：将原始图像或像素颜色值模糊处理后，将模糊结果和原始图像像素值相加 |
| 考虑 Bloom 性能优化 |
| 使用合适的模糊算法 |

| 类型 | AI Coding 指南 |
|------|----------------|
| **交互提示** | 说明场景、目标；如「高亮提取」「高斯模糊」「双线性模糊」「光晕」 |
| **方法** | 让 AI 生成 Bloom 后处理 Shader；或设计多级模糊、阈值提取、性能优化方案 |
| **应用** | 发光效果、魔法光晕、霓虹灯 |
| **提示词范例** | 「Unity URP 需要实现 Bloom 后处理：提取高亮区域、高斯模糊、与原图相加，请给出 Shader 实现和可调参数（阈值、强度、模糊半径）」 |

## 渲染路径

| 维度 | 内容 |
|------|------|
| **作用** | 选择不同的渲染管线来处理光照和渲染 |
| **应用场景** | 所有3D游戏、不同性能要求的游戏、不同平台适配 |
| **做什么的？** | 选择不同的渲染管线来处理光照和渲染 |
| **在哪用？** | 所有3D游戏都需要选择合适的渲染路径 |

| 问题 | 解决方向 |
|------|----------|
| **如何选择合适的渲染路径？** 需要根据项目需求选择合适的渲染路径 | Forward Rendering：适合移动平台、支持半透明、光照数量有限；Deferred Rendering：适合PC/主机、支持大量光源、不支持半透明；根据平台和需求选择 |
| **Forward 和 Deferred 有什么区别？** 需要理解两种渲染路径的区别 | Forward：逐物体渲染，兼容性好；Deferred：先渲染 G-Buffer 再计算光照，性能更好 |

| 类型 | 特点 |
|------|------|
| **Forward Rendering（前向渲染）** | 传统方式，兼容性好；适合移动平台；支持半透明物体；光照数量有限 |
| **延迟渲染（Deferred Rendering）** | 性能更好；限制更多（不支持半透明）；适合PC和主机；支持大量光源 |
