# SubSkill: 叙事一致性校验

## 触发条件
当用户要求检查/校验/审核叙事内容的一致性时激活。

## 校验维度

### 1. 世界观一致性
- [ ] 所有设定是否符合世界观基础规则（科技水准、力量体系、社会结构）
- [ ] 各势力/阵营的描述是否自洽
- [ ] 历史事件的因果链是否合理
- [ ] 地理/空间描述是否前后一致

### 2. 角色一致性
- [ ] 角色性格是否在不同场景中保持一致（或有合理的变化原因）
- [ ] 角色知识边界是否正确（不应知道未获取的信息）
- [ ] 角色语言风格是否符合其身份和背景
- [ ] 角色关系网是否有矛盾

### 3. 时间线一致性
- [ ] 事件发生顺序是否合理
- [ ] 角色年龄/成长与时间线是否匹配
- [ ] 是否存在时间悖论
- [ ] 跨章节/跨版本的时间衔接是否正确

### 4. 叙事逻辑一致性
- [ ] 伏笔是否全部回收
- [ ] 转折是否有足够铺垫
- [ ] 角色动机是否合理（为什么做这件事）
- [ ] 任务目标链是否清晰（玩家是否始终知道自己在做什么）

### 5. 文案风格一致性
- [ ] 全文语气/语体是否统一（文言/白话/半文言）
- [ ] 专有名词拼写是否一致
- [ ] UI文本/系统文本与剧情文案风格是否协调
- [ ] 翻译/本地化用词是否统一

### 6. 玩法-叙事耦合度
- [ ] 剧情展示是否影响了核心玩法节奏
- [ ] 叙事承诺的能力是否在玩法中可实现
- [ ] 角色叙事强度是否与游戏中的强度匹配
- [ ] 叙事分支是否有对应的玩法差异

## 输出格式

```markdown
## 叙事一致性校验报告

### 总评: [通过/有风险/不通过]

### 问题清单
| # | 维度 | 严重度 | 问题描述 | 位置 | 建议修复 |
|---|------|--------|---------|------|---------|
| 1 | 世界观 | 高 | XXX与YYY矛盾 | 第3章第2节 | 统一为XXX |
```
