# SubSkill: 数值合理性校验

## 触发条件
当用户要求检查/验证数值设计的合理性时激活。

## 校验维度

### 1. 数值区间合理性
- [ ] 属性数值是否在合理范围内（不出现异常高/低值）
- [ ] 成长曲线是否平滑（无突变/断层）
- [ ] 各等级间的数值差异是否符合设计预期

### 2. 战斗平衡性
- [ ] DPS天花板是否合理
- [ ] 各职业/角色间的强度差异是否在可接受范围
- [ ] 克制关系是否形成闭环
- [ ] Boss血量与预期通关时间是否匹配

### 3. 经济系统健康度
- [ ] 货币产出/消耗是否平衡
- [ ] 是否存在无限刷资源的漏洞
- [ ] 通胀率是否在可控范围
- [ ] 付费与免费玩家的体验差距是否合理

### 4. 概率公平性
- [ ] 抽卡概率是否符合公示值
- [ ] 保底机制是否生效
- [ ] 随机事件的分布是否符合预期
- [ ] 是否存在概率陷阱

### 5. 成长节奏
- [ ] 玩家养成周期是否合理
- [ ] 各养成系统间是否有合理的优先级
- [ ] 是否存在养成断层（某一阶段无事可做）
- [ ] 版本更新的数值膨胀率是否可控

## 输出格式

```markdown
## 数值合理性校验报告

### 总评: [健康/需关注/有风险]

### 问题清单
| # | 维度 | 严重度 | 数据点 | 问题 | 建议 |
|---|------|--------|-------|------|------|
| 1 | 战斗平衡 | 高 | 角色A的DPS | 超出平均值200% | 降低XXX系数 |
```
