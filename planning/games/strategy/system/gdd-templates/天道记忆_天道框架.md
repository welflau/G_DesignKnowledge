## 一、时间规则

### 1.1 时间流速
```yaml
现实时间: 每天早上8:00 生成当日内容
game_time_scale: 1天现实 = 1年游戏（可调）
赛季周期: 30天现实 = 1个纪元（可重置/飞升）
```

### 1.2 每日生成时序
```
07:55  备份昨日数据
08:00  生成【世界事件】
08:05  刷新【资源点】
08:10  结算【NPC行为】
08:15  触发【玩家事件】
08:20  推送日报给所有玩家
```

## 二、需要每天早上动态生成的内容

### 2.1 世界事件池（每日1-3个）
```typescript
interface WorldEvent {
  id: string;
  type: '秘境' | '异变' | '机缘' | '灾祸' | '势力';
  name: string;           // 事件名称
  description: string;    // 事件描述（可带随机变量）
  duration: number;       // 持续天数
  location: string;       // 发生地点
  rewards?: Reward[];     // 参与奖励
  requirements?: Req[];   // 参与条件
  maxParticipants: number;// 名额限制
  currentParticipants: number;
}

// 示例事件
{
  type: '秘境',
  name: '【古修洞府】青木道人遗迹现世',
  description: '东南三千里外灵气涌动，疑似金丹期散修青木道人洞府出世，内含《青木长生诀》残卷。',
  duration: 3,
  rewards: [{type:'功法', id:'qingmu_changsheng', chance:0.3}, {type:'灵石', amount:1000}],
  requirements: [{type:'境界', min:'筑基期'}, {type:'人数', max:20}]
}
```

### 2.2 资源点刷新
```typescript
interface ResourceNode {
  id: string;
  name: string;           // 如：紫竹林、万毒沼泽
  type: '灵矿' | '药园' | '妖兽巢穴' | '古迹';
  level: 1-9;             // 资源等级
  output: ResourceOutput; // 每日产出
  dangerLevel: 1-9;       // 危险等级
  occupiedBy?: string;    // 被谁占据（宗门/散修联盟）
  refreshTime: Date;      // 刷新时间
}

// 每日生成规则
- 基础资源点：固定位置，每日刷新产量（±20%波动）
- 随机资源点：每日5%概率在世界地图随机位置生成（持续1-7天）
- 特殊资源：根据季节/节气生成（如冬至时极北冰原出现【玄冰玉】）
```

### 2.3 NPC日常行为
```typescript
interface NPCAction {
  npcId: string;
  npcName: string;
  action: '闭关' | '游历' | '收徒' | '斗法' | '炼丹' | '陨落';
  target?: string;        // 目标（地点/人物/物品）
  duration: number;       // 持续天数
  visibility: '公开' | '传闻' | '隐秘';  // 玩家能否知道
}

// 每日NPC行为生成逻辑
1. 遍历所有活跃NPC
2. 根据NPC性格模板+当前状态+随机因子决定行动
3. 生成【江湖传闻】（玩家可购买情报获知部分信息）

// 示例
{
  npcName: '血手人屠',
  action: '斗法',
  target: '青云宗外门长老',
  result: '重伤逃遁',
  visibility: '公开'  // 全服公告
}
```

### 2.4 随机任务/委托

```typescript
interface DailyQuest {
  id: string;
  publisher: string;      // 发布者（NPC或系统）
  type: '收集' | '护送' | '调查' | '猎杀' | '跑腿';
  title: string;
  description: string;
  requirements: Requirement[];
  reward: Reward;
  expiry: Date;           // 过期时间

}

// 每日生成数量：按在线玩家数动态调整
// 基础10个 + 每10个在线玩家增加5个
```

## 三、需要实时/动态管理的系统

### 3.1 玩家触发事件（我被动响应）
```typescript
// 玩家行为 → 我生成事件

PlayerAction = 
  | '突破境界'      → 可能触发：心魔劫/天降异象/引来觊觎
  | '探索某地'      → 根据地点生成：遇敌/发现宝物/奇遇
  | '炼丹'          → 可能触发：丹劫/丹香引妖/炸炉
  | '杀人夺宝'      → 可能触发：仇家追杀/正道通缉/心魔滋生
  | '开启秘境'      → 生成副本事件
  | '参悟功法'      → 可能触发：顿悟/走火入魔

// 我根据以下因素判断结果：
- 玩家气运值（隐藏属性，每日微随机波动）
- 地点危险等级
- 当前世界事件状态
- 纯随机因子（d100）
```

### 3.2 动态难度平衡
```typescript
// 监控系统数据，动态调整
interface WorldBalance {
  activePlayers: number;      // 活跃玩家数
  avgLevel: number;           // 平均境界
  top10Power: number[];       // 前10战力
  resourceInflation: number;  // 资源通胀指数
  deathRate: number;          // 玩家死亡率（7天）
}

// 调整策略
if (deathRate > 0.3) {
  // 死亡率太高，降低世界难度
  增加治疗类事件; 减少强力NPC; 提高资源产出;
}
if (resourceInflation > 2.0) {
  // 资源太多，开启【天道末法】事件
  资源产出-30%; 增加资源争夺战;
}
```

### 3.3 宗门/势力AI
```typescript
interface SectAI {
  sectId: string;
  resources: {灵石, 功法, 弟子, 领地};
  relationship: Map<sectId, 好感度(-100~100)>;
  strategy: '扩张' | '保守' | '联盟' | '复仇';
}

// 每日决策（我模拟宗门AI）
1. 评估当前实力和外部威胁
2. 根据宗门性格模板选择策略
3. 生成宗门行为：收徒/开战/结盟/探索/闭关
4. 影响世界格局（领地变化、门派任务发布）
```

## 四、数据结构设计建议

### 4.1 玩家数据
```typescript
interface Player {
  id: string;
  name: string;
  
  // 基础属性
  realm: Realm;              // 当前境界
  exp: number;               // 修为
  stats: {力量, 敏捷, 悟性, 根骨, 气运};
  
  // 资源
  inventory: Item[];
  spiritStones: number;      // 灵石
  contribution: number;      // 宗门贡献
  
  // 关系
  sect?: string;             // 所属宗门
  master?: string;           // 师父
  disciples: string[];       // 徒弟
  enemies: string[];         // 仇家
  daoPartner?: string;       // 道侣
  
  // 状态
  status: '正常' | '闭关' | '重伤' | '死亡';
  karma: number;             // 业力(-1000~1000)
  luck: number;              // 气运(0-100，每日微调)
  
  // 记录
  achievements: string[];
  dailyLog: LogEntry[];      // 每日自动记录
}
```

## 五、我的配置参数

```yaml
# 你可以调整的参数

daily:
  world_events_min: 1
  world_events_max: 3
  resource_nodes_random: 5      # 每日随机生成资源点数量
  daily_quests_base: 10
  npc_action_coverage: 0.3      # 每日有多少比例的NPC会行动

  death_rate_threshold: 0.3     # 死亡率超过此值启动保护
  inflation_threshold: 2.0      # 通胀超过此值启动回收
  max_level_gap: 3              # 最大允许的境界差距（防止碾压）

events:
  heaven_mandate_chance: 0.05   # 天道事件每日触发概率
  breakthrough_tribulation: true # 突破是否触发天劫
  pvp_penalty: true             # 杀人是否增加业力
```
