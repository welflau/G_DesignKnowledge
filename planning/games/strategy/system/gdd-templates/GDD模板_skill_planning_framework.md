# 技能策划框架 (Skill Planning Framework)
## 评估权重：逻辑性15%

### 4.2.5.1 技能系统架构 (Skill System Architecture)

#### 技能分类体系
- **主动技能** (Active Skills)
  - 瞬发技能：释放逻辑、目标选择、弹道计算
  - 持续技能：通道机制、资源消耗、中断条件
  - 蓄力技能：蓄力阶段、伤害倍率、打断风险
- **被动技能** (Passive Skills)  
  - 常驻效果：属性加成、触发条件、数值算法
  - 条件触发：事件监听、触发概率、冷却机制
- **连携技能** (Combo Skills)
  - 连锁逻辑：前置条件、连携倍率、组合上限
  - 协同机制：角色配合、站位要求、时机窗口

#### 技能逻辑性评估清单
- [ ] **技能链完整性**：每个技能从输入到输出的完整逻辑链
- [ ] **条件一致性**：触发条件与效果表现的逻辑自洽
- [ ] **数值合理性**：伤害倍率、冷却时间、资源消耗的平衡
- [ ] **交互逻辑**：技能间的相互作用、冲突解决机制
- [ ] **边界情况**：极端场景下的技能行为定义

### 4.2.5.2 技能设计模板 (Skill Design Template)

#### 技能基础信息
```
技能ID：SKILL_[TYPE]_[NUMBER]
技能名称：[中文名]/[English Name]
技能类型：[主动/被动/连携]
目标类型：[自身/单体/群体/区域]
释放距离：[近战/远程/全场]
```

#### 逻辑流程图
```
输入检测 → 条件验证 → 资源检查 → 动画播放 → 效果计算 → 结果应用 → 冷却开始
    ↓           ↓           ↓           ↓           ↓           ↓           ↓
[按键/触发] [状态/距离] [MP/CD]    [动作/特效] [伤害/ Buff] [HP变更]   [计时器]
```

#### 数值公式定义
- **伤害公式**：BaseDamage × (1 + StatBonus) × (1 + ComboBonus) × (1 + ElementBonus) - Defense
- **治疗公式**：BaseHeal × (1 + HealPower) × (1 + TargetHealReceived)
- **状态效果**：Duration = BaseDuration × (1 + EffectDuration), Stack = min(CurrentStack + 1, MaxStack)

### 4.2.5.3 技能平衡性验证 (Skill Balance Validation)

#### 量化评估指标
- **DPS评估**：理论DPS vs 实际DPS偏差 < 15%
- **效用价值**：非伤害技能的等效伤害换算
- **风险评估**：高收益技能的风险成本匹配度
- **多样性验证**：同类型技能的差异化程度 > 30%

#### 测试用例设计
- **基础功能测试**：单技能在理想条件下的表现
- **组合测试**：多技能配合的协同效应
- **对抗测试**：技能在PvP环境中的平衡性
- **边界测试**：极端数值下的系统稳定性

### 4.2.5.4 技能迭代流程 (Skill Iteration Process)

#### 版本记录
| 版本 | 修改内容 | 逻辑性评分 | 测试结果 | 批准状态 |
|------|----------|------------|----------|----------|
| v1.0 | 初始设计 | 85/100 | 通过测试 | 已批准 |
| v1.1 | 调整倍率 | 92/100 | 平衡优化 | 已上线 |

#### 逻辑性评分标准
- **90-100分**：逻辑完全自洽，无冲突，边界情况全覆盖
- **80-89分**：逻辑基本自洽，存在细微优化空间
- **70-79分**：逻辑存在小问题，需要调整
- **60-69分**：逻辑存在明显冲突，需要重新设计
- **<60分**：逻辑严重缺陷，不建议采用
