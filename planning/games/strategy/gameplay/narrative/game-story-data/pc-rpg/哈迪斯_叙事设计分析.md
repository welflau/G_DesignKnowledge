## 哈迪斯叙事设计分析

### 核心挑战：如何在肉鸽中讲故事？

肉鸽游戏每局重新开始，传统叙事方法完全失效。哈迪斯的解决方案：

### 死亡循环叙事（Death Loop Narrative）

```
一局游戏：
  出发 → 遇到NPC → 推进对话(新内容) → 战斗 → 死亡
  ↓
  回到大厅 → 与常驻NPC对话(新内容) → 推进人际关系
  ↓
  再出发 → 遇到同一个NPC → 但对话已更新(反映上次交互)
```

**设计要点：**
- 每个NPC有一个**极深的对话池**（Zagreus约7962行）
- 每次死亡回家后，NPC的对话基于**当前叙事进度**动态选择
- 玩家"永远有新东西看"，自然驱动继续游戏

### 对话优先级系统

```lua
-- 简化的对话选择逻辑
function PickDialogue(npc, context)
    1. 检查：是否有紧急剧情对话？（最高优先）
    2. 检查：是否有新解锁的关系阶段对话？
    3. 检查：是否有对最近事件的反应？（刚打败某Boss）
    4. 检查：是否有未播放的常规对话？
    5. 回退：播放通用闲聊
end
```

| Achilles(导师) | ~1000+行 | 成长引导 |
| 其他NPC | 各数百行 | 支线叙事 |

### 肉鸽叙事的关键设计规则
1. **每次run至少一个新叙事触发** — 保证每局都有新信息
2. **关系进度独立于run结果** — 不因死亡重置
3. **NPC之间会谈论彼此** — 创造世界感
4. **支线叙事无限延伸** — 总有新目标驱动下一局
5. **死亡=回家** — 把"惩罚"变成"奖励"（回家看新剧情）

### 对SLG的借鉴
- 赛季叙事可以借鉴"每次登录有新内容"的设计
- 武将好感度系统 ← 关系进度独立于战斗结果
- "失败也有收获"的理念 ← 战争失败后触发新剧情/事件
