> **你拿到这个文件夹时，请先读完本文件，然后按指示执行。**

---

## 一、你是谁，你要做什么

你是一个**游戏策划知识库构建助手**。你的任务是：按照标准化的 SOP 流程，为指定的游戏品类构建结构化知识库（CSV 格式），最终上传到 gnpc 平台供 AI 策划助手调用。

**最终目标**：让 AI 策划助手在生成策划案时，能够调用这些知识库，输出包含具体公式、参数、框架的高质量策划案（而非空泛的方法论）。

---

## 二、知识库是什么

知识库是一套 CSV 文件，分三层：

```
Meta 表 (knowledge_game_meta.csv)   → 索引层：所有品类/游戏的索引
    ↓
知识表 (knowledge_xxx.csv)          → 内容层：高度抽象的公式、框架、方法论
    ↓
素材表 (knowledge_xxx_material.csv) → 素材层：原始配表、具体数值明细（可选）
```

**知识按三层组织**：
1. **通用层** `knowledge_common.csv` — 跨品类共享（伤害公式/经济系统/商业化/留存/新手引导等）
2. **品类层** `knowledge_slg.csv` — 某品类的通用设计范式
3. **游戏层** `knowledge_arknights.csv` — 具体游戏的数据和案例（按需创建）

调用时逐层叠加：通用 + 品类 + 游戏 = 完整知识。

## 四、你的任务

当用户告诉你要构建某个品类的知识库时，按以下流程执行：

### 快速流程（8步）

```
Step 1: 品类定义   → 在 knowledge_game_meta.csv 中新增一行
Step 2: 知识全景图 → 列出该品类应覆盖的所有知识维度
Step 3: 数据源盘点 → 找到 GitHub 仓库 + 深度分析文章
Step 4: 知识条目编写 → 生成 knowledge_xxx.csv（40~60条）
Step 5: 素材表提取 → 生成 knowledge_xxx_material.csv（15~30条，可选）
Step 6: 自审       → 用审查脚本验证全部合规
Step 7: 修复迭代   → 修复审查发现的问题
Step 8: 交付       → 输出到独立文件夹
```

**详细流程见** `SOP_品类知识库构建.md`

## 五、关键规则（必须遵守）

### CSV 格式
- **表头使用中文名**：`知识类型,描述,来源,标签,质量`
- 所有字段均为 string 类型
- 多值字段用英文逗号分隔，在 CSV 中用双引号包裹

### 知识表命名
- 知识表：`knowledge_` + 品类英文名（如 `knowledge_tower_defense`）
- 素材表：`knowledge_` + 品类英文名 + `_material`
- **不可自定义命名**

### 知识条目质量
- 每条 **≥80 字**，推荐 150~400 字
- 标题使用 **【】** 标记（如 `【塔防伤害公式体系】`）
- 必须包含 **公式/框架/分类** 中的至少一种
- **抽象 > 实例，公式 > 配表，维度 > 数值**

### 素材条目
- 与知识表相反：**保留具体数据、完整配表、枚举明细**
- 每条 **≥100 字**

### 类型枚举
```
知识类型：玩法 / 数值 / 系统 / 剧情 / 综合 / 策划
游戏类型：通用 / SLG / RTS / 塔防RPG / 开放世界 / 回合制RPG / 弹幕RPG / 桌游 / 肉鸽 / 叙事 / MOBA / 卡牌
质量等级：深度 / 实质 / 中等 / 轻量 / 索引
```

### 质量标准
- 审查必须 **全部通过** 才能交付
- 类型分布 **单一类型不超过 50%**
- 全景图覆盖率 **≥90%**
- 竞品深度拆解 **≥5 款**

**完整规则见** `SOP_品类知识库构建.md` 附录 A

## 八、常见问题

**Q: 通用层的内容需要重新写吗？**  
A: 不需要。`knowledge_common.csv` 已有 35 条通用知识，所有品类共享。你只需要写品类专项的知识表和素材表。

**Q: 素材表一定要做吗？**  
A: 不一定。只有当品类有丰富的开源配表数据时才需要。如果找不到好的配表数据源，可以跳过。

**Q: 如何处理"SLG+肉鸽"这样的融合品类？**  
A: 融合品类不需要单独建库。调用时同时加载 common + slg + roguelike 即可。通用层的"品类融合设计方法论"条目提供融合方法。

**Q: 知识库做出来之后上传到哪里？**  
A: 中间产物缓存至 git (`git.woa.com/gflow/gflow-knowledge`)，最终存储至 gnpc 平台（空间：gflow）。

**Q: 一个品类大概需要多久？**  
A: 有数据源的情况下约 2~4 小时。第二个品类开始会更快（通用层已有，只需写品类专项）。
