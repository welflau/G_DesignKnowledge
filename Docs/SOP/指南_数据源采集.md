# 数据源采集指南

> 配套文档：品类知识库构建 SOP v2.0  
> 本指南解决：**去哪找数据、怎么判断质量、怎么处理**

---

## 一、数据源类型与优先级

| 优先级 | 来源类型 | 典型内容 | 适用层 | 获取难度 |
|--------|---------|---------|--------|---------|
| P0 | **GitHub 开源配表仓库** | JSON/CSV 配表（角色属性/技能/道具/关卡） | 知识表+素材表 | 低（直接 clone） |
| P0 | **游戏反编译数据仓库** | 完整游戏数据（ExcelOutput/*.json） | 素材表为主 | 低（直接 clone） |
| P1 | **深度拆解文章** | 知乎/机核/GameRes 的产品分析 | 知识表（竞品拆解） | 中（需搜索+筛选） |
| P1 | **GDC 演讲/开发者博客** | 设计理念、数值公式、开发经验 | 知识表（方法论） | 中（英文为主） |
| P2 | **Wiki/Fandom 数据库** | 角色/道具/技能完整列表 | 素材表 | 低（但需手工整理） |
| P2 | **游戏 API/数据站** | spire-archive/tboi.com 等在线数据库 | 素材表 | 低 |
| P3 | **网络搜索综合** | 零散的攻略/讨论/评测 | 知识表（补充） | 高（需交叉验证） |

## 二、GitHub 搜索方法（最重要的数据来源）

### 搜索关键词模板

```
搜索格式：{游戏英文名} + {数据类型关键词}

示例：
  "slay the spire" data json cards relics
  "hades" game data json boon weapon
  "arknights" gamedata json excel
  "genshin" data ExcelBinOutput
  "star rail" data ExcelOutput
  binding of isaac api items database
```

### 高价值仓库特征（快速判断值不值得 clone）

| 特征 | 好信号 | 坏信号 |
|------|--------|--------|
| Stars 数 | >100 | <10 |
| 最近更新 | 1 年内 | >3 年未更新 |
| 文件格式 | JSON/CSV/YAML | 纯图片/纯代码无数据 |
| 数据量 | >1MB 结构化数据 | <100KB |
| 语言 | 多语言支持 | 仅单一语言 |
| 文档 | 有 README 说明数据结构 | 无文档 |

## 三、深度拆解文章搜索方法

### 搜索平台与关键词

| 平台 | 搜索关键词模板 | 适用内容 |
|------|--------------|---------|
| **知乎** | `{游戏名} 系统拆解 数值分析 产品分析` | 竞品拆解（中文最佳来源） |
| **机核 GCORES** | `{品类名} 设计方法论` | 方法论（质量极高） |
| **GameRes** | `{游戏名} 产品模型 解析` | 商业化/系统拆解 |
| **GDC Vault** | `{game name} game design postmortem` | 开发者一手设计经验 |
| **Gamasutra/Game Developer** | `{game name} design deep dive` | 深度设计分析（英文） |

### 文章质量判断标准

| 质量等级 | 特征 | 适用 |
|---------|------|------|
| ★★★★★ | 含具体数值/公式/配表截图+作者为从业者 | 直接入知识表 |
| ★★★★ | 有系统拆解框架+竞品对比+数据支撑 | 提炼后入知识表 |
| ★★★ | 有观点但缺少数据支撑/个人经验为主 | 交叉验证后参考 |
| ★★ | 纯主观评价/攻略性质 | 不入库 |
| ★ | 营销软文/AI生成水文 | 丢弃 |

## 四、数据处理流程

```
原始数据 → 判断类型 → 分流处理

类型 A: 结构化配表(JSON/CSV)
  → 直接读取 → 提炼抽象规则入知识表
  → 保留原始明细入素材表

类型 B: 设计文档/README
  → 阅读理解 → 提炼公式/框架/分类入知识表
  → 具体数据实例入素材表

类型 C: 深度分析文章
  → 提炼核心公式/框架 → 交叉验证 → 入知识表
  → 标注来源 URL

类型 D: 代码文件
  → 提炼算法逻辑/公式 → 转化为设计语言入知识表
  → 具体参数配置入素材表
```

### 知识表 vs 素材表分流标准

| 问自己 | 答案=是 → 知识表 | 答案=是 → 素材表 |
|--------|-----------------|-----------------|
| 这是一个公式/框架/方法论吗？ | ✅ | |
| 这是一个具体角色/道具的属性值吗？ | | ✅ |
| 换一个游戏还能用吗？ | ✅ | |
| 只有这个游戏才有这些具体数字吗？ | | ✅ |

## 五、注意事项

1. **优先找有开源配表的游戏** — 有 JSON/CSV 配表的游戏，知识提取效率是纯搜索的 5~10 倍
2. **知乎文章需要登录才能看全文** — 如果搜到高价值文章但抓不到内容，让有账号的人帮忙复制
3. **GitHub 仓库先看 README** — 90% 的仓库 README 就能判断值不值得深入
4. **每个品类至少 3 款竞品深度拆解** — 头部 2~3 款 + 创新 1~2 款
5. **标注来源** — 每条知识/素材必须标注 `来源` 字段，方便溯源和更新
