# 竞品拆解模板

> 配套文档：品类知识库构建 SOP v2.0  
> 本模板解决：**怎么选竞品、怎么拆、拆哪些维度、输出什么格式**

---

## 一、竞品选择标准

每个品类选 **5~8 款**竞品，分三层：

| 层级 | 数量 | 选择标准 | 拆解深度 |
|------|------|---------|---------|
| 头部标杆 | 2~3 款 | 品类定义者+收入 TOP3 | 每款 3 条知识（战斗/系统/商业化） |
| 数据源游戏 | 2~3 款 | 有开源配表/API 可提取 | 知识表+素材表 |
| 提及级 | 2~3 款 | 创新方向/差异化代表 | 在条目中提及即可 |

## 二、拆解流程（以杀戮尖塔为例）

### Step 1: 搜索资料（30 分钟）

```
搜索顺序：
1. GitHub: "slay the spire" data json → 找到 spire-archive(361卡/181遗物全数据)
2. 知乎: "杀戮尖塔 系统拆解 数值分析" → 找到多篇深度文章
3. 专题站: spireofchaos.com → 找到卡牌价值模型(1能量≈6伤害)
4. GDC: "slay the spire GDC" → 找到开发者分享(关卡时长优化)
5. Steam: 查看好评率(97%)+评测中的玩家反馈
```

### Step 2: 阅读+标注关键数据（20 分钟）

边读边标注以下信息：
- [ ] 核心循环是什么？画出循环图
- [ ] 有哪些具体数值/公式？（伤害公式/概率/经济参数）
- [ ] 与同品类其他游戏的核心差异是什么？
- [ ] 商业模式是什么？收入数据？
- [ ] 有什么独特的设计创新？

### Step 3: 写 3 条知识条目（30 分钟）

**固定 3 条结构：**

| 条目 | 维度 | 标题格式 | 核心内容 |
|------|------|---------|---------|
| 第 1 条 | 核心系统 | 【竞品·{游戏名}核心系统拆解】 | 核心循环+系统架构+核心数据+核心创新 |
| 第 2 条 | 数值/构筑 | 【竞品·{游戏名}数值与{特色系统}拆解】 | 数值公式+构筑系统+角色差异化+平衡性 |
| 第 3 条 | 商业化/特色 | 【竞品·{游戏名}商业化/{特色}拆解】 | 付费模型+收入数据+独特设计 |

### Step 4: 写素材条目（如有配表数据）

如果该游戏有开源配表，额外写 1~3 条素材条目，保留完整的配表明细数据。

## 三、每条竞品拆解的必备要素

```
一条合格的竞品拆解 = 
  【标题含"竞品·"前缀】
  + 游戏定位一句话（品类定义者/收入TOP/创新方向）
  + 核心循环描述（动词连接：选→打→获→强化→…）
  + ≥3 个具体数值（卡牌数/角色数/收入/公式参数）
  + ≥1 个核心创新点（该游戏做了什么别人没做的）
  + ≥1 个与同品类的对比（vs 其他竞品差异化）
  + 标注来源（URL/仓库名）
```

### 自检清单

| 检查项 | 要求 |
|--------|------|
| 标题含"竞品·" | ✅ 必须 |
| 知识类型="策划" | ✅ 竞品拆解统一归类"策划" |
| 有具体数字 | ✅ 至少 3 个（无数字的拆解是废话） |
| 有差异化分析 | ✅ 这个游戏做了什么别的游戏没做的 |
| ≥150 字 | ✅ 竞品拆解需要足够的信息密度 |
| 来源可溯 | ✅ 标注 URL 或仓库名 |

## 四、拆解维度速查表

### 通用维度（所有品类都要拆）

| 维度 | 要提取的信息 |
|------|------------|
| 核心循环 | 用动词画出循环图（选→打→获→强→…） |
| 数值模型 | 伤害公式类型/成长曲线/膨胀倍率 |
| 角色/职业 | 数量/差异化方式/平衡性 |
| 商业模式 | 买断/内购/DLC/赛季制 |
| 收入数据 | 月流水/累计收入/畅销榜排名 |
| 留存机制 | D1/D7/D30 目标/具体留存设计 |
| 核心创新 | 该游戏做了什么前所未有的事 |

### SLG 专项维度

| 维度 | 要提取的信息 |
|------|------------|
| 战斗系统 | 伤害公式+兵种体系+集结机制 |
| 城建/科技 | 升级时间/解锁条件/科技树结构 |
| 联盟/社交 | 联盟系统/KvK/社交深度 |
| 赛季制 | 周期/重置规则/赛季奖励 |
| 商业化 | VIP/月卡/抽卡/加速/付费深度 |

### 肉鸽专项维度

| 维度 | 要提取的信息 |
|------|------------|
| 构筑系统 | 卡牌/boon/遗物的获取和组合方式 |
| 随机性 | 掉落概率/保底/伪随机方案 |
| 地图/关卡 | 层数/节点类型/分支路径 |
| 难度系统 | Ascension/热度/递进方式 |
| 叙事方式 | 死亡循环/事件系统/碎片叙事 |

## 五、完整示例：从零拆解一款游戏

### 目标：拆解《小丑牌》(Balatro)

**Step 1 搜索（10 分钟）**
- GitHub 搜 "balatro data" → 找到手牌/小丑/星球/塔罗配表
- 知乎搜 "小丑牌 拆解" → 找到数值分析文章
- Steam 确认：97%好评/2024年度最佳

**Step 2 标注关键数据**
- 核心循环：选小丑→打牌→凑牌型→乘分→过关→Boss盲注
- 150+小丑牌/22种牌型/8个关卡(ante)
- 四种乘区：Chips×Mult×(小丑乘区)×(行星乘区)
- 商业化：买断$14.99/无内购无DLC
- 核心创新：扑克牌型+乘法膨胀+小丑修正规则

**Step 3 输出 3 条知识**

条目 1：【竞品·小丑牌核心系统拆解】…（核心循环+架构+数据+创新）
条目 2：【竞品·小丑牌数值与乘法膨胀拆解】…（乘区体系+小丑分类+牌型评分）
条目 3：【竞品·小丑牌商业化与社区生态拆解】…（买断定价+Mod生态+种子分享）
